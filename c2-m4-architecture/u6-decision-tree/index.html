(1)

为什么需要决策树?

你现在知道 4 种架构各自的优缺点. 但面对具体需求时, 还是会纠结. 原因: 缺一个系统化的判断流程.

决策树 = 把 "纠结" 变成 "流程".

每个分支都是一个明确的 Yes/No 问题. 回答完, 自动得出推荐. 不用凭感觉.

先看决策树的 6 个问题 ↓
(2)

决策树的 6 个问题

Q1模型自带的知识够用吗?No → RAG
Q2需要溯源 (引用来源) 吗?Yes → RAG
Q3Prompt + Few-shot 效果满意吗?Yes → Prompt-only
Q4问题是 "风格/语调" 不对吗?Yes → 继续 Q5
Q5有 500+ 条标注数据 + 已 PMF?Yes → Fine-tune
Q6任务 3+ 步, 各步需要不同专长?Yes → Multi-Agent
每个问题的细节 ↓
(3)

Q1-Q2: 知识够不够? 要不要溯源?

这两个问题筛出 RAG 场景.

Q1: 模型自带的知识够用吗?

如果用户会问模型训练时不知道的事 (你的产品文档 / 最新行业报告 / 公司内部知识), 模型自带的知识不够 → RAG.

Q2: 需要溯源吗?

如果用户需要看到 "这个建议出自哪份指南", 或者合规要求必须标注信息来源 → RAG. Prompt-only 没法引用来源.

Q3: Prompt 够不够? ↓
(4)

Q3: Prompt + Few-shot 效果满意吗?

如果到了这一步, 说明知识不是问题. 那先试 Prompt.

试过才知道够不够.

很多人跳过这一步, 直接上 RAG 或 Fine-tune. 但 2026 年的模型 + 精心设计的 Prompt + 3-5 个 Few-shot examples, 能解决的问题比你想的多得多.

效果满意 → 就用 Prompt-only. 最简单就是最好.

Q4-Q5: 风格问题? ↓
(5)

Q4-Q5: 是风格问题吗? 有数据吗?

Prompt 效果不满意. 到底是什么不满意?

A

风格/语调不对

内容对, 但说话方式不像品牌要求的.

→ 有 500+ 数据 + 已 PMF → Fine-tune
→ 没有 → 继续用 Prompt, 积累数据
B

不是风格问题

可能是任务太复杂, 一步做不完.

→ 继续 Q6: 是否需要多 Agent 协作
Q6: 需要多 Agent 吗? ↓
(6)

Q6: 需要多个 Agent 协作吗?

最后一个问题, 也是门槛最高的选项.

Multi-Agent 的判断标准:

  • 任务有 3 步以上 (不是 1-2 步能搞定的)
  • 不同步骤需要不同 "专长" (搜索 / 分析 / 生成 / 审核)
  • 需要调用多种工具 (数据库 / API / 搜索引擎)

全满足 → Multi-Agent. 不满足 → 大概率用 RAG + 工具调用就够了.

完整决策树图 ↓
(7)

完整决策树一览

Q1 知识够吗? → No → RAG
Q2 要溯源吗? → Yes → RAG
Q3 Prompt 够吗? → Yes → Prompt-only
Q4 风格问题? + Q5 有数据+PMF? → Yes → Fine-tune
Q6 多步骤+多专长? → Yes → Multi-Agent
其他 → RAG + 工具调用

面试技巧: 面试官问 "你怎么选 AI 架构?", 画这棵决策树, 从 Q1 开始一步步走. 比说 "看情况" 强 100 倍.
