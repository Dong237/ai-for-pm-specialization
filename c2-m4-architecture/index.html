架构决策 (选型)

从 Prompt-only 到 RAG、Fine-tune、Multi-Agent — 4 种架构怎么选, 怎么权衡成本和效果, Vitamin 该用哪种.

1

四种 AI 架构概览

Prompt-only / RAG / Fine-tune / Multi-Agent. 复杂度递增, 成本递增 — 先看全貌再逐个拆.

→
2

Prompt-only 适用场景

任务标准化、知识不变、规模小. 80% 的初创 AI 产品只需要这个 — 别过度工程.

→
3

RAG 适用场景

知识动态更新 + 需要溯源. 医疗 / 法律 / 客服的标配. Vitamin 医学知识库一定要 RAG.

→
4

Fine-tune 适用场景

风格特殊、术语密度大、Prompt 写不下了. 一般 PMF 后才考虑 — 贵, 但真有用的时候很有用.

→
5

Multi-Agent 适用场景

任务步骤多、不同步骤需要不同专长. Vitamin 终版用 Multi-Agent 编排多个专家.

→
6

架构选型决策树

6 个 yes/no 问题, 自动推荐架构. 面试时画这棵树, 直接加分.

→
7

成本 / 效果 / 复杂度三角

每种架构在三维上的取舍. 只能优化两个, 牺牲第三个 — 这就是 PM 的决策空间.

→
8

Vitamin 架构选型

身体翻译器为什么选 RAG + Multi-Agent 而不是 Fine-tune, 完整论证过程.
