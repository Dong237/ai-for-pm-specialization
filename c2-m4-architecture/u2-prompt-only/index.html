一个真实教训: 过度工程的代价

某创业公司要做 "客服自动回复". 他们的选择:

他们做的
¥20 万
搭了完整 RAG: 向量数据库 + Embedding pipeline + 检索优化 + 3 个工程师花 6 周
其实只需要
¥200
一段精心设计的 System Prompt + Few-shot examples, 1 天搞定

为什么会这样? 因为他们的客服问题就那 50 个, 答案不怎么变. 全部写进 Prompt 的 Few-shot examples 里, 模型就能答对 95%.

这个故事来自真实案例 — 很多团队在 "该用什么技术" 上花太多时间, 忘了先问 "最简单的方案够不够".

那 Prompt-only 到底有多强? ↓
(2)

Prompt-only 的 "隐藏能力"

别小看 "只写个 Prompt". 2026 年的 Prompt Engineering 已经非常强大:

3 个让 Prompt-only 超出预期的技巧:

  1. Few-shot Examples (示例) — 给 3-5 个 input-output 示例, 模型就能学会你的模式. 不需要训练, 不需要数据库.
  2. Chain-of-Thought (思维链) — 加一句 "先分析再回答", 准确率提升 10-20%. 复杂推理也能搞定.
  3. Long Context (长上下文) — 2026 年的模型支持 100K-200K token. 知识库小于 20 万字? 直接塞进 Prompt, 比 RAG 还简单.

那 Prompt-only 适合什么任务? ↓
(3)

5 类天然适合 Prompt-only 的任务

类型 1

分类 (Classification)

情绪分类、意图识别、投诉分级. 给几个示例就行.
类型 2

改写 (Paraphrase)

正式→口语、中文→英文、长→短. 模型最擅长的活.
类型 3

摘要 (Summarize)

长文总结、会议纪要、对话要点提取.
类型 4

翻译 (Translate)

中英互译、术语保留、语气适配.
类型 5

生成 (Generate)

文案写作、邮件起草、回复建议 — 只要不需要专有知识.

共同点: 任务本身不依赖外部知识, 模型自带的能力就够.

什么任务做不了? ↓
(4)

Prompt-only 做不了什么

✗ Prompt-only 搞不定

  • 需要最新/专有知识 (模型训练截止日后的事)
  • 需要引用来源 (用户要看 "这个建议出自哪份指南")
  • 知识库太大, 塞不进 Context Window
  • 需要特殊风格, Prompt 怎么写都不像
  • 需要多步骤, 且步骤间有复杂依赖
  • 需要调用外部工具 (数据库/API/计算器)

✓ 怎么判断?

  • 问: "模型自带的知识够吗?" — 不够 → RAG
  • 问: "需要溯源吗?" — 要 → RAG
  • 问: "风格调了 Prompt 还不对?" — 不对 → Fine-tune
  • 问: "一个 Agent 能一次做完吗?" — 不能 → Multi-Agent
  • 问: "需要调用外部工具?" — 要 → Agent
  • 如果以上全是 "不需要" → Prompt-only!

怎么写一个好的 Prompt? ↓
(6)

一个好 Prompt 的 4 段结构

Anthropic 的官方建议: 每个 Prompt 都该有这 4 段.

Role "你是 Vitamin 的健康顾问, 语气温暖关怀, 使用 '你' 称呼用户."
Task "根据用户描述的身体状态, 分类到5种状态之一, 并给出一句关怀提醒."
Context "5种状态: 活力期/平稳期/疲惫期/经期/恢复期. 分类依据: ..."
Format "输出 JSON: {state: '状态名', message: '关怀提醒', confidence: 0-1}"

为什么这 4 段很重要?

Role 让模型进入角色. Task 告诉它干什么. Context 给它足够的信息. Format 约束它的输出格式. 缺了任何一段, 输出质量都会下降.

什么信号告诉你 "Prompt-only 不够了"? ↓
(7)

3 个 "该升级了" 的信号

#1
模型答错了, 而且原因是 "不知道"
用户问了训练截止后的事, 或者你们的专有知识. 模型不可能知道. → 加 RAG.
#2
Prompt 已经 4000 字了还不够
塞了太多规则/示例, 快超 Context Window 了. → 考虑 RAG 或 Fine-tune.
#3
风格就是不对, 怎么调都不像
你要的是很特殊的行文风格, Prompt 控制不住. → Fine-tune.

记住: 只有当你真的遇到这三个信号时才升级. 不要提前焦虑 "将来会不会不够" — 到时候再说.

回到 Vitamin — 哪些功能只需要 Prompt-only? ↓
(9)

Vitamin 的 Prompt-only 功能清单

这些功能, MVP 阶段全部用 Prompt-only 就够:

✓ Prompt-only 够用

  • 5 状态情绪分类 (给 5 个示例)
  • 每日健康提醒文案生成
  • 用户输入 → 关键词提取
  • 对话摘要 (最近 3 轮对话总结)
  • 食谱建议 (基于通用营养知识)

✗ 需要升级

  • 医学知识问答 → RAG
  • 药物交互查询 → RAG + 规则
  • 品牌语调统一 → Fine-tune (后期)
  • 身体翻译器全流程 → Multi-Agent

"Vitamin 的 MVP 阶段, 我会先用 Prompt-only 实现 5 个核心功能, 验证用户需求. 只有在验证完成、确认需要专有医学知识时, 才升级到 RAG. 这样 time-to-market 从 2 个月压缩到 2 周."
