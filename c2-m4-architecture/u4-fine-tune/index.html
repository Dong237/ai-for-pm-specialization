RAG 解决了知识问题, 但语气还是不对.

Vitamin 要的是 "闺蜜式关怀", 不是 "百科全书式".

(1)

Fine-tune 的本质: 改 "说话习惯"

一个类比:

RAG = 给实习生一本参考书 (改输入)

Fine-tune = 把实习生送去培训 3 个月 (改模型)

培训完, 他不用再翻书, 因为 "习惯" 已经内化了.

Fine-tune 改的是模型的行为模式:

  • 说话风格 (正式/口语/闺蜜式/学术式)
  • 术语使用习惯 (医学术语/金融术语/特定行业黑话)
  • 判断倾向 (保守/激进/特定领域的决策逻辑)
  • 输出格式偏好 (JSON/表格/特定模板)
最常见的误解来了 ↓
(2)

最大误解: Fine-tune ≠ 教新知识

✗

"我们的产品信息, 模型不知道, 所以要 Fine-tune"

错! 这是 RAG 的活.

Fine-tune 教的是 "怎么说", 不是 "知道什么". 用 Fine-tune 教新知识 = 用锤子拧螺丝.
✗

"Fine-tune 了就不会幻觉了"

错! Fine-tune 可能让幻觉更自信.

Fine-tune 让模型更 "像" 你要的风格, 但不保证内容正确. 降幻觉靠 RAG + 规则, 不靠 Fine-tune.
记住

知识变了 → RAG
风格变了 → Fine-tune

行业共识: "把变化的知识放检索, 把稳定的行为放权重, 别搞反了."

什么时候真的需要 Fine-tune? ↓
(3)

4 个 "需要 Fine-tune" 的检查项

✓ 检查 1

Prompt-only 先试过了?

如果连 Prompt-only 都没试, 别谈 Fine-tune. 很多人以为需要微调, 其实 Few-shot 就够了.
✓ 检查 2

问题是 "风格/语调" 而非 "知识"?

如果模型答的内容对但语气不对 → Fine-tune. 如果内容就错了 → RAG.
✓ 检查 3

有 500+ 条标注数据?

Fine-tune 需要高质量的 input→output 对. 数据不够, 效果不好甚至变差.
✓ 检查 4

产品已经 PMF?

PMF 前需求会变, 训了白训. PMF 后风格确定了, Fine-tune 才有长期价值.

4 个都是 Yes → Fine-tune. 任何一个 No → 先不要.

数据准备是最花时间的.

每条数据 = 一个 "input → 理想 output" 对. 比如: "用户说 '好累' → 理想回复: '今天辛苦了~ 要不要喝杯热水, 休息一下? 🫶'". 质量比数量重要.

2026 年好消息: LoRA (Low-Rank Adaptation) 技术让微调成本下降了 10 倍. 你不需要训练整个模型 — 只调少量参数, 效果差不多.

GPT-4 的完整训练据估计花了 1 亿美元. 但 LoRA 微调一个 7B 模型, 可能只要 500 块人民币.

LoRA 是什么? ↓
(6)

LoRA: 微调不用倾家荡产

传统 Fine-tune 要改模型全部参数 (几十亿个). LoRA 只改 0.1%-1% 的参数.

类比:

传统微调 = 把整栋楼拆了重建
LoRA = 只重新装修一间屋子

效果差不多, 但成本低 10 倍, 时间快 5 倍.

PM 需要知道的:

  • LoRA 让微调从 "天文数字" 变成 "几千块" — 初创公司也能用
  • QLoRA 更进一步: 用量化技术, 一张消费级 GPU 就能跑
  • 这意味着 "要不要 Fine-tune" 的门槛降低了, 但 "该不该 Fine-tune" 的判断标准没变
什么时候不要 Fine-tune? ↓
(7)

4 个 "不要 Fine-tune" 的反指标

✗ 这些情况别 Fine-tune

  • "模型不知道我们的产品" → 这是 RAG 的活
  • "数据只有 50 条" → 太少, 效果不好
  • "产品还在 MVP 阶段" → 需求会变, 训了白训
  • "想让模型完全不犯错" → Fine-tune 不能消灭幻觉

✓ 这些情况考虑 Fine-tune

  • 品牌语调是核心竞争力
  • 术语密度大, Few-shot 不够
  • Prompt 已经超过 4000 字还不够
  • 推理成本需要优化 (小模型 + Fine-tune)

(9)

Vitamin 的 Fine-tune 规划

Vitamin 现在 (MVP 阶段) 不需要 Fine-tune. 但 PMF 后, 品牌语调 是 Fine-tune 的候选.

Vitamin Fine-tune 时间表:

  • 现在 用 Prompt-only + Few-shot 控制语气
  • PMF 后 积累 1000+ 条 "理想回复" 数据, 开始标注
  • 规模化后 用 LoRA 微调一个小模型, 专门处理 Vitamin 风格的对话

面试金句:

"Fine-tune 我放在 PMF 后. 原因: 一, 现阶段 Prompt + Few-shot 已经能控制语气到 80 分; 二, 需求还在变, 训了可能白训; 三, 等积累够数据再做, 效果更好成本更低."
