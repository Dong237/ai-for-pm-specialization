(1)

"我记得你之前说过..." — AI 经常在瞎编

看这段 Vitamin 的对话, 问题出在哪里:

#1
用户
你好, 我想用 Vitamin 管理健康.
#2
AI
欢迎! 请告诉我你的基本情况, 我来为你定制健康计划.
#3
用户
我 28 岁, 经期一般 28 天周期. 另外, 我对虾过敏, 吃了会严重起疹子.
#4
AI
收到! 28 天周期很规律. 我会记住你的过敏信息.
... (中间经过了 10 轮关于经期、运动、睡眠的对话) ...
#15
用户
今天黄体期第 3 天, 推荐点晚餐吧.
#16
AI (危险!)
黄体期建议多补充蛋白质! 推荐: 蒜蓉虾仁炒西兰花, 配糙米饭, 富含...

发生了什么? 用户在第 3 条消息明确说了"对虾过敏". 但到了第 15 条, 中间隔了 12 轮对话, AI 推荐了虾仁. 过敏信息在 Context Window 的中间位置, 被 AI "遗忘"了.

对健康产品来说, 这不是体验问题, 是安全问题.

为什么会这样? 不是说 128K Context Window 能装很多吗? ↓
(2)

128K 的容量 ≠ 128K 的利用率

✗

错觉: "Context Window 越大, 记忆力越强"

128K 够装 20 万字, 所以 AI 能完美记住这 20 万字的每个细节.

为啥不对: 能"装下"不代表能"注意到". 就像你的书架能放 200 本书, 但你常翻的只有最上面和最下面那几本. 中间的积灰了.
✗

错觉: "AI 平等对待每一条消息"

所有信息在 Context Window 里权重一样.

为啥不对: Transformer 的注意力机制对位置敏感. 开头和结尾的信息天然获得更多注意力权重, 中间的信息被压缩.
真相

Stanford & Google 发现了一个 U 型曲线:

开头记得清, 结尾记得清, 中间会模糊

这就是 "Lost in the Middle" 效应 — 2023 年 Stanford 最有影响力的 LLM 研究之一.

→ 论文: Liu et al., "Lost in the Middle: How Language Models Use Long Contexts", Stanford & UC Berkeley, 2023.

具体是什么样的 U 型曲线? ↓
(3)

U 型注意力 — 看这张图就懂了

研究者在文档的不同位置放一个关键事实, 测试 AI 能不能找到它. 结果:

← 开头 (位置 1) 中间 (位置 10-11) 结尾 (位置 20) →

纵轴 = AI 找到关键信息的准确率. 中间位置低到只有 ~42%.

跟人类很像!

心理学上有个 "序列位置效应" (Serial Position Effect): 让人记一串词, 开头的记得最牢 (首因效应), 结尾的也记得清 (近因效应), 中间的模糊.

AI 的 Transformer 架构也表现出这个效应 — 虽然原因不完全一样, 但结果惊人地相似.

为什么 Transformer 会这样? ↓
(4)

原因: 注意力权重天然偏向两头

下面 20 个方块代表 Context Window 里的 20 个位置. 颜色越深 = AI 给的注意力越多:

高注意力
中注意力
低注意力

为什么会这样? 两个原因:

1. 开头的"首因优势" (Primacy):

Transformer 使用 causal masking — 每个 token 只能看到它前面的 token. 所以开头的 token 被后面所有 token 注意到, 累积的注意力权重最高.

2. 结尾的"近因优势" (Recency):

最新的 token 离输出位置最近, 在模型的"工作记忆"中最鲜活. 就像你读完一封长邮件, 最后几句话印象最深.

中间 = "双重劣势": 既不是被所有后续 token 关注的开头, 也不是最新鲜的结尾. 注意力权重在这里最低 — 这就是信息的"记忆黑洞".

这对产品意味着什么? ↓
(5)

对产品的影响: 多轮对话的"危险地带"

在一个 20 条消息的对话中, AI 对不同位置信息的记忆力:

1-3
4-6
7-14
15-17
18-20
🟢 安全区 (开头) 🔴 危险区 (中间) 🟢 安全区 (结尾)

Vitamin 的危险场景:

  • 用户在第 3 条说了过敏信息 → 到第 15 条已经在"中间"了
  • 用户在第 5 条提到正在吃某种药 → AI 第 18 条给建议时忘了药物交互
  • 用户早期说过"我在备孕" → AI 后来推荐了备孕禁忌的运动

这些都不是假设 — 是真实会发生的问题.

怎么解决? 4 个策略 ↓
(6)

四种缓解策略 — PM 必知

1

关键信息放开头

过敏、用药、身体状态等核心信息, 放在 System Prompt 的最前面. 开头是注意力最集中的位置.

2
关键信息在末尾重复

在每次 API 调用时, 在用户最新消息之前重新附上关键信息. "提醒: 用户对虾过敏."

3

用摘要代替完整历史

不要把 20 轮对话全塞进去. 每隔几轮做一次摘要, 只保留关键事实 + 最近 3-5 轮原文.

4

精简上下文, 不要过度填充

Context Window 不是越满越好. 塞太多无关信息反而稀释关键信息的注意力权重.

Anthropic 的官方建议: "Place the most important directives at the beginning to set the context or at the end to keep them fresh." (把最重要的指令放在开头定调, 或放在结尾保持新鲜.)

→ 来源: Claude Prompting Best Practices

"大海捞针" 测试是怎么做的? ↓
(7)

Needle-in-a-Haystack 测试 — 业界标准

Greg Kamradt 发明了一个著名测试: 在不同长度的文档的不同位置放一句关键信息 (needle), 看 AI 能不能找到它.

↑ 上下文越长 (4K → 128K)
← 开头 Needle 位置 (深度) 结尾 →

绿色 = 找到了 · 黄色 = 勉强 · 红色 = 丢失了. 中间位置 + 长上下文 = 红色重灾区.

关键发现:

  • 短上下文 (4K-8K): 各位置表现都不错
  • 中等上下文 (32K): 中间位置开始出现明显下降
  • 长上下文 (128K): 中间位置准确率可能低于 50%
  • 即使号称支持 128K 的模型, 实际有效利用率远低于 100%

→ 这就是为什么"Context Window 大" 不能作为模型选型的唯一标准.

动手试 — Needle Finder

下面 20 个方块代表一篇长文档. 点击任意位置, 把"关键信息" (📌) 放进去, 看 AI 找到它的概率:

AI 注意力分布 (每个位置的关注度):
试试把 📌 放在最中间 (位置 10-11), 再放到开头/结尾对比

你会发现:

  • 位置 1-3: 准确率 85-92%, 安全
  • 位置 9-12: 准确率 42-48%, 危险
  • 位置 18-20: 准确率 85-91%, 安全

这意味着: 如果用户的过敏信息恰好落在对话的"中间地带", AI 有将近一半的概率忽略它.

Vitamin 怎么应对这个问题? ↓
(9)

Vitamin 的解法: 结构化"状态卡"

不要依赖对话历史来传递关键信息. 每次 API 调用时, 在 System Prompt 开头放一个结构化状态卡:

<user_state> <basic> 年龄: 28 | 周期: 28天 | 当前阶段: 黄体期第3天 </basic> <allergies> 虾 (严重, 会起疹子) </allergies> <medications> 叶酸 0.4mg/天 </medications> <goals> 调理月经 · 改善睡眠 </goals> </user_state>

这个状态卡解决了什么:

  • 过敏信息永远在 Prompt 开头 — 注意力最集中的位置
  • 不依赖 AI "记住"历史对话 — 每次调用都显式提供
  • 用 XML 标签结构化 — 比纯文本更容易被 AI 解析
  • 信息可以从数据库动态生成 — 不占对话历史的 token

✓ 状态卡应该包含

  • 过敏信息 (食物、药物)
  • 当前身体状态 (经期/备孕/孕期/产后/更年期)
  • 正在服用的药物
  • 重要健康目标
  • 近期异常症状

✗ 不应该塞进状态卡

  • 完整对话历史 (太长, 适得其反)
  • 不影响安全的偏好 (最喜欢的颜色)
  • 已经解决的历史问题
  • 重复冗余的信息
  • 大段未结构化的笔记
一句话规则 + 作业 ↓
(10)

一句话规则 + 今晚作业

一句话:

Context Window 大 ≠ 能用.
重要信息放开头或结尾,
中间是记忆黑洞.

别靠 AI 的"记性", 靠你的 Prompt 结构设计.

课后作业 (30 分钟):
  1. 打开 ChatGPT / 豆包, 先在第 1 条消息里告诉它一个"关键信息" (比如"我对花生过敏"). 然后随便聊 10-15 轮无关话题. 最后问一个跟关键信息相关的问题, 看它还记不记得.
  2. 把同样的实验重做, 但这次把关键信息放在最后一条消息的前面重复一遍. 比较两次结果.
  3. 为 Vitamin 设计一个"用户状态卡" XML 模板, 包含你认为最重要的 5-8 个字段. 想想: 哪些信息被遗忘了会有安全风险?
  4. 用 200 字给非技术朋友解释: 为什么 AI 的"记忆"不是均匀分布的, 为什么重要的事情要放在对话的开头.
