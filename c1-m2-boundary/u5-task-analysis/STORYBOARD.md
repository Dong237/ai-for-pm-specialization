## (1) "Add AI to everything!" — but which features first?
Boss wants AI everywhere. You have limited resources. Need a framework.

## (2) Wrong intuitions vs truth
Wrong A: "做最酷的 AI 功能" → 可能技术不可行
Wrong B: "做最简单的 AI 功能" → 可能没商业价值
Truth: 同时评估两个维度 — Feasibility AND Value

## (3) Feasibility axis — what makes AI technically feasible
5 sub-factors: data availability, model capability, accuracy requirements, integration complexity, cost

## (4) Value axis — what makes an AI feature valuable
5 sub-factors: user impact, business value, competitive advantage, frequency of use, strategic alignment

## (5) The 4 quadrants explained
Star (high-high), Question Mark (high value low feasibility), Quick Win (high feasibility low value), Don't Bother (low-low)

## (6) How to score: 1-5 scale
Concrete scoring criteria for each axis. Examples.

## (7) Common mistakes
Overestimating feasibility. Underestimating value of simple features. Tech-driven vs user-driven.

## (8) WIDGET: 2×2 Matrix
8 Vitamin feature cards draggable into 4 quadrants. Each card shows name + short description.
Quadrants: Star / Question Mark / Quick Win / Don't Bother

## (9) Vitamin: plot 8 features, identify Stars
Which features are Stars? Discussion of priorities.
