为什么能差这么多? ↓
(2)

为什么价格差这么大?

三个原因叠加:

原因 1

架构效率: MoE

DeepSeek V4: 1.6T 总参数, 但每次推理只激活 49B (3%). 省算力 = 省钱. GPT-5.5 是密集模型, 每次调用要跑全部参数.
原因 2

商业策略

DeepSeek: MIT 开源, 用低价抢市场. OpenAI: 闭源垄断, 高毛利补贴研发. 两种完全不同的商业模式.
原因 3

训练成本分摊

DeepSeek 用华为 Ascend 芯片训练, 成本更低. OpenAI 用 NVIDIA H100, 加上更大的研发团队, 固定成本更高.

PM 视角: 价格差 ≠ 能力差. DeepSeek V4 在 GPQA Diamond 上得分 90.1%, GPT-5.5 得 93.6%. 差 3 个百分点, 价格差 107 倍. 对大部分产品来说, 3% 的能力差距远不如 107x 的成本差距重要.

Input 和 Output 为什么价格不同? ↓
(4)

Input vs Output — 为什么 Output 更贵?

Input (你发给 AI 的)
  • System prompt
  • 用户消息
  • 历史对话
计算量: 中等
一次性读完, 可以并行处理
Output (AI 回复你的)
  • AI 生成的回答
  • 一个 token 一个 token 生成
  • 每个 token 都要跑一遍模型
计算量: 更大
串行生成, 每个 token 都要推理一次

Vitamin 的 IO 比例:

用户输入 (问题): ~100-200 tokens → Input
System prompt + 历史: ~300-500 tokens → Input
AI 回复: ~200-400 tokens → Output

→ 大约 Input:Output = 3:2. 但 output 单价贵 3-6 倍, 所以output 占总成本的 60-80%.

知道了单价, 怎么算月度总成本? ↓
(5)

月成本公式 — 4 个数字相乘

算 AI API 月成本, 就是一个乘法:

月成本 =
DAU × 日调用次数 × 单次 tokens × 单价 × 30 天
Vitamin 示例:
  • DAU: 1,000 (早期产品)
  • 日调用: 5 次/人 (早餐建议 + 情绪分析 + 日记总结 + 2 次问答)
  • 单次 tokens: ~500 (input 300 + output 200)
  • 月总 tokens: 1,000 × 5 × 500 × 30 = 75,000,000 = 75M tokens

→ 其中约 45M input + 30M output (按 3:2 比例)

还有一个省钱大招: cache hit ↓
(7)

Cache Hit — 再省 80% 的秘密武器

当你的 System Prompt 每次调用都一样 (Vitamin 就是这样), 可以用缓存:

DeepSeek V4 Flash 正常 input
$0.14 / M
DeepSeek V4 Flash cache hit
$0.028 / M
省 80%!
怎么触发 cache hit?
  1. 固定 System Prompt — Vitamin 的人设、健康知识库、回复格式要求, 这些每次调用都一样
  2. Prompt 前缀不变 — 把固定内容放在 prompt 最前面, 变化内容 (用户消息) 放最后
  3. 频繁调用 — 调用越频繁, 缓存命中率越高

→ Vitamin 1000 DAU, 每天 5000 次调用, System Prompt 占 input 的 60%. 如果全部缓存命中, input 成本再降 ~50%.

Anthropic、OpenAI、DeepSeek 都支持 Prompt Caching. Google Gemini 有类似的 Context Caching 功能. 这是 PM 必须知道的省钱策略.
