(1)

同一个功能, 国内 vs 出海, 选模型完全不同

想象两个版本的 Vitamin:

🇨🇳

Vitamin 国内版

服务中国用户, App Store 中国区上架

必须: 模型备案, 数据境内存储, 内容审核符合国内标准.
推荐: DeepSeek / Qwen / Doubao / Kimi — 天然合规, 中文优.
🌏

Vitamin 海外版

服务东南亚/欧美华人, Google Play 全球上架

必须: GDPR 合规 (如进欧洲), 多语言支持, 全球 API 可用性.
推荐: GPT / Claude / Gemini — 全球部署, 多语言强.

这不是"哪个更好"的问题, 是"你的市场在哪"的问题.

决策的 3 个核心因素 ↓
(2)

决策的 3 个核心因素

因素 1

合规 Compliance

国内有大模型备案制度, 数据不出境要求, 内容审核标准. 不合规 = 不让上线.
因素 2

中文能力

做中文产品, 中文能力是刚需. 国产模型在方言、网络用语、文化语境上有优势.
因素 3

成本 + 可用性

国产模型用 RMB 计价通常更便宜. 国际模型在海外延迟更低、API 更稳定.
常见错误:

"国际模型一定更好" — 错. 在中文场景, Qwen/Doubao/Kimi 的中文能力不输 GPT/Claude. 在合规场景, 国产模型是唯一选择.

先看路径 A: 纯国内产品 ↓
(3)

路径 A: 纯国内产品 — 合规优先

如果你的产品只在中国市场运营, 选模型的首要考虑是合规:

路径 A · 国内产品

合规要求 (截至 2026 年 4 月):

  • 大模型备案 — 网信办要求, "属地初审 + 中央终审". 截至 2025 年底已有 748 款模型完成备案.
  • 数据境内存储 — 用户数据不能出境. 境外数据占训练语料不得超过 30%.
  • 内容安全 — 敏感问题拒答率 ≥ 95%, 需建立 "AI 过滤 + 人工复审" 双重机制.
  • 隐私保护 — 符合《个人信息保护法》, 健康数据属于敏感个人信息.

推荐模型:

DeepSeek V4 Qwen3.6-Max Doubao-Seed-2.0 Kimi K2.6 GLM-5

这些模型已完成备案或由备案企业运营, 数据中心在国内, 天然满足合规要求.

路径 B: 出海产品 ↓
(4)

路径 B: 出海产品 — 全球可用 + 多语言

如果你的产品面向海外市场 (东南亚/欧美), 考虑点不同:

路径 B · 出海产品

关键考虑:

  • 全球 API 可用性 — 需要在目标市场延迟低、服务稳定.
  • 多语言支持 — 英语/日语/韩语/印尼语等, 国际模型覆盖更广.
  • 当地合规 — 欧洲 GDPR, 美国各州隐私法等.
  • 无中国合规限制 — 不需要大模型备案, 选择自由度更大.

推荐模型:

GPT-5/5.5 Claude Sonnet/Opus Gemini Llama 4 (自部署)

全球数据中心, API 覆盖广, 多语言能力强. Llama 4 开源可自部署到任何地区.

国产模型出海?

MiniMax 海外收入占比超 70%, Kimi 也在积极出海. 但面临: 算力瓶颈 (Huawei Ascend 海外供应受限), 地缘政治合规, 多语言能力仍需提升. 目前国产模型出海主要走 to-B 路线, to-C 还是国际模型为主.

路径 C: 两个市场都做 ↓
(5)

路径 C: 两者兼顾 — Hybrid 架构

如果产品同时服务国内和海外, 用 Hybrid 方案:

路径 C · 两者兼顾
你的产品代码
API 抽象层 (统一接口)
中国区
DeepSeek / Qwen
数据存储: 国内
合规: 已备案
海外区

实现方式: 代码里只调一个统一接口, 根据用户 region 路由到不同模型. 用 LiteLLM 等工具可以几行代码搞定.

详细看看合规要求 ↓
(6)

合规 Checklist — PM 必须知道的 4 件事

在中国做 AI 产品, 这 4 项合规是硬性要求, 不是 "nice to have":

1

大模型备案

向网信办提交安全评估报告 (30-100 页), 语料标注规则, 评估测试题集. "属地初审 + 中央终审".

用已备案模型 (DeepSeek/Qwen 等) 的 API = 不需要自己备案.

2

数据不出境

用户数据必须存储在中国境内服务器. 调用海外 API (如 OpenAI) = 数据出境, 需做数据出境安全评估.

Vitamin 用户健康数据属于敏感信息, 出境风险更高.

3

内容审核

31 类安全风险需要覆盖. 敏感问题拒答率 ≥ 95%. 人工审核率 ≥ 10%.

国产模型已内置合规过滤, 比自己做省力.

4

隐私政策

符合《个人信息保护法》. 需要用户明确同意. 健康/生理数据是敏感个人信息, 需要单独授权.

Vitamin 的经期/身体数据需要特别注意.

PM 结论: 如果你用国产模型的 API (如 DeepSeek API / 通义千问 API), 备案和内容审核已经由模型提供方搞定了. 你只需要管好数据存储和隐私政策. 这是最省事的方式.

中文能力: 国产 vs 国际, 谁更懂中文?

SuperCLUE 中文榜单 (截至 2026 年 3 月):

闭源模型 Top:

  1. Claude Opus 4.6 — 国际模型中文最强
  2. Gemini 3.1 Pro
  3. GPT-5.4
  4. Doubao-Seed-2.0 — 国产闭源最强

开源模型 Top:

  1. Kimi K2.5-Thinking — 开源中文最强
  2. Qwen3-Max
  3. DeepSeek
  4. GLM
意外发现:

Claude Opus 在中文榜上排名第一 — 国际模型的中文已经非常好了. 但在特定中文场景下, 国产模型仍有优势:

  • 方言/网络用语: "姨妈来了" = 月经, 国产模型更准
  • 文化背景: 中医概念、农历、节气, 国产模型理解更深
  • 合规内容: 国产模型的合规过滤更符合国内标准
