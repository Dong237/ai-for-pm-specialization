AI 版 PRD
4 模块写法

传统 PRD 描述 "点击什么 → 发生什么". AI PRD 描述 "概率性输出 + 兜底 + 评测". 8 个单元, 从零写出一份完整的 AI 产品需求文档.

8 单元
~80 分钟阅读
8 互动小工具
1

传统 PRD vs AI PRD 的本质差异

传统 PRD 写 "点击什么 → 发生什么", AI PRD 多了一个不确定性维度: 概率性输出 + 兜底 + 评测.

→
2

AI PRD 模块 1: 模型 IO 描述

Input Schema + Output Schema = 给算法工程师看的合同. 定义清楚 "什么进去, 什么出来".

→
3

AI PRD 模块 2: 效果指标定义

准确率 / 接受率 / 拒答率 / 平均延迟. 每个指标的目标值, 不达标怎么兜底.

→
4

AI PRD 模块 3: 错误兜底机制

AI 答错了怎么办? 5 类兜底: 重试 / 切模型 / 切规则 / 转人工 / 直接拒答.

→
5

AI PRD 模块 4: Prompt 模板管理

Prompt 也是产品资产. 版本号 / 变更日志 / A/B 实验 / 灰度发布.

→
6

PRD 中的成本预算栏

单次调用 token 数 × 模型单价 × 调用频率 = 月成本上限. 算清楚再开工.

→
7

PRD 中的数据来源 + 隐私要求

训练 / 检索 / 日志数据从哪来, 保留多久, 谁能看. 合规不是事后打补丁.
