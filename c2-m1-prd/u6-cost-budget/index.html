每次调用都在烧钱

传统功能: 服务器固定成本. AI 功能: 每次 API 调用都按 token 收费. 1 万 DAU, 一个月可能烧掉几千到几万美元.

财务问: "这个 AI 功能一个月要花多少钱?"

我说 "不知道, 要看用户用多少..."

这个答案不能接受. 必须在 PRD 里写清楚.

(1)

传统功能: 服务器包月. AI 功能: 按次收费.

传统 App 的服务器: 固定月费, 用户多了加服务器. 成本可预测.

AI 功能: 每一次 API 调用都按 token 数计费. 用户越多, 成本线性增长. 如果不提前算, 可能月底收到一张天价账单.

真实案例: 某 App 上线 AI 聊天功能, 没限制调用频率. 用户发现可以无限聊天, 第一个月 API 账单: $47,000. 预算: $5,000.

计费模型是什么? ↓
(2)

LLM API 计费: Input Token + Output Token

几乎所有 LLM API 都按 token 计费, 分两部分:

Input Token (输入): 你发给模型的所有文字 (Prompt + 上下文 + 用户问题). 相对便宜.

Output Token (输出): 模型生成的回答. 比 Input 贵 2-5 倍.

模型Input ($/1M tokens)Output ($/1M tokens)适用场景
GPT-4o$2.50$10.00高质量场景
GPT-4o-mini$0.15$0.60性价比之选
Claude Sonnet$3.00$15.00长文本/分析
Claude Haiku$0.25$1.25轻量级任务
DeepSeek V3$0.27$1.10中文优化
公式很简单 ↓
(3)

成本公式: 一句话搞定

月成本 = (Input token × 输入单价 + Output token × 输出单价)
× DAU × 人均调用次数 × 30 天

举例: Vitamin 每日总结, 用 GPT-4o-mini:
Input 500 token × $0.15/1M + Output 300 token × $0.60/1M = $0.000255/次
10,000 DAU × 1 次/天 × 30 天 = 300,000 次
300,000 × $0.000255 = $76.5/月

看起来不多? 但如果用 GPT-4o, 同样计算: $4,000/月. 模型选择直接决定成本 50 倍的差异.

怎么在 PRD 里写? ↓
(4)

PRD 成本预算栏的格式

在 PRD 里, 成本预算栏写成一个表:

项目值
模型GPT-4o-mini
Input token/次~500
Output token/次~300
单次成本$0.000255
DAU10,000
人均调用/天1
月调用量300K
月成本~$77
月预算上限$200
超标处理限制调用频率 / 切更小模型

最后两行是关键: 预算上限 + 超标处理. 这是给财务和技术的 "安全阀". 没有上限 = 敞口风险.

降成本的方法 ↓
(5)

5 种降成本方法

1. 选小模型: 不是所有场景都需要 GPT-4o. 情绪分类用 mini 就够, 成本降 95%.

2. 缩短 Prompt: 精简 System Prompt 从 1000 token 到 300 token. 每次调用少 700 input token.

3. 加缓存: 同类问题的回答缓存 24 小时. "今天天气怎么影响运动" — 同一天不需要重复问模型.

4. 限制频率: 每用户每天最多 5 次 AI 问答. 超出后展示 "今日额度已用完, 明天再来".

5. 分级调用: 简单问题用小模型, 复杂问题才路由到大模型. 80% 的请求可以用便宜模型搞定.

成本和质量的权衡 ↓
(6)

成本 vs 质量: PM 的决策

降成本不是免费的. 每种方法都有代价:

方法省多少代价
选小模型80-95%准确率可能降 5-15%
缩短 Prompt20-40%可能丢失有用的上下文
加缓存30-60%回答可能不够实时/个性化
限制频率取决于限额影响用户体验
分级调用50-70%需要额外的路由逻辑开发

PM 决策: 在 PRD 里明确: "本功能优先保证准确率 ≥ 90%, 在此前提下尽量降低成本. 如果月成本超过预算, 优先限制频率, 其次考虑切小模型."

成本监控 ↓
(7)

上线后的成本监控

PRD 里还要写: 上线后怎么监控成本:

监控内容:

  • 每日 API 调用次数 + 总 token 消耗
  • 单次平均 token 数 (是否有异常大的请求)
  • 日成本 / 周成本趋势图
  • 预算消耗进度 (月预算用了百分之多少)

告警规则: 日成本 > 预算/30 的 150% → 告警给 PM + 技术 lead. 月成本预计超过预算 → 自动限制新用户的调用频率.
