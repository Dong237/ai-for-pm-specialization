回顾: AI PRD 的完整结构

先看全貌, 再逐块填写. Vitamin PRD v2 一共 10 个模块:

1. 需求背景

为什么做, 解决什么问题

2. 功能描述

用户视角的功能说明

3. 交互流程

UI 状态 + 加载态 + 兜底态

4. IO Schema

Input/Output 的字段和约束

5. 效果指标

准确率/接受率/拒答率/延迟

6. 兜底机制

错误时的 5 级降级链

7. Prompt 管理

版本号/审批/A/B/回滚

8. 成本预算

token × 单价 × 频率 = 月成本

9. 数据隐私

来源/传输/存储/权限

10. 验收标准

怎么算 "做完了"

蓝色 是传统 PRD 共有的. 绿 / 红 / 黄 / 紫 是 AI PRD 新增的.

开始填写 ↓
(2)

模块 1-3: 传统部分 (你已经会了)

1. 需求背景: Vitamin 用户每天记录健康数据 (睡眠/心率/饮食/运动/经期), 但缺少一个 "每日总结" 功能. 用户不想自己分析数据, 希望 AI 帮忙做总结. 使用 AI 而非模板, 因为每个人的数据组合不同, 需要个性化生成.

2. 功能描述: 用户打开首页, 看到 AI 生成的每日健康总结卡片. 内容包括: 今日核心数据、7 天趋势对比、1-2 条个性化建议.

3. 交互流程: 首页加载 → 骨架屏 (AI 生成中) → 总结卡片展示 → 可点击展开详情 → 底部 "不准确" 反馈按钮.

这 3 个模块你已经会写了. 关键是接下来的 AI 专属模块.

IO Schema ↓
(3)

模块 4: IO Schema (Unit 2 复习)

Input Schema:

{ user_id: string (必填), date: string (必填, ISO 8601), metrics: array (必填, 至少 1 项), body_state: string (选填, 5 状态之一), history_days: number (选填, 默认 7, 范围 1-30), max_tokens: number (选填, 默认 300, 范围 50-500) }

Output Schema:

{ summary_text: string (必填, 50-300 字), confidence: number (必填, 0.0-1.0), tags: array (选填, 0-5 个标签), suggestions: array (选填, 0-3 条建议), risk_flag: boolean (选填, 默认 false), token_count: number (选填) }

效果指标 ↓
(4)

模块 5: 效果指标 (Unit 3 复习)

准确率: ≥ 90% (通过人工标注评测集, 周度抽检 50 条)

接受率: ≥ 85% (通过前端埋点, "不准确" 按钮点击率 < 15%)

拒答率: ≤ 5% (模型返回拒答标记的比例)

P95 延迟: ≤ 3 秒 (首页总结从请求到展示)

不达标处理: 准确率 < 80% → 自动切模板兜底 + 告警. P95 延迟 > 5s → 展示缓存的昨日总结.

兜底机制 ↓
(5)

模块 6: 兜底机制 (Unit 4 复习)

触发条件兜底用户看到
confidence < 0.85L1 重试加载动画 (无感知)
重试仍 < 0.85L3 模板"基础版总结" 标签
超时 > 5sL3 缓存"先看看昨天的总结"
涉及医疗禁忌L5 拒答"建议咨询医生"
Prompt 管理 ↓
(6)

模块 7: Prompt 管理 (Unit 5 复习)

当前版本: v2.0 (全量上线)

审批流程: PM 起草 → 算法 Review → 评测集跑分 → PM + 算法双签

发布策略: 小修 (v2.x) 可热更新; 大改 (v3.0) 灰度 7 天

回滚: 保留最近 5 个版本, 一键回滚

A/B: 重大改动先 A/B 7 天, 以接受率为主指标

成本预算 ↓
(7)

模块 8-9: 成本 + 隐私 (Unit 6-7 复习)

成本预算:

模型: GPT-4o-mini | Input: ~500 token | Output: ~300 token | 单次: $0.000255

DAU: 10,000 | 频率: 1 次/天 | 月成本: ~$77 | 预算上限: $200

超标处理: 限制低活跃用户频率 → 切更小模型

数据隐私:

数据来源: 用户记录的健康指标 (自有数据库)

传输: 脱敏后传给 API, HTTPS 加密, API 承诺不用于训练

保留: 日志 30 天, 反馈 90 天, 到期自动删除

用户权利: 可查看/删除, 设置页 "我的数据" 入口

动手: 填写你自己的 Vitamin PRD v2

展开每个模块, 用你自己的话填写. 写完后可以预览完整 PRD 并复制:

Vitamin PRD v2 — 每日健康总结 (你的版本)
1 需求背景▶

为什么做这个功能? 为什么用 AI 而不是规则?

2 功能描述▶

用户看到什么? 能做什么?

3 交互流程▶

加载态、成功态、兜底态分别是什么?

4 IO Schema▶

Input 哪些字段? Output 什么格式? 每个字段的类型和约束?

5 效果指标▶

4 个指标的目标值和不达标处理?

6 兜底机制▶

每种错误场景的兜底链和用户文案?

7 Prompt 管理▶

当前版本、审批流程、发布策略、回滚机制?

8 成本预算▶

模型选择、token 数、单次成本、月成本上限?

9 数据隐私▶

数据来源、传输方式、保留时长、用户权利?

10 验收标准▶

怎么算 "做完了"? 上线前需要通过什么检查?

评审视角 ↓
(9)

用评审视角检查你的 PRD

写完了, 用这个 checklist 自检:

  • 需求背景说清楚了 "为什么用 AI 而非规则"?
  • IO Schema 每个字段都有类型和约束?
  • 4 个效果指标都有目标值?
  • 每种错误场景都有对应的兜底和文案?
  • Prompt 有版本号和审批流程?
  • 成本预算有公式有数字有上限?
  • 数据隐私写清楚了 "传给谁" 和 "保留多久"?
  • 验收标准是可量化的, 不是 "差不多就行"?
面试金句:

"我用 10 个模块的结构写 AI PRD: 3 个传统模块 (需求/功能/交互) + 4 个 AI 核心模块 (IO/指标/兜底/Prompt) + 成本预算 + 数据隐私 + 验收标准. 这个框架确保了 AI 功能的不确定性被完整覆盖 — 从 '什么进什么出' 到 '错了怎么办', 从 '花多少钱' 到 '数据谁能看'."

一句话规则 + 课后作业

一句话:

AI PRD = 传统 3 模块 +
IO + 指标 + 兜底 + Prompt + 成本 + 隐私.

课后作业 (60 分钟):
