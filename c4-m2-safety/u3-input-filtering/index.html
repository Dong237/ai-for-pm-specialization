想象你是一个门卫

你站在 LLM 的大门前. 所有用户输入都要先经过你, 你决定: 放行 or 拦截.

1
输入过滤 (Input Filter) ← 本节课
用户输入 → 检查 → 放行或拦截
2
LLM 推理
只有通过过滤的输入才到这里
3
输出审核 (Output Moderation) → U4
LLM 输出 → 再查一遍 → 放行或修改
4
用户看到的结果
只有两层都通过的才展示

输入过滤就是第一道门. 挡住了, LLM 连"看都看不到"这条恶意输入. 性价比最高.

但门卫怎么判断谁是坏人? ↓
(2)

两种判断方法: 关键词 vs 语义分类

A

方法 A: 关键词黑名单

维护一个"坏词表": "忽略指令", "system prompt", "DAN"... 输入里包含这些词就拦截.

优点: 速度快 (微秒级), 实现简单, 零成本
缺点: 容易绕过 (加空格/谐音/编码), 误杀率高 (正常讨论"什么是 prompt injection"也被拦)
B

方法 B: 分类模型 (ML Classifier)

用一个小模型 (如 Llama Guard, ShieldGemma) 判断输入意图: 安全 / 注入 / 越狱 / 敏感.

优点: 理解语义 (加空格也能识别), 覆盖变体, 准确率高
缺点: 有延迟 (50-200ms), 需要部署模型, 有成本
最佳实践

两个都用 — 关键词做第一层快速拦截, 分类模型做第二层语义兜底.

就像机场安检: 先过金属探测器 (快), 再过 X 光 (准).

关键词黑名单怎么设计? ↓
(3)

关键词黑名单: 必须有, 但别只靠它

Vitamin 的关键词黑名单应该包含:

注入类关键词:

忽略指令 · ignore previous · system prompt · 你的初始指令 · 输出你的prompt

越狱类关键词:

DAN · do anything now · 没有限制 · 无限制模式 · 你现在扮演 · act as

医疗越界关键词:

开处方 · 确诊 · 替我诊断 · 该吃什么药

关键词黑名单的致命缺陷:

"忽略指令" → 被拦. "忽 略 指 令" → 通过. "请不要遵循前面的规定" → 通过.

所以需要分类模型做兜底.

分类模型怎么工作? ↓
(4)

分类模型: 门卫的"直觉"

分类模型不看精确字符串 —— 它理解意图.

常用开源分类模型:
  • Llama Guard (Meta) — 专门为 LLM 安全设计的分类器, 支持 13 种风险类别
  • ShieldGemma (Google) — 轻量级安全分类器
  • Granite Guardian (IBM) — 企业级安全检测
  • NeMo Guardrails (NVIDIA) — 可编程安全护栏框架

这些模型能做到:

  • "忽 略 指 令" (加空格) → 识别为注入 ✓
  • "请不要遵循前面的规定" (换说法) → 识别为注入 ✓
  • "经期能跑步吗" → 识别为正常健康问题 ✓
但分类模型也不完美 — 灵敏度怎么调? ↓
(5)

灵敏度: 安全 vs 可用性的拉锯战

过滤器太松 → 攻击通过. 太严 → 正常用户被误杀. PM 的核心决策: 灵敏度定多高?

灵敏度 拦截率 误杀率 适用场景
低 ~60% ~1% 聊天娱乐类产品
中 ~85% ~5% 一般工具类产品
高 ~95% ~12% 医疗 / 金融 / 法律

Vitamin 的选择: 健康产品 → 选高灵敏度. 宁可误杀 (用户重新措辞就行), 也不能漏放危险内容.

误杀了怎么处理? ↓
(6)

误杀处理: 被拦了不能"冷冰冰"

用户正常问"经期能吃止痛药吗", 被误杀了. 怎么办?

✗

差体验: 硬拦截

"抱歉, 你的输入包含违规内容, 已被系统拦截."

问题: 用户懵了, 不知道哪里"违规", 体验极差, 流失.
✓

好体验: 友好重定向

"这个问题涉及用药建议, 我需要更谨慎地回答. 能帮你查一下常见缓解方式, 具体用药建议请咨询医生哦."

关键: 不说"违规", 说"我需要更谨慎". 给出替代方案 + 引导正确渠道.

PM 设计原则: 被拦截的回复要做到 3 点:

  1. 不羞辱用户 —— 不说"违规/恶意/不当"
  2. 给替代路径 —— "我可以帮你查..."
  3. 严重场景转人工 —— 自伤相关 → 直接弹出危机热线
输入过滤的技术架构是什么样? ↓
(7)

输入过滤的完整流程

用户消息从输入到 LLM, 要经过 4 步:

Step 1: 预处理 — 去除零宽字符 → 解码 Base64/URL encoding → 统一大小写

Step 2: 关键词扫描 — 匹配黑名单 → 命中则标记, 但不立即拦截

Step 3: 分类模型 — 语义分析意图 → 输出风险分数 (0-1)

Step 4: 决策引擎 — 综合关键词 + 分类模型结果 → 放行 / 重写 / 拦截

决策逻辑:

  • 风险 < 0.3 → ✅ 放行
  • 0.3 ≤ 风险 < 0.7 → ⚠️ 放行但在 System Prompt 中追加安全提醒
  • 风险 ≥ 0.7 → 🚫 拦截, 返回友好重定向
  • 自伤类 → 🆘 立即触发危机干预流程

Vitamin 的输入过滤该怎么设计? ↓
(9)

回到 Vitamin — 设计输入过滤规则

Vitamin 输入过滤配置建议:
类别 灵敏度 处理方式
注入攻击 (忽略指令/偷 prompt) 高 直接拦截 + 友好提示
越狱尝试 (DAN/角色切换) 高 拦截 + 重申身份
医疗越界 (开处方/诊断) 中 放行但追加安全提醒 + 转人工入口
自伤风险 最高 立即触发危机干预 + 热线号码
正常健康问题 低 放行

面试金句:

"输入过滤是 AI 安全的第一道防线, 但不是唯一防线. 关键词做快速拦截, 分类模型做语义兜底. 对于健康类产品, 我会选高灵敏度 — 宁可误杀让用户换个问法, 也不能漏放危险内容到 LLM."
