(1)

为什么输入过滤不够?

用户问了一个完全正常的问题: "怀孕初期注意什么?" 输入过滤: 通过. 但 LLM 的回复里出现了:

LLM 输出 (幻觉):
"怀孕初期可以服用阿司匹林来预防血栓, 建议每日 100mg..."

问题: 这不是用户攻击导致的 —— 是 LLM 自己"幻觉"出了一个错误且危险的医疗建议.

怀孕初期服用阿司匹林需要严格的医嘱, 不能随便推荐.

输入过滤完全无法拦截这种情况 — 因为输入是安全的, 危险在输出端.

所以需要第二道防线 ↓
(2)

核心原理

LLM 生成完回答后, 不直接给用户看 — 先过一遍审核模型.

审核模型检查: 有没有医疗错误? 有没有有害内容? 有没有隐私泄露?

通过 → 展示. 有问题 → 修改/拦截/追加免责声明.

但审核会增加延迟 ↓
(3)

代价: 延迟 — 慢但稳

每次回复多过一遍审核模型, 会增加 50-200ms 延迟.

审核方式 延迟 准确率 成本
规则匹配 (关键词) ~5ms 中 极低
小模型分类器 ~50ms 高 低
LLM 作为审核员 ~200ms 很高 中

PM 权衡: 对于 Vitamin 这种健康产品, 200ms 的延迟完全可以接受 — 用户不会察觉. 但安全性的提升是巨大的.

审核模型检查什么? ↓
(4)

输出审核检查的 5 个维度

  1. 医疗安全 — 是否包含具体药物剂量、处方建议、诊断结论
  2. 有害内容 — 是否包含暴力、歧视、自伤引导
  3. 隐私泄露 — 是否在回复中暴露了用户的私人数据
  4. 角色越界 — AI 是否声称自己是医生 / 专家
  5. 事实准确性 — 是否有明显的医学错误 (幻觉检测)

对于 Vitamin, 维度 1 (医疗安全) 和维度 4 (角色越界) 是最高优先级.

审核完了怎么处理? ↓
(5)

三种处理方式: 放行 / 修改 / 拦截

✅ 放行 (Pass)

审核通过, 原样输出. 可选择自动追加免责声明.

⚠️ 修改 (Rewrite)

检测到可改善内容 (如包含剂量但非危险). 自动删除具体剂量, 追加"请咨询医生". 用户看到的是修改后版本.

🚫 拦截 (Block)

检测到严重风险. 不展示 LLM 原始输出, 替换为安全回复 + 转人工入口.

三档严格度的效果差异 ↓
(6)

严格度: 宽松 / 标准 / 严格

和输入过滤一样, 输出审核也有灵敏度档位.

  • 宽松: 只拦截明确有害内容 (暴力/自伤). 适合日记/情绪类功能.
  • 标准: 拦截有害 + 标记医疗建议 + 追加免责声明. 默认推荐.
  • 严格: 任何涉及药物/诊断/治疗的内容都拦截或改写. 适合直接面向患者的功能.
免责声明怎么写? ↓
(7)

医疗免责声明: 必须有, 但别招人烦

✗

差: 冗长法律语言

"本内容仅供参考, 不构成医疗建议, 诊断或治疗. 用户应咨询合格医疗专业人员以获取个性化医疗建议. 本平台不对因依赖本内容而产生的任何损失承担责任."

✓

好: 简洁友好

"⚕️ 以上仅供参考, 不是医嘱. 身体有问题请看医生哦."

PM 原则:

  • 免责声明 ≤ 20 字 (用户才会看完)
  • 用日常语言, 不用法律术语
  • 不是每条都加 — 只在涉及健康建议时追加
  • 同时在用户协议里放完整版法律免责
来试试不同严格度的效果 ↓
(8)

动手试: Output Moderator

切换审核严格度, 看同样的 AI 回复在不同档位下的命运.

审核延迟: ~120ms 每次调用成本: ~$0.001

观察:

  1. 宽松模式 → 药物剂量建议直接通过 (对健康产品不安全)
  2. 标准模式 → 剂量建议被标记, 追加免责 (推荐)
  3. 严格模式 → 任何药物提及都被改写为"请咨询医生" (最安全)
Vitamin 该选哪档? ↓
