Vitamin 需要什么样的知识库?

Vitamin 是一个女性健康 AI 应用, 有 5 个身体状态追踪. 它的 RAG 知识库需要覆盖:

1

妇科医学

经期、排卵、孕期、更年期的医学知识
2

营养学

食物成分、营养建议、饮食禁忌
3

中医养生

体质辨识、食疗方案、穴位按摩
4

产品文档

功能说明、会员政策、FAQ
5

研究文献

最新论文、行业报告、WHO 指南
每类文档的特点和策略不一样 ↓
(2)

RAG 知识库的第一法则: 内容质量

铁律

垃圾进, 垃圾出.

RAG 只能搜到你知识库里有的东西. 知识库的内容过时、错误、冗余 → 再好的 Pipeline 也救不了.

"最关键的不是 AI, 而是你给它的知识库." — 行业共识

Vitamin 知识库的质量要求:

  • 所有医学内容必须经过医学顾问审核
  • 定期更新 — 不同文档有不同的更新频率
  • 去重 — 不能有多个版本的同一文档
  • 标注权威性 — 区分"WHO 指南" 和 "个人博客"
知识库的技术架构是什么? ↓
(3)

Vitamin RAG 技术架构

数据层 5 类文档 → 清洗 → 去重 → 质量审核 → 入库
处理层 Chunking (Recursive/语义/不切) → Embedding (bge-large-zh) → Vector DB
检索层 Hybrid Search → Reranking → Metadata 过滤 → Top-5 chunks
生成层 LLM + Citation Prompt → 生成回答 + 行内引用 → Post-hoc 验证
监控层 RAGAS 自动评测 → 人工抽查 → 知识库更新触发器
每类文档的具体策略 ↓
(4)

妇科医学文档: 精度优先

文档特点: 长篇 PDF, 多话题, 有章节结构, 更新周期长.

策略:

  • Chunking: 语义切分 (话题转折处切), 512 tokens, 15% overlap
  • Metadata: 章节标题 + 页码 + 来源 + 审核日期
  • Retrieval: Hybrid + Reranking, Top-K=50, N=5
  • 安全: 涉及用药/诊断的 chunk 标记为 "需要免责声明"
营养学文档策略 ↓
(5)

营养学 + 中医: 结构化优先

营养学文档

  • 食物表: 每种食物一个 chunk, 不切
  • 营养指南: Recursive 切, 400 tokens
  • 表格数据: 特殊处理, 保留表头作为 context
  • Metadata: 食物分类 + 营养素 + 适用人群

中医养生文档

  • 方剂: 每个方剂一个 chunk, 不切
  • 体质辨识: 每种体质一个 chunk
  • 食疗: 按段落切, 400 tokens
  • Metadata: 体质类型 + 症状 + 适应症
产品文档 + 研究文献 ↓
(6)

产品文档: 短文档不切, 研究文献: 按页切

用户手册
→
按功能章节切, 每个功能 1 个 chunk. 更新频率: 每版本.
FAQ
→
不切! 每个 Q&A 就是一个 chunk. 更新频率: 每月.
会员政策
→
不切. 文档短, 整篇 Embed. 更新频率: 每季度.
研究论文
→
按页切分. 保留论文标题 + 作者 + 年份作为 metadata.
WHO 指南
→
语义切分. 标记权威性等级: "国际标准".
知识库怎么更新? ↓
(7)

知识库更新策略

知识库不是"搭好就完了". 医学知识在更新, 产品在迭代, 知识库也要跟着更新.

Vitamin 更新策略:

  • 医学文档: 每季度审核一次, 有新指南立即更新
  • FAQ: 每月根据客服反馈新增 Q&A
  • 研究文献: 每季度添加最新论文
  • 产品文档: 每次版本更新时同步更新

更新流程:

  1. 新文档 → 医学顾问审核 → 清洗 → Chunking
  2. 新 chunks → Embedding → 追加到 Vector DB
  3. 删除/替换旧版本的 chunks
  4. 跑 RAGAS 验证更新后的检索质量没有下降

最后的整体蓝图 ↓
(9)

Vitamin RAG 完整蓝图

把 Module 3.3 学到的所有概念串在一起:

数据: 5 类 × 20+ 篇文档 × ~2000 chunks

Embedding: bge-large-zh-v1.5, 1024 维, 本地部署 (数据安全)

Chunking: 混合策略 — 医学用语义切, FAQ 不切, 其余 Recursive

Vector DB: Milvus (开源) 或阿里云 DashVector

Retrieval: Hybrid (BM25 + 向量) → Reranking → Top-5

Generation: 通义千问 / GPT-4o, T=0.2 (医学要稳)

Citation: 行内引用 [1][2] + 底部来源 + 免责声明

Evaluation: RAGAS 周报, Faithfulness >0.9 为及格线

更新: 分类更新, 最快每月 (FAQ), 最慢每年 (WHO)

这个方案能解决什么?

  • "经期能吃冰吗?" → 检索到经期饮食指南 + 中医养生, 准确回答 + 引用来源
  • "会员怎么退款?" → 检索到会员政策, 准确告知流程
  • "最新的叶酸研究怎么说?" → 检索到研究文献, 给出摘要

Module 3.3 一句话

RAG = 先搜后答.
知识库质量 > 检索策略 > LLM 选型.

垃圾知识库 + 最好的 LLM = 还是垃圾. 先把内容做好.
