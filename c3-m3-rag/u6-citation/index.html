引用机制 (Citation)

"这个结论哪来的?" — 让 RAG 回答可追溯

RAG 回答了用户的问题, 但用户不信...

"你说经期不能吃冰, 依据是什么?"

Perplexity 每句话都带引用 — 我们怎么做到?

—— Citation 机制.

10 步学完, 你会:
  • 理解为什么 Citation 是 RAG 产品的"信任基石"
  • 知道 3 种常见的引用实现方式
  • 亲手体验一个可点击的引用界面
  • 为 Vitamin 设计引用展示方案
走起 →
(1)

没有引用的 RAG = 高级版"编"

RAG 减少了幻觉, 但用户怎么知道它没有编? 答案: 标注来源.

没引用时: "经期应避免寒凉食物." — 用户: "你编的吧?"

有引用时: "经期应避免寒凉食物 [1]. 中医认为寒凉伤脾 [2]." — 用户: "嗯, 有出处, 可信."

尤其是医学健康领域 — 没有来源的建议, 用户凭什么信你?

Citation 的好处不只是"让用户信" ↓
(2)

Citation 的四大价值

可验证
→
用户能点进原始文档, 自己确认信息准确性
建信任
→
标注来源 = 告诉用户"这不是我编的, 有据可查"
可审计
→
出了错能追溯: 是哪篇文档的信息有误? 还是 LLM 理解错了?
减幻觉
→
要求 LLM "每句话都标来源" 时, 它更倾向于忠实引用, 减少编造
Citation 有几种实现方式? ↓
(3)

三种引用实现方式

方式 1: 行内引用

在回答文本中直接插入 [1] [2] 标记, 每个标记对应一个来源. Perplexity 用的就是这种.

方式 2: 底部来源列表

回答下面附一个 "来源" 列表, 列出所有参考文档. 简单, 但用户不知道哪句话来自哪个文档.

方式 3: 可展开卡片

关键句子旁边有一个"查看来源"按钮, 点击展开显示原始文档片段. 体验最好, 开发成本最高.

PM 建议: 先做方式 1 (行内引用 + 底部列表), 快速上线. 后续迭代升级到方式 3 (可展开卡片).

行内引用怎么实现? ↓
(4)

Prompt Engineering 让 LLM 标引用

Citation 的实现分两部分: Prompt 设计 + 前端展示.

System Prompt (示例):
你是 Vitamin 的健康顾问. 根据以下资料回答用户问题. 规则: 1. 只根据给定资料回答, 不要编造 2. 每个事实性陈述后面标注来源 [1] [2] 等 3. 如果资料中没有相关信息, 说"我没有找到相关信息" 4. 在回答末尾列出所有引用的来源 资料: [1] 经期饮食指南.pdf, p.12: "经期应避免..." [2] 中医养生.docx, ch.3: "中医认为..." [3] 营养学手册.pdf, p.45: "铁质补充..."

关键技巧:

  • 每个检索到的 chunk 都要有编号 [1] [2] [3]
  • 明确要求 LLM "每个事实后标注来源编号"
  • 明确要求 "没有信息就说不知道" (防止编造)
  • 来源信息包含: 文档名 + 页码/章节 (方便用户定位)
LLM 的引用准确率其实不太高 ↓
(5)

引用准确率的挑战: 只有 74%

行业研究显示, LLM 生成的引用准确率大约只有 74%. 也就是说, 4 个引用里有 1 个可能是错的.

常见错误:

  • 张冠李戴: 把 [1] 的内容标成了 [2] 的来源
  • 过度综合: LLM 把多个来源的内容混合改写, 标了一个来源但实际用了多个
  • 虚假引用: LLM 编了一个听起来合理的说法, 然后随便标了一个来源

改进方法:

  • Post-processing: 生成后用算法检查每个引用是否真的匹配来源内容
  • 降低 Temperature: T=0.1~0.3, 让 LLM 更"忠实"
  • Few-shot 示例: 在 prompt 里给 2~3 个正确引用的示例
另一个思路: 不让 LLM 自己标引用 ↓
(6)

Post-hoc Attribution: 后处理标引用

不信任 LLM 标的引用? 可以在 LLM 输出后, 用另一个算法检查和修正.

LLM 生成回答 → 拆成逐句 claims → 每句 vs 每个 chunk 算相似度 → 自动标注最匹配的来源

这种方式的优点:

  • 引用准确率可以提高到 90%+
  • 不依赖 LLM 的"自觉性"
  • 可以发现 LLM 回答中没有来源支持的句子 → 标为 "未验证"

缺点: 需要额外的计算步骤, 增加延迟.

Grounding = 确保回答忠实于来源 ↓
(7)

Grounding: 回答必须"脚踏实地"

Grounding 是 Citation 的哲学基础: 每个 claim 都必须有据可查, 没有依据就不说.

Grounding 的核心规则:

  1. Claim → Source 映射: 回答中的每个事实陈述都必须对应至少一个检索到的 chunk
  2. 无证据就弃权: 如果 claim 没有任何 chunk 支持, 系统应该说"我没有找到相关信息"
  3. 弃权是特性, 不是 bug: 宁可不回答, 也不编

尤其是医学领域: 一个没有来源的健康建议, 可能比不回答更危险.

来体验一个有引用的回答界面 ↓
(8)

动手: Citation Viewer

下面是一个 RAG 回答. 点击 [1] 等引用标记, 看它对应的来源文档.

问: 经期可以吃冰的吗?

经期建议避免食用过于寒凉的食物,如冰淇淋、冰镇饮料等 [1]. 中医认为经期食用寒凉食物可能导致宫寒,影响血液循环,加重痛经症状 [2]. 但偶尔少量常温饮品是可以的,建议根据个人体质判断 [1].

作为替代,推荐饮用生姜红糖水等温热饮品,有助于暖宫活血 [3]. 同时经期应注意补充铁质,可以多吃菠菜、红瘦肉等含铁食物 [1].
[1] 经期饮食指南.pdf · 第12页
"经期应避免食用过于寒凉的食物,如冰淇淋、冰镇饮料等,因为寒凉食物可能会加重痛经症状...要注意补充铁质,经期失血较多...偶尔少量常温饮品是可以的"
[2] 中医养生.docx · 第3章
"中医认为,经期食用寒凉食物会导致宫寒,影响血液循环,气血运行不畅则加重痛经..."
[3] 经期饮食指南.pdf · 第15页
"建议多吃温热性食物,红枣、姜茶、桂圆等有助于暖宫活血,特别是生姜红糖水,是经期首选饮品"

体验要点:

  • 点击 [1] — 左边的来源卡片高亮, 你能看到原始文档内容
  • 注意: 同一个来源 [1] 在回答中出现了 3 次 — 说明这个 chunk 提供了最多信息
  • 每句话都有据可查 — 这就是 Citation 的用户价值
回到 Vitamin ↓
(9)

Vitamin 的 Citation 设计

Vitamin 是健康应用, Citation 不只是"好看的功能", 而是合规和信任的底线.

Vitamin Citation 方案:

  1. 行内标注: 每个健康建议后面标 [1] [2], 可点击查看来源
  2. 来源卡片: 底部显示来源文档名 + 章节 + 关键引文
  3. 信心标记: 高信心 (多个来源支持) 用绿色, 低信心 (单一来源) 用黄色
  4. 免责声明: "以上信息仅供参考, 不构成医疗建议. 详情请咨询医生."
  5. 后处理验证: 每个引用经过 post-hoc attribution 检查, 确保准确

合规红线: 涉及用药建议、疾病诊断的回答, 必须标注来源 + 加免责声明. 没有来源支持的医学 claim, 一律不输出.
