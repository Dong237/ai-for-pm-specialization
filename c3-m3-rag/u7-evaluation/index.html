走起 →
(1)

"回答不好" 到底是哪里不好?

RAG 出错时, 可能是 3 个环节中任何一个:

检索错
→
搜到了不相关的文档. 后面再好也没用.
检索漏
→
遗漏了关键文档. 回答虽然对, 但不完整.
生成错
→
检索到了对的文档, 但 LLM 理解错了或者编了.

RAGAS (Retrieval Augmented Generation Assessment) 就是把这三类问题量化的框架. 它给你 4 个分数, 告诉你问题出在哪.

4 个核心指标是什么? ↓
(2)

RAGAS 四大指标

Faithfulness
→
忠实度: 回答中的每句话, 是否有来源支持? 没来源的 = 幻觉. 目标: >0.9
Relevancy
→
相关度: 回答是否直接回答了用户的问题? 目标: >0.85
Context Precision
→
检索精度: 检索到的文档是否都是相关的? 不相关的越多, 精度越低. 目标: >0.8
Context Recall
→
检索召回: 是否找到了所有需要的文档? 遗漏越多, 召回越低. 目标: >0.8
这 4 个指标怎么配合使用? ↓
(3)

2x2 矩阵: 快速诊断 RAG 问题

✅
高 Faithfulness + 高 Relevancy
理想状态. 回答准确且相关.
⚠️
高 Faithfulness + 低 Relevancy
回答忠实但没用. 检索搜偏了.
⚠️
低 Faithfulness + 高 Relevancy
回答看似相关但不可信. LLM 在编.
❌
低 Faithfulness + 低 Relevancy
最差: 既不忠实又不相关. 全面返工.

PM 用法: 定期跑 RAGAS, 看分数落在哪个象限. 右上角 = 继续优化. 左下角 = 重大问题, 优先排查检索.

每个指标具体怎么算? ↓
(4)

Faithfulness 怎么算: 逐句验证

Faithfulness = 有来源支持的 claims 数 / 总 claims 数

计算步骤:

  1. 把 LLM 回答拆成多个 claims (事实陈述)
  2. 对每个 claim, 检查是否有检索到的 chunk 支持它
  3. Faithfulness = 被支持的 claims / 总 claims

例子:

"经期应避免寒凉食物 [来源支持 ✓]. 否则会导致不孕不育 [无来源 ✗]."

Faithfulness = 1/2 = 0.50 — 很低, 有幻觉!

Relevancy 怎么算? ↓
(5)

Relevancy 和 Context Metrics

Answer Relevancy:

  • 从回答中生成 N 个"假问题"
  • 计算这些假问题和原始问题的相似度
  • 如果回答真的在回答原始问题, 假问题应该和原始问题很像

Context Precision:

  • 检索到的 5 个 chunks, 有几个是真正相关的?
  • 5 个都相关 = 1.0, 只有 2 个相关 = 0.4

Context Recall:

  • 理想回答中需要的信息, 在检索到的 chunks 中找到了多少?
  • 全部找到 = 1.0, 只找到一半 = 0.5
RAGAS 不需要人工标注吗? ↓
(6)

Reference-Free: 不需要人工标注

RAGAS 的厉害之处: 它不需要"标准答案". 它用 LLM 来评估 LLM.

传统评测: 需要人工准备 (问题, 标准答案) 对 → 成本高, 难扩展

RAGAS: 只需要 (问题, 检索上下文, 生成回答) → 用另一个 LLM 打分

注意: 因为用 LLM 打分, 结果可能有波动. 建议跑多次取平均, 或结合少量人工抽查.

实践建议:

  • 自动化: 用 RAGAS 跑 200 个测试 query, 全自动打分
  • 人工补充: 随机抽 20 个结果, 人工检查 RAGAS 评分是否合理
  • 持续监控: 每天/每周跑一次, 看分数趋势
分数低了怎么改进? ↓
(7)

分数低 → 对症下药

Faithfulness 低
→
LLM 在编. 降低 Temperature, 加强 "只根据资料回答" 的 prompt, 加 post-hoc 验证.
Relevancy 低
→
答非所问. 检查检索结果, 可能搜偏了. 加 Reranking, 改 query expansion.
Precision 低
→
检索到太多不相关 chunk. 减小 Top-K, 加 Reranking, 优化 chunk size.
Recall 低
→
遗漏了关键文档. 增大 Top-K, 加 Hybrid Search, 用 Query Expansion.
来动手给几种 RAG 回答打分 ↓
(8)

动手: RAG Evaluator

切换不同的场景, 看 RAGAS 4 个指标的变化. 注意分数怎么"诊断"问题.

观察:

  • "幻觉回答": Faithfulness 极低 (25), 但 Relevancy 还行 — 说明 LLM 在"相关地编"
  • "答非所问": Faithfulness 高 (90), 但 Relevancy 极低 — 说明检索搜偏了
  • "不完整回答": Recall 极低 (30) — 说明遗漏了很多相关文档
  • 每种问题的"指纹"不同 — RAGAS 帮你精确定位
回到 Vitamin ↓
(9)

Vitamin 的 RAG 评测方案

Vitamin 评测目标:

  • Faithfulness > 0.90 (医学内容, 零容忍幻觉)
  • Answer Relevancy > 0.85
  • Context Precision > 0.80
  • Context Recall > 0.75

评测流程:

  1. 准备 200 个测试 query (覆盖经期/备孕/营养/中医/客服)
  2. 每周自动跑 RAGAS, 输出分数报告
  3. 每月人工抽查 30 个, 校准 RAGAS 的评分准确性
  4. 分数低于阈值 → 自动告警, 分类排查
