(1)

上节课的一个善意谎言

Unit 1 我们一直说 "猜下一个字". 这个说法帮你理解了核心原理, 但它是个简化.

真相是: LLM 看到的不是 "字", 也不是 "词". 它看到的是一种叫 Token 的东西.

试试这个 —— 把 "今天天气真好" 喂给 GPT, 它看到的是:

GPT-4 看到的:

6 个字, 但只有 4 个 token. 因为 "今天" 和 "天气" 各被当成一整块.

这就是今天的主题: Token 到底是什么, 为什么它比 "字" 更重要.

先看看你可能有的两个猜测 — 都不对 ↓
(2)

两个常见猜测 — 又都不对

✗

猜测 A: Token = 一个汉字

一个字一个 token, 简单吧?

为啥不对: "今天" 是 2 个字, 但在 GPT-4 里只算 1 个 token. "你好" 也是 1 个 token. 常见组合会被 "粘" 在一起.
✗

猜测 B: Token = 一个英文单词

英文按空格分词, 一个词一个 token?

为啥不对: "unhappiness" 不是 1 个 token — 它被拆成 "un" + "happiness" 两块. 长词、罕见词会被拆开.
真相

Token 是 Tokenizer 按算法切出来的最小单位.

不是字, 不是词, 是 "拼图块".

每个模型有自己的 tokenizer, 切法不一样. 同一句话, GPT 切成 4 块, DeepSeek 可能切成 5 块.

→ 就像乐高: 有的套件把 "窗户" 做成一个大块, 有的拆成 4 个小零件. 最终都能拼出房子, 但用的块数不一样.

那 tokenizer 到底怎么决定哪里切、哪里不切? ↓
(3)

看一句话是怎么被切的

拿两句同样意思的话, 看 GPT-4 tokenizer 怎么切:

🇨🇳 中文: "今天天气真好"
🇺🇸 英文: "The weather is great today"

注意看 ——

  1. 中文: 6 个字 → 4 个 token. 常见组合 ("今天", "天气") 被粘成了一块
  2. 英文: 26 个字符 → 5 个 token. 基本上一个词一个 token
  3. 字符数和 token 数不一样. 这是关键.

→ 既然不是按字切的, 那到底按什么规则切? 答案是一个叫 BPE 的算法.

BPE 听着吓人, 其实就是 "把常用的拼在一起" ↓
(4)

BPE — 把最常见的拼在一起

Byte Pair Encoding (BPE) 是大多数 LLM 用的 tokenizer 算法. 原理超简单:

0 初始: 每个字符单独一块
lower
1 → "e" 和 "r" 最常相邻, 合并!
lower
2 → "l" 和 "o" 常相邻, 合并!
lower
3 → "low" 常出现, 合并!
lower
✓ → 最终: "lower" = 2 个 token
lower
类比: 想象你在玩乐高 ——

刚开始全是最小的 1×1 颗粒. 你发现 "这两颗总是一起用", 就把它们粘成一块 2×1. 再发现 "这块 2×1 总跟另一颗一起用", 粘成 3×1.

最终, 常用组合变成大积木, 罕见组合还是小颗粒. GPT-4 的 "乐高盒" 里有 ~100,000 块不同的积木.

100,000 块积木 — 这就是模型的 "词表" ↓
(5)

词表 = AI 认识的全部 "拼图块"

BPE 跑完之后, 产出一个固定大小的词表 (vocabulary). 每个 token 在词表里都有一个编号.

GPT-4 的词表大小: ~100,256 个 token.

这就是它认识的全部 "积木". 遇到任何文字, 都要用这 100K 块积木来拼.

词表里的 token 长短不一:

  • 超常见词 → 整个词 = 1 token: the, 今天, hello, Vitamin
  • 普通词 → 拆成 2-3 块: un + happiness, 经 + 期
  • 罕见词 / 错别字 → 拆成很多小块: 每个字符甚至每个字节一块

类比: 就像你认字 ——

"你好" — 一眼认出, 不用拆.
"尴尬" — 还行, 两个字分开也认识.
"耄耋" — 这啥? 要拆偏旁: "耳" + "毛" + "至" + "老"...

AI 也一样: 常见的一口吞, 罕见的逐字节啃.

那同样意思的东西, 中英文 token 数差多少? ↓
(6)

同一个意思, 不同的 "价格"

来看一个让 PM 直冒冷汗的对比:

英文
Vitamin
Token 数:
1
常见英文词, 一口吞
中文
维他命
Token 数:
3
每个字单独切, 3 倍!

再看一组:

英文
artificial intelligence
Token 数:
2
中文
人工智能
Token 数:
2
这次一样! 因为这个词够常见
PM 认知:

规律: 越常见的词, 被压缩得越小 (token 数越少). 越罕见的词, token 数越多.

API 按 token 计费. 同样意思的 prompt, 中文通常比英文多 1.5-2 倍 token — 因为大多数 tokenizer 是英文数据训出来的.

→ 下节课 (U3) 会深挖中英文差异. 今天先记住: token 数 ≠ 字数.

还有个有趣的: 英文长词怎么拆? ↓
(7)

Subword — AI 认 "偏旁部首" 的方式

BPE 不只是在压缩 — 它还教会了模型一个技能: 理解词的组成部分.

unhappiness
↓ tokenizer 切成 ↓
un + happiness
前缀 "un" = 否定 · 词根 "happiness" = 快乐
模型从 "un-" 学到了 "取反" 这个概念

这意味着:

  • "unhappiness" 可能模型没见过这整个词
  • 但它见过 "un-" (取反) 和 "happiness" (快乐)
  • 所以它能推出 "不快乐" 的意思 — 靠的是拼积木, 不是死记
中文也一样:

"机器学习" → "机器" + "学习" (2 tokens). 模型分别认识这两个词, 拼在一起就懂了.

"强化学习" → "强化" + "学习" (2 tokens). 换个前缀, 意思变了, 但 "学习" 的含义复用了.

→ Subword 让模型用有限的 100K 词表, 理解几乎无限的词组合. 聪明.

(9)

回到 Vitamin — Token 数 = 你的钞票

API 按 token 计费. Token 数就是你的成本单位.

来算一笔 Vitamin 的账:

日活用户 (DAU) 1,000 人
每人每天对话次数 5 次
每次对话 token 数 (输入+输出) ~500 tokens
每日总 token 2,500,000
模型单价 (DeepSeek V3) ¥2 / 百万 token
每月成本 ≈
¥150 / 月
用 DeepSeek V3. 如果用 GPT-4o, 大约 ¥3,000/月 (贵 20 倍)
PM 认知 — Token 数直接影响 3 件事:
  1. 成本: 每个 token 都是钱. 少说废话 = 省 token = 省钱
  2. 速度: Token 越多, 生成越慢. 用户等得久 = 体验差
  3. 中文税: 同样意思, 中文多 ~1.5x token. 中文产品天然比英文产品贵
Vitamin 优化思路:
  • System prompt 精简: 删掉冗余指令, 省 token
  • 用户输入限长: 限制每次对话字数, 控制 token 上限
  • 选对模型: 中文友好的模型 (DeepSeek / Qwen) token 效率更高
  • 缓存常用回复: 相同问题不重复调 API
最后, 一句话 + 作业 ↓
(10)

一句话规则 + 今晚作业

一句话:

Token 是模型的 "最小拼图块".
1 token ≈ ¾ 个英文单词 ≈ ½ 个中文汉字.
Token 数 = 你的成本单位.

看 AI 产品成本时, 永远先问: 一次对话多少 token?

课后作业 (今晚 30 分钟):
  1. 打开 tiktokenizer.vercel.app, 输入 3 段 Vitamin 可能用到的 prompt, 记下每段的 token 数.
  2. 在同一个工具里, 分别输入 "你好吗" 和 "How are you" — 哪个 token 多? 多多少?
  3. 用第 9 步的公式, 估算 Vitamin 如果用 GPT-4o 替代 DeepSeek V3, 每月成本会变成多少.
  4. 写 100 字: 用 "拼图块" 比喻, 向一个非技术朋友解释 token.

→ 第 1 项: 如果总 token 超过 500, 想想怎么精简 prompt 省钱.
