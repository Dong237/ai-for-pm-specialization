# U4 · Context Window — Storyboard

## Hook (START)
"你跟 AI 聊了 20 条, 它突然忘了你第 3 条说的事. 为什么?"

## (1) AI 的 "失忆" 现象
演示: 对话到第 10 轮, AI 忘了你开头说的名字

## (2) 错猜测: AI 有长期记忆? 
真相: 没有. 每次对话, AI 只看一个固定大小的 "窗口"

## (3) Context = System Prompt + 对话历史 + 你的输入
画一个框, 里面装着三块内容, 总大小有上限

## (4) 窗口大小对比
GPT-3.5 (4K) → GPT-4 (128K) → Claude (200K) → Gemini (1M)
128K ≈ 一本 200 页的书

## (5) 大 ≠ 好: "Lost in the Middle"
Google 论文: 模型对开头和结尾的信息最敏感, 中间的容易 "遗忘"

## (6) 满了之后怎么办
3 种策略: 截断最早的 / 报错 / 压缩摘要

## (7) Token 价格 × Context 长度 = 成本
Context 越长, 每次调用越贵. 128K 的调用比 4K 贵 32 倍

## (8) 互动: Memory Bucket
模拟上下文窗口: 点击添加消息, 看窗口填满, 旧消息被挤出

## (9) Vitamin 案例
5 状态追踪需要记住用户历史. 但 context 有限. 方案: 摘要压缩 / 只保留最近 N 轮

## (10) 规则 + 作业
"Context Window = AI 的短期记忆. 满了就失忆. 管好这个窗口是 PM 的活."
