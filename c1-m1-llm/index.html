LLM 工作机制
+ 8 个核心概念

从 "LLM 在猜下一个字" 到 Token、Temperature、Hallucination... 9 个单元, 建立你作为 AI PM 的核心认知.

9 单元
~90 分钟阅读
9 互动小工具
1

LLM 是什么: Next-Token Prediction

LLM 不是 "理解了语言", 是 "在算下一个最可能的词". 这一句解释为什么 LLM 像人但又不像人.

→
2

Token 是什么

不是字、不是词, 是模型词表里的最小单位. 一句话的 token 数 ≠ 字数 — 而 token 数 = 你的成本.

→
3

Tokenizer 的中英文差异

同样意思的句子中文 token 数通常是英文的 1.5 倍. 这直接影响中文 AI 产品的成本和模型选型.

→
4

Context Window

模型的 "短期记忆" 上限. 满了之后会失忆. Vitamin 多轮对话的天花板就在这里.

→
5

Temperature 参数

控制 "创意 vs 严谨". T=0 永远同一答案, T=1 每次不同. Vitamin 健康建议为什么必须设低.

→
6

Top-P / Top-K

Temperature 的 "安全网". Top-P 自适应砍长尾, Top-K 固定名额. 健康产品的双保险.

→
7

幻觉 (Hallucination)

不是 bug 是 feature — 概率分布的必然产物. 知道什么时候最容易幻觉, 怎么从工程角度防住.

→
8

知识截止 (Knowledge Cutoff)

模型为什么不知道 "今天的事". RAG 和 Web Search 是怎么补这个洞的.

→
9

多模态 (Multimodal)

文本 → 图像 → 语音 → 视频的能力扩展. Vitamin 如果加图片识别意味着什么.

→
