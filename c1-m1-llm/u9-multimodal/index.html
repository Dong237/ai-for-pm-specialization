到目前为止, 全是文字游戏

回顾前 8 课: Token 是文字碎片, Context Window 是文字容量, Temperature 控制文字概率分布...

前 8 课的模式:

输入: 文字 → 模型处理 → 输出: 文字

无论是翻译、写代码、做摘要, 永远是 text in → text out.

但真实世界不只有文字:

  • 用户拍了一张照片问 "这是什么食物?"
  • 用户用语音说 "今天身体不舒服"
  • 用户想要一段视频教她做瑜伽

如果 AI 只能处理文字, 这些需求全做不了.

GPT-4Vtext + image → text

GPT-4otext + image + audio → all
所以 AI 进化了 — 进入 "多模态" 时代 ↓
(2)

什么是 "多模态" (Multimodal)

"模态" = 信息的类型/形式. 多模态 = 能处理多种类型.

核心定义

Multimodal AI = 能接受多种输入 (文本、图片、语音、视频) 并产生多种输出的 AI 系统.

"Modal" 来自拉丁语 modus (方式). Multi = 多种方式.

四种主要模态:
模态输入方向输出方向
文本理解文字生成文字
图像理解图片生成图片
语音听懂语音合成语音
视频理解视频生成视频

关键区分: 每种模态都有 "理解" (输入) 和 "生成" (输出) 两个方向. 一个模型不一定两个方向都会.

先看最成熟的非文本模态: 图像理解 ↓
(3)

图像理解: AI 能 "看" 了

2023 年, GPT-4V 和 Claude Vision 让 LLM 学会了看图说话.

图像理解能做什么:
  • 图片描述: 发一张照片 → AI 告诉你图里有什么
  • OCR 识别: 拍文档/菜单/处方 → 提取文字
  • 图表分析: 截图一张图表 → AI 解读数据趋势
  • 物体识别: 拍一盘菜 → AI 说出菜名和大概热量
Vitamin 应用场景:

用户拍一张午餐照片 → AI 识别: "鸡胸肉沙拉, 约 350 kcal, 蛋白质 32g"

用户拍经期日历截图 → AI 自动录入到 5-state body tracking

主要玩家: GPT-4V (OpenAI), Claude 3 Vision (Anthropic), Gemini Pro Vision (Google)

PM 注意: 图像理解 ≠ 100% 准确. 医疗图片识别有法规风险, 不能直接做诊断依据.
AI 除了看图, 还能画图 ↓
(4)

图像生成: 文字 → 图片

DALL-E, Midjourney, Stable Diffusion — 用文字描述, AI 画给你.

图像生成能做什么:
  • 文生图: "一个女孩在海边做瑜伽, 日落" → 生成图片
  • 图片编辑: 上传照片 + "把背景换成森林" → 修改
  • 风格转换: "把这张照片变成水彩画风格"
  • 设计辅助: 产品概念图、UI 原型、营销素材
!

常见误解

"AI 画的图可以直接商用"

版权问题未完全解决. Midjourney 有商用条款, DALL-E 3 允许商用, 但训练数据来源仍有争议.
!

常见误解

"AI 能精确控制细节"

文字描述越精确, 图越接近. 但 "左手拿苹果、右手拿书" 这种精确空间指令, AI 还经常搞错.
Vitamin: 生成个性化的健康食谱图, 给用户做膳食计划时配插图, 比纯文字更有吸引力.
接下来: 声音 ↓
(5)

语音: 听和说

AI 现在能听懂你说话, 也能用自然语音回答你.

三大语音能力:
能力方向代表产品用途
STT语音 → 文字Whisper, 讯飞语音输入、会议转录
TTS文字 → 语音OpenAI TTS, ElevenLabs朗读、语音助手
Voice Clone样本 → 模仿ElevenLabs, VALL-E个性化语音、有声书
Vitamin 场景: 语音日记

睡前, 用户对 Vitamin 说: "今天感觉腰酸, 晚上吃了火锅, 心情还行."

→ Whisper 转文字 → AI 提取: {身体: 腰酸, 饮食: 火锅, 情绪: 一般}

→ 自动更新 5-state body tracking. 用户什么都不用打字.

PM 风险点:
  • Voice clone 的伦理风险 — 模仿他人声音可能被滥用
  • STT 对方言/口音的准确率仍然不稳定
  • 实时语音对延迟要求极高 (< 300ms 才自然)
最后一种模态: 视频 ↓
(6)

视频: 理解 + 生成

视频 = 图像 + 音频 + 时间. 它是最复杂的模态.

视频理解:
  • Gemini 1.5 Pro: 上传 1 小时视频, AI 能回答关于视频内容的问题
  • GPT-4o: 实时分析摄像头画面, 描述看到的东西
  • 应用: 安防监控、视频搜索、内容审核
视频生成:
  • Sora (OpenAI): 文字描述 → 生成 60 秒高质量视频
  • Runway Gen-3: 文字/图片 → 短视频
  • Pika, Kling: 各有特色的视频生成工具

2025 年视频生成仍有限制: 长度短、一致性不稳定、人物动作有时不自然.

Vitamin 场景:

AI 根据用户的身体状态, 生成个性化的 30 秒拉伸教学视频: "你说腰酸? 试试这 3 个动作."

目前更现实的方案: AI 从预制视频库中推荐, 而非实时生成.

单独的模态都会了 — 但最厉害的是融合 ↓
(7)

融合: 一个模型搞定所有模态

最大的趋势不是 "每种模态一个工具" — 而是一个模型原生支持多种模态.

关键趋势

从 "拼装" 到 "原生融合":

早期: STT → LLM → TTS (三步拼装)
现在: GPT-4o 直接 audio in → audio out

原生融合 = 延迟更低, 理解更深, 能力更强.

文本
+
图像
+
语音

代表模型:
模型支持的模态特点
GPT-4o文本 + 图像 + 语音"o" = omni (全能). 原生语音, 延迟 300ms
Gemini 1.5文本 + 图像 + 音频 + 视频支持 1 小时视频理解, 100 万 token context
Claude 3.5文本 + 图像图像理解强, 但不生成图像 (专注准确性)
PM 思考: 选融合模型还是拼装?
  • 融合: 体验好 (低延迟, 跨模态理解), 但贵, 且受限于一家供应商
  • 拼装: 灵活, 可以每个模态选最好的, 但集成复杂, 延迟高
  • Vitamin 建议: 核心对话用融合模型, 图像/视频用专门工具

(9)

回到 Vitamin — 多模态功能矩阵

Vitamin 不需要一次上所有模态. 按优先级分阶段:

Vitamin 多模态路线图:
阶段模态功能模型选择
Phase 1文本健康问答 + 状态分析GPT-4o / Claude
Phase 2图像拍照识食物 → 营养分析GPT-4o Vision
Phase 3语音语音日记 → 自动记录身体状态Whisper + GPT-4o
Phase 4视频个性化运动教学预制库 + AI 推荐

多模态能做的

  • 拍照记录饮食, 不用手动输入
  • 语音描述身体状态, 比打字快 3 倍
  • AI 看用户上传的化验单, 提取关键指标
  • 生成图文并茂的健康报告

多模态的限制

  • 医疗图像识别 ≠ 诊断 (法规限制)
  • 语音识别对方言/口音仍不够准
  • 视频生成成本极高, 不适合 MVP
  • 多模态 API 调用费用是纯文本的 5-20 倍
面试金句, 抄走:

"多模态不是 feature list, 是 roadmap. 我会按 ROI 排序: 文本先上 (成本低、能力成熟), 图像第二 (拍照识食物解决真实痛点), 语音第三 (降低输入门槛), 视频最后 (技术最贵但刚需最弱). 每个阶段都要评估: 这个模态解决的用户问题, 值不值它的 API 成本."

一句话 + 作业 ↓
(10)

一句话规则 + 今晚作业

一句话:

多模态 = AI 的感官扩展.
文本是大脑, 图像是眼睛, 语音是嘴和耳朵, 视频是全部加在一起.

PM 不需要全部上. 找到你的用户最需要哪种 "感官", 先做那个.

课后作业 (今晚 30 分钟):
  1. 打开 ChatGPT, 上传一张食物照片, 问 "这是什么食物? 大概多少卡路里?" — 记录 AI 的回答准确度.
  2. 给 Vitamin 写一个 "拍照识食物" 功能的 PRD 大纲: 用户流程、调用哪个 API、成本估算 (每次调用约 $0.01-0.03).
  3. 对比 GPT-4o 和 Gemini 1.5 Pro 的多模态能力: 列一个表格, 比较支持的模态、上下文长度、价格.
  4. 写 100 字: 如果 Vitamin 只能加一种非文本模态, 你选哪个? 为什么?
