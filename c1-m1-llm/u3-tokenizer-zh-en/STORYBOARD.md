# U3 · Tokenizer 的中英文差异 — Storyboard

## Hook (START)
"上节课你发现 'Vitamin' = 1 token, '维他命' = 3 tokens. 今天来回答: 为什么中文天然更贵?"
4 bullets: 中英文 token 差异的根因 / UTF-8 字节数 / 各模型对比 / Vitamin 该选哪个

## (1) 账单惊吓: 同一个 prompt, 中文版贵了 70%
一段 Vitamin 健康建议, 英文 ~120 tokens, 中文 ~200 tokens. 同样的意思, 多花 70%.

## (2) 为什么? 两个错猜测
✗ 中文字符多所以贵? 不完全对, 中文很紧凑
✗ 中文语言本身更复杂? 不是语言问题, 是 tokenizer 的问题
真相: GPT 的 tokenizer 是英文数据为主训出来的, 中文压缩率低

## (3) 根因: UTF-8 字节与 BPE 压缩
英文: 1 char = 1 byte, BPE 容易合并
中文: 1 char = 3 bytes, BPE 要先处理 3 倍的字节量
训练语料英文占 90% → 英文组合被充分合并, 中文组合不够

## (4) 对比实验: 5 组等义句子
5 组预设句子, 中英对照, 显示 token 数对比柱状图

## (5) 不同 tokenizer 的效率差异
GPT-4 (cl100k_base): 英文优化, 中文效率低
DeepSeek V3: 中文训练数据更多, 中文效率提高 ~30%
Qwen (通义): 阿里专为中文优化, 中文效率最高

## (6) 这对产品意味着什么
同一个功能, 用 GPT-4 做中文版 vs 用 Qwen 做中文版, 成本差 30-50%
选对 tokenizer ≈ 省钱

## (7) 一个反直觉: 中文并不总是更贵
短文本: 中文和英文差不多 (因为词表里有常见中文词)
长文本: 差距拉大 (罕见组合被逐字切)
代码/数字: 差距最小 (都是 ASCII)

## (8) 互动: 并排对比器
选 5 组预设句子, 看中英文 token 数对比 + 成本倍数

## (9) Vitamin 决策
选模型时考虑 tokenizer 效率: DeepSeek / Qwen 对中文产品更友好
估算: 从 GPT-4o 切到 DeepSeek V3, token 省 30%, 单价再省 10x

## (10) 规则 + 作业
规则: "估算中文 AI 产品成本, 在英文基准上乘 1.5. 选中文友好的 tokenizer 可以缩小差距."
