Top-P / Top-K

Temperature 的 "安全网" — 决定谁有资格被选.

Temperature 我懂了: 调分布形状.

但 API 里还有 Top-P 和 Top-K... 那是啥?

走起 →
(1)

Temperature 有个问题

上节课学了 Temperature. 但只调 Temperature 有一个风险:

即使 T=0.7 (正常值), 那个 0.001% 概率的 "饿" 也有可能被选中.

"今天天气真饿" — 虽然概率极低, 但只要不是 0, 就有机会.

问 1000 次, 大约有 1 次会出现这种 "离谱" 输出.

如果是写故事, 这种意外也许是惊喜. 但如果是 Vitamin 给健康建议:

"经期期间应该多喝紫薯汤" — 不行, 我们需要一道安全线.

这道安全线就是 Top-K 和 Top-P.

先看简单的那个: Top-K ↓
(2)

Top-K: 只保留前 K 名

Top-K 极其简单 — 按概率排序, 只保留前 K 个候选, 其余全部淘汰.

Top-K = 3 时:

候选: 好(35%) 棒(18%) 不(12%) 错(8%) 糟(6%) 冷(5%)...

只有前 3 名有资格. 其余直接淘汰, 概率清零.

然后在 "好/棒/不" 三个里重新按比例采样.

  • K=1: 等于 Greedy (永远选第一名)
  • K=5: 保留 5 个候选, 过滤掉长尾
  • K=50: 保留很多, 几乎不过滤
  • K=全部: 等于不用 Top-K (OpenAI 默认)

问题: K 是固定数字. 但不同的概率分布, "有意义" 的候选数量不一样.

所以有了 Top-P ↓
(3)

Top-P (Nucleus): 保留到累计概率够了

Top-P 更聪明 — 它不固定候选数量, 而是 按累计概率决定保留多少.

Top-P = 0.9 时:

好(35%) → 累计 35%. 还不到 90%, 保留

棒(18%) → 累计 53%. 继续

不(12%) → 累计 65%. 继续

错(8%) → 累计 73%. 继续

糟(6%) → 累计 79%. 继续

冷(5%) → 累计 84%. 继续

闷(4%) → 累计 88%. 继续

热(3.5%) → 累计 91.5%. 超过 90% → 到此为止!

→ 保留 8 个候选. 后面 7 个全部淘汰.

为什么这比 Top-K 好?

  • 分布很尖 (一个词占 90%) → Top-P 自动只保留 1-2 个 → 输出稳定
  • 分布很平 (每个词 5-10%) → Top-P 自动保留 10+ 个 → 输出多样
  • 它自适应分布形状, 不需要你手动调 K
为什么 Top-P 通常更好? ↓
(4)

Top-P vs Top-K — 一张图说清

场景 1: 分布很尖 ("2+2=" → "4" 占 99%)
  • Top-K=5: 保留 5 个, 但 4 个都没意义 (概率 ≈ 0)
  • Top-P=0.9: 自动只保留 "4" 一个 → 完美!
场景 2: 分布很平 ("推荐一个颜色" → 红/蓝/绿/黄各 15%)
  • Top-K=3: 强制砍到 3 个 → 丢掉了合理的 "黄"
  • Top-P=0.9: 自动保留 6-7 个 → 合理的都留着

结论:

Top-P 自适应分布形状,
Top-K 是 固定的门槛.
大多数情况 Top-P 更好用.

那这三个参数怎么配合工作? ↓
(5)

完整采样流水线

Temperature、Top-K、Top-P 不是三选一 — 它们是流水线上的三个工序:

原始分数
(logits)
→
÷ Temperature
(调形状)
→
Top-K
(砍尾巴)
→
Top-P
(砍更多)
→
随机采样
(选一个)
先后顺序:
  1. Temperature 改变分布形状 (锐化或拉平)
  2. Top-K 砍掉排名靠后的 (硬门槛)
  3. Top-P 再砍掉累计概率之外的 (软门槛)
  4. 从剩下的候选里随机采样

→ Temperature 调 "形状", Top-P/K 调 "范围". 它们是互补的.

PM 实用建议:
  • 大多数 API 默认 T=1, Top-P=1 → 等于不过滤, 你必须自己设

  • 如果用 OpenAI: 推荐 调 Temperature OR Top-P, 不要同时调两个 (官方建议)
什么时候用哪个? ↓
(7)

实用指南: 什么时候调什么

场景推荐设置为什么
确定性输出 (医疗/法律)T=0直接 greedy, 最稳
稳定但有变化 (健康建议)T=0.3, Top-P=0.5温度低 + 过滤尾巴
自然对话T=0.7, Top-P=0.9默认偏稳, 少量过滤
创意写作T=1.0, Top-P=0.95温度高但有安全网
最大多样性T=1.2, Top-P=1.0放飞, 不过滤

经验法则:

  • 90% 的场景只调 Temperature 就够了
  • Top-P 是 Temperature 的 "安全网" — 防止低概率怪词漏进来
  • Top-K 很少需要手动调 — 除非你特别需要 "恰好 N 个候选"
