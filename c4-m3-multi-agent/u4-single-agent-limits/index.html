当 1 个 LLM 不够用

Vitamin AI 管家要同时管营养、周期、情绪... 一个 Agent 扛不住.

一个 ChatGPT 就能搞定所有事... 对吧?

等等, 用户问营养时它突然变成了周期专家?

对话长了它就开始重复自己? 还忘了我上次说了啥?

走起 →
(1)

简单任务, 单 Agent 确实够用

问"今天吃什么好", 一个 LLM 回答得很好. 问"帮我翻译这段话", 也没问题. 简单任务 = 单 Agent 的舒适区.

单 Agent 舒适区:
  • 单一领域的问答 (营养知识 OR 周期知识)
  • 短对话 (3-5 轮)
  • 不需要调用外部工具
  • 不需要记住历史对话
但 Vitamin 的需求远不止这些 ↓
(2)

错误直觉: "用更长的 Prompt 就行了"

✗

把所有指令塞进一个 System Prompt

"你是营养专家, 也是周期专家, 也是情绪顾问, 也是日程管理员..."

为啥不对: Prompt 越长, 模型越容易"忘记"中间的指令. 而且不同角色的指令会互相冲突.
✗

用更大的 Context Window 解决一切

"128K context 够放所有历史对话了吧?"

为啥不对: Context 长 ≠ 处理得好. 研究发现模型在超长 context 中容易"迷路", 中间部分的信息被忽略 (Lost in the Middle).
真相

单 Agent 有 4 个硬性极限:

复杂度 · 上下文 · 人格 · 工具

不是"够不够聪明"的问题, 是架构性的限制.

逐个看这 4 个极限 ↓
(3)

极限 1: 复杂多步骤任务

极限 1
任务太复杂, 需要多步推理 + 多次决策

"分析我的饮食记录 → 对比经期数据 → 生成个性化建议 → 设置提醒" — 这是 4 个子任务, 每个需要不同的能力.

单 Agent 处理复杂多步任务时, 错误会在步骤间累积. 第 1 步小错 → 第 4 步大错.

Multi-Agent 解法: 每个子任务交给专门的 Agent. 营养 Agent 分析饮食, 周期 Agent 分析经期, 规划 Agent 整合生成方案.
极限 2 ↓
(4)

极限 2: 上下文溢出

极限 2
对话太长, Context Window 装不下

用户连续聊 50 轮, 加上 System Prompt + 历史记录, 轻松超过 32K token. 旧的信息被截断, 模型"忘了"前面说过什么.

即使在 128K context 内, 模型对中间段信息的注意力也会显著下降 (Lost in the Middle 效应).

Multi-Agent 解法: 每个 Agent 只处理自己领域的 context. 营养 Agent 只看饮食相关上下文, 不用装 50 轮完整对话.
极限 3 ↓
(5)

极限 3: 人格 / 风格冲突

极限 3
同一个 Prompt 无法同时扮演多种角色

Vitamin 需要: 营养师(专业严谨) + 闺蜜(温柔共情) + 提醒助手(简洁高效). 一个 Prompt 塞进三种人格, 模型会"精神分裂".

用户说"我心情不好", 你希望闺蜜模式回应. 但模型可能突然切成营养师模式说"建议补充维生素D".

Multi-Agent 解法: 每个 Agent 有专属 Prompt 和人格. 情绪 Agent 是"温柔闺蜜", 营养 Agent 是"专业营养师". 各司其职, 不冲突.
极限 4 ↓
(6)

极限 4: 无法使用工具

极限 4
纯 LLM 不能查数据库、调 API、设提醒

"帮我查一下我上周的饮食记录" — LLM 本身无法访问数据库. "设一个明天早上 8 点的提醒" — LLM 无法调用日历 API.

没有工具调用能力的 Agent = 只能说不能做. 用户要的是行动, 不是建议.

Multi-Agent 解法: 每个 Agent 配备专属工具. 营养 Agent 连接食物数据库, 周期 Agent 连接经期记录, 提醒 Agent 连接日历 API.
4 个极限总结 ↓
(7)

4 个极限, 一张图

单 Agent 的 4 大极限:

复杂度 多步任务错误累积
上下文 Context 溢出/迷路
人格 多角色 Prompt 冲突
工具 纯 LLM 只能说不能做

解法: 拆 → 每个子任务交给专门的 Agent

一句话: 单 Agent 像一个全能型员工 — 什么都能做一点, 但什么都做不精. Multi-Agent 像一个专业团队 — 每人负责一块, 效率更高.

Vitamin 的升级计划 ↓
(9)

Vitamin: 从单 Agent 到 Multi-Agent 的升级理由

Vitamin 单 Agent 现状: 一个 LLM 扛所有功能 — 营养咨询 + 周期跟踪 + 情绪支持 + 提醒管理.

遇到的问题:

  • System Prompt 2000 token, 太长, 中间指令被忽略
  • 聊营养时突然切成"心理咨询师"语气
  • 无法查用户的历史饮食记录 (没有工具调用)
  • 复杂问题 (综合分析) 经常给出不一致的答案
升级方案预览 (后续 5 节会详细展开):
  • U5: 用 ReAct 框架让 Agent 边想边做
  • U6: 给 Agent 装 Tools (查数据库、调 API)
  • U7: 用 Router 把用户请求分发到对应的子 Agent
  • U8: 设计 Agent 间的 Handoff (工作交接)
  • U9: 搭建完整的 Vitamin Multi-Agent 架构
面试金句:

"我最初用单 Agent 实现 Vitamin AI 管家, 但发现 4 个瓶颈: 复杂度、上下文、人格冲突、工具缺失. 所以我升级到 Multi-Agent 架构, 用 Router + 3 个专属子 Agent + 共享 Memory 的方案."
