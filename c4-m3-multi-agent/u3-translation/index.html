(1)

"AI 不准" — 算法工程师最怕听到的四个字

这四个字传达了零信息. 算法不知道: 哪个功能不准? 不准的表现是什么? 多少不准算不可接受?

模糊版
"AI 不准"
→
工程版
"情绪分类模块的 Recall 只有 70%, 有 30% 的焦虑用户没被识别到. 目标: Recall ≥ 90%."

看到区别了吗? 工程版有三个要素:

  1. 指标 (Metric): 具体是什么指标 — Recall
  2. 数值 (Value): 当前值 70%, 目标值 90%
  3. 场景 (Context): 哪个模块, 什么后果
这就是翻译的核心框架 ↓
(2)

模糊不是错 — 但模糊的需求无法执行

核心认知

AI PM 的翻译框架:

模糊描述 → 指标 + 数值 + 场景

每次听到模糊需求, 脑子里自动启动这个翻译器: 什么指标? 当前多少? 目标多少? 在什么场景?

为什么这个能力重要?
  • 对上: 用业务语言和老板/用户沟通, 接收模糊需求
  • 对下: 用工程语言和算法/开发沟通, 输出可执行需求
  • 双向翻译 = AI PM 区别于传统 PM 的核心能力

(3)

案例 1: "推荐不准"

模糊
"推荐不准, 推的东西用户都不想看"
→
翻译
"推荐CTR当前2.1%, 目标5%+. 前10条曝光位相关性得分阈值从0.3→0.5"
翻译过程:
  1. 指标: "不准" → 用什么衡量? → 推荐场景看 CTR(点击率)
  2. 数值: 当前 CTR 2.1%, 行业平均 5%, 差距明确
  3. 场景: 前 10 条曝光位 (用户注意力集中的区域)
  4. 可执行动作: 提高相关性阈值, 加入近期行为特征

(4)

案例 2: "AI 太慢了"

模糊
"AI 回复太慢, 用户等不了"
→
翻译
"P95延迟3.2s, 目标≤1s. TTFT≥1.5s, 需启用streaming, 考虑降级到mini模型"
"慢"要区分两种:
  • TTFT Time To First Token — 用户按发送到看到第一个字的时间. 影响"反应感"
  • P95 延迟 95% 的请求完成时间. 影响整体体验
  • 优化方向: streaming(边生成边显示), 用更小模型, 加缓存层

(5)

案例 3: "AI 说错话"

模糊
"AI 经常说错话, 给了不靠谱的建议"
→
翻译
"有害回复率~5%, 目标≤0.5%. 需加safety guardrail + bad case库 + 每周人工审核"
"说错话"有不同严重等级:
  • 轻微 措辞不够好, 但信息正确 → 优化 Prompt
  • 中度 信息不准确, 但无害 → 加事实检查 + 引用来源
  • 严重 医疗/法律等有害建议 → Safety Guardrail + 硬过滤

案例 4 & 5: "成本太高" & "效果到瓶颈了"

案例 4: "成本太高"

模糊: "AI 功能成本太贵了"

翻译: "月API成本$2,400 (日8万次 × 600token × $0.005/1K). 优化: 短对话用mini模型(省70%), 高频问答缓存(减30%调用), 精简Prompt(从800→300token)."

案例 5: "效果到瓶颈了"

模糊: "模型效果到瓶颈了, 再优化空间不大"

翻译: "情绪分类F1=0.82, 误差分析: 30%错误来自'平静vs轻微开心'混淆. 方案: 合并相似类别(5→3类), 补充边界case标注500条, 尝试更大base model."

注意: 案例5 中, PM 要做的不是催算法"继续优化", 而是和算法一起做误差分析(Error Analysis), 找到错误集中在哪里, 然后针对性解决.

速查表: 5 大翻译模板

遇到模糊需求, 直接套这个模板.

模糊需求核心指标翻译模板
"不准"Precision / Recall / F1 / CTR"[功能]的[指标]当前X%, 目标Y%. 错误集中在[场景]."
"太慢"TTFT / P95 延迟"[功能]P95延迟Xs, 目标≤Ys. 考虑[streaming/小模型/缓存]."
"说错话"有害率 / 准确率"有害回复率X%, 目标≤Y%. 需加[guardrail/审核/bad case库]."
"太贵"单次成本 / 月总成本"月成本$X (日N次×M token×单价). 可优化: [模型降级/缓存/精简Prompt]."
"到瓶颈了"F1 / 误差分析"[指标]=X, 误差集中在[类型]. 方案: [合并类别/补数据/换模型]."
面试技巧: 面试官问"你怎么和算法沟通需求?" — 直接说这个框架: "我把模糊需求拆解为: 指标 + 当前值 + 目标值 + 场景, 确保算法能直接执行."

(9)

Vitamin 的 5 个需求翻译

"Vitamin 情绪识别不准" → 情绪分类 F1=0.78, 目标 ≥ 0.85. "平静"和"轻度焦虑"混淆率 35%, 需补充 300 条边界标注.

"AI 管家回复太慢" → 对话 TTFT=2.1s, 目标 ≤ 0.8s. 启用 streaming, 非复杂问题用 mini 模型.

"营养建议不靠谱" → 营养回复准确率 88%, 有害建议率 2%. 目标: 准确率 ≥ 95%, 有害率 ≤ 0.1%. 加医学知识库 RAG + guardrail.

"API 太贵了" → 月成本 ¥15,000. 60% 花在 System Prompt (太长). 优化: 精简 Prompt, 对话缓存, 分级调用.

"经期预测不准" → MAE(平均绝对误差)=3.2天, 目标≤1.5天. 需要更多用户历史数据 + 个性化特征.
