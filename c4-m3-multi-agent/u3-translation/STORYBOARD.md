# U3 Storyboard: 模糊需求 → 工程语言

## Hook
"用户说'AI不准'. 你该怎么跟算法工程师说?"

## Steps
1. "AI不准" 是最常见的模糊需求 — 算法听到后一脸问号
2. 错误直觉: 模糊 ≠ 错. 但模糊的需求无法执行
3. 翻译框架: 模糊描述 → 指标 + 数值 + 场景
4. 案例1: "推荐不准" → "点击率从2%提到5%, 前10条曝光"
5. 案例2: "AI太慢" → "P95延迟从3s降到1s"
6. 案例3: "AI说错话" → "有害回复率从5%降到0.1%"
7. 5个高频翻译模板 (速查表)
8. **Widget: Translation Simulator** — PM说模糊话, 你写工程翻译, 对比专家答案
9. Vitamin: 5个真实模糊需求翻译
10. 一句话规则 + 作业 + 资源延伸
