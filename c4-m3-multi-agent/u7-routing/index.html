Unit 7·
Multi-Agent 路由设计

用户说"我心情不好", 该交给情绪 Agent 还是营养 Agent? 路由 = Multi-Agent 的大脑.

用户的问题千变万化...

"经期能喝冰的吗?" — 这是营养问题还是周期问题?

"心情不好吃什么?" — 情绪还是营养?

—— 路由决定了用户体验的天花板.

10 步学完, 你会:
  • 理解 3 种路由策略 (规则/LLM/混合)
  • 处理模糊意图和多意图情况
  • 在路由设计器中实操连接意图到 Agent
  • 为 Vitamin 设计完整的路由方案
走起 →
(1)

路由 = 医院的导诊台

你去医院, 先到导诊台. 护士问你"哪里不舒服?", 然后告诉你"去3楼骨科" 或 "去2楼内科". 路由就是 Multi-Agent 的导诊台.

用户说一句话 → Router 判断意图 → 分发到对应的子 Agent.

Router 不回答问题, 它只做一件事: 判断这个问题该交给谁.

有哪些路由策略? ↓
(2)

3 种路由策略

规则路由

用关键词匹配: 包含"吃/饮食/营养" → 营养Agent; 包含"经期/月经/周期" → 周期Agent.

优点: 快速, 成本为0. 缺点: 处理不了模糊表达.

LLM 路由

用一个小 LLM 做意图分类: 输入用户消息, 输出 {intent: "nutrition"}.

优点: 理解自然语言. 缺点: 增加一次 LLM 调用成本.

混合路由

先用规则快速匹配, 匹配不到再用 LLM 兜底.

优点: 兼顾速度和准确. 推荐方案.

模糊意图怎么处理? ↓
(3)

难题: 模糊意图

"经期期间该怎么吃?" — 这既是周期问题, 也是营养问题. Router 该发给谁?

3 种处理策略:
  1. 主意图优先: "怎么吃" 是核心问题 → 营养 Agent (带上经期状态作为上下文)
  2. 链式调用: 先问周期 Agent 当前状态 → 再把状态传给营养 Agent 生成建议
  3. 询问用户: "你想了解经期相关知识, 还是想要这个阶段的饮食建议?"
Router 怎么实现? ↓
(4)

Router 的实现: 一个 Prompt 就够了

最简单的 LLM Router: 一个带分类指令的 Prompt.

System Prompt:
你是一个意图分类器. 用户发来一条消息,
你需要判断它应该交给哪个Agent处理.

可选Agent:
- nutrition: 饮食、营养、食物相关
- cycle: 经期、月经、周期相关
- mood: 情绪、心情、压力相关

输出JSON: {"agent": "xxx", "confidence": 0.9}

User: "我心情不好想吃点好的"
→ {"agent": "mood", "confidence": 0.7}
信心度 (confidence) 有什么用? ↓
(5)

Confidence 阈值 — 路由的安全阀

Router 不确定的时候怎么办? 用 confidence 阈值控制:

  • confidence ≥ 0.8 → 直接路由到对应 Agent
  • 0.5 ≤ confidence < 0.8 → 路由到 Agent, 但标记为"低信心", Agent 可以追问
  • confidence < 0.5 → Fallback: 用通用 Agent 回答, 或反问用户"你是想了解...还是...?"
PM 决策: confidence 阈值设多高, 直接影响用户体验. 太高 = 很多问题走 fallback, 太低 = 经常路由错误.
多意图怎么办? ↓
(6)

多意图: 一句话包含多个需求

"我心情不好, 经期快来了, 今晚吃什么好?" — 三个意图, 三个 Agent.

处理策略:
  1. 拆分意图: Router 输出 [{agent: "mood"}, {agent: "cycle"}, {agent: "nutrition"}]
  2. 优先级排序: 情绪优先 (用户说"心情不好"是主诉) → 先回应情绪, 再给建议
  3. 合并回复: 三个 Agent 的回复由一个 Composer Agent 合并成连贯的一段话
Fallback 设计 ↓
(7)

Fallback: 当 Router 搞不定的时候

用户说了一句完全不相关的话: "明天天气怎么样?" — 三个子 Agent 都不合适.

Fallback 策略 (按优先级):
  1. 通用 Agent 兜底: 配一个通用对话 Agent, 处理所有非核心意图
  2. 诚实说不: "抱歉, 我主要帮你管理饮食、周期和情绪. 天气查询可以试试手机自带的天气 App 哦~"
  3. 引导回核心: "说到天气, 今天温度挺高的, 记得多喝水! 对了, 你今天吃了什么呀?"

第 3 种最巧妙 — 既不拒绝用户, 又把对话引回产品核心场景.

Vitamin 路由设计 ↓
(9)

Vitamin 的路由设计方案

Vitamin 路由架构:
  • Router: 混合路由 (关键词规则 + GPT-4o-mini 兜底)
  • 营养 Agent: 饮食建议、营养分析、食物查询、喝水提醒
  • 周期 Agent: 经期状态、预测、阶段知识、异常提醒
  • 情绪 Agent: 情绪安抚、压力缓解、心情日记、正念引导
  • Fallback: 通用对话 + 引导回核心场景
  • Confidence 阈值: 0.7 (低于0.7走追问策略)
面试金句:

"Vitamin 的 Multi-Agent 路由我用混合策略 — 规则匹配覆盖 70% 的高频意图(成本为零), LLM 分类处理剩余 30% 的模糊意图. confidence 低于 0.7 时走追问策略, 避免路由错误."
