Unit 5·
ReAct 框架

Reasoning + Acting = 边推理边行动. Agent 不再只是"说", 它开始"做"了.

ChatGPT 是"想完再说".

但 Vitamin AI 管家需要"想一步做一步".

查数据 → 分析 → 再查 → 给建议.

—— 这就是 ReAct: 边想边做.

10 步学完, 你会:
  • 理解 ReAct 的 Thought→Action→Observation 循环
  • 区分 Chain-of-Thought 和 ReAct 的差异
  • 通过动画追踪器看 Agent 的推理过程
  • 为 Vitamin AI 管家设计 ReAct 工作流
走起 →
(1)

你是怎么帮朋友找餐厅的?

朋友说"帮我找个好吃的日料". 你不会闭着眼睛猜 — 你会:

  1. 想: "她在哪个区? 预算多少? 有没有忌口?"
  2. 做: 打开大众点评搜"日料 + 她的位置"
  3. 看结果: "评分 4.8, 人均 200, 评价说刺身很新鲜"
  4. 再想: "她不吃生的... 这家不行, 再找"
  5. 再做: 搜"熟食日料"...

这个"想 → 做 → 看结果 → 再想"的循环, 就是 ReAct.

跟 Chain-of-Thought 有什么不同? ↓
(3)

CoT vs ReAct — 关键区别

Chain-of-Thought (CoT)

LLM 在脑子里想完所有步骤, 然后一次输出答案.

问题: 如果中间某步需要外部数据 (如查数据库), CoT 只能编造.

类比: 闭着眼睛做数学题, 靠记忆, 可能算错.

ReAct

LLM 想一步, 做一步, 看到真实结果再继续想.

优势: 每步都用真实数据验证, 大幅减少幻觉.

类比: 睁着眼睛做题, 每步查一下, 基本不会算错.

PM 要点: CoT 适合纯推理任务. ReAct 适合需要外部数据/工具的任务. Vitamin 大部分场景都需要查数据, 所以 → ReAct.
具体怎么循环? ↓
(4)

Thought: Agent 的内心独白

Thought 是 Agent 的"想法". 它不直接输出给用户, 而是用来规划下一步.

示例 Thought:
"用户问了关于经期饮食的问题. 我需要先查她的经期状态, 然后根据当前阶段给营养建议. 先调用 get_cycle_status."

Thought 的价值: 可追溯 + 可调试. 出了问题, PM 可以回看 Agent 的 Thought, 理解它为什么做了错误决策.

Thought 之后是 Action ↓
(5)

Action: Agent 开始"做事"

Action 是 Agent 调用外部工具的步骤. 不是幻想, 是真的去查/去做.

示例 Action:
Action: get_cycle_status(user_id="u_12345")
→ 真的发了一个 API 请求给经期数据库

Action 可以是: 查数据库, 调 API, 搜索网页, 执行计算... 任何你给 Agent 配置的工具.

Action 之后是 Observation ↓
(6)

Observation: 真实世界的反馈

Observation 是工具返回的真实数据. Agent 根据这个数据决定下一步.

示例 Observation:
{phase: "黄体期", day: 22, next_period: "约5天后"}

关键: Observation 来自真实数据, 不是 LLM 编造的. 这就是 ReAct 减少幻觉的核心机制 — 用真实数据"锚定"推理.

循环几次后, 给出最终答案 ↓
(7)

循环 → 直到足够回答

ReAct 不是固定循环 3 次. 而是 Agent 自己判断: "信息够了吗? 够了就回答, 不够就再查一轮."

典型循环次数:
  • 1 次 简单查询: "我现在第几天?" → 查一次经期数据库就够了
  • 2-3 次 中等复杂: "经期饮食建议" → 查经期状态 + 查营养库
  • 4-5 次 复杂综合: "综合分析我的健康状况" → 多次查询 + 综合推理

循环越多 = 成本越高 (每轮都是一次 LLM 调用). PM 要设置最大循环次数, 防止 Agent 陷入无限循环.

Vitamin 的 ReAct 设计 ↓
(9)

Vitamin AI 管家的 ReAct 工作模式

Vitamin 的 ReAct 设计:
  • 可用工具: get_cycle_status(), query_nutrition_db(), get_diet_log(), set_reminder(), get_weather()
  • 最大循环: 5 轮 (防止无限循环, 超过 5 轮强制回答)
  • Thought 可见性: debug 模式下可以看到 Agent 的 Thought, 生产环境隐藏
  • 错误处理: 如果 Action 失败 (如数据库超时), Agent 应该在 Thought 中说明并尝试替代方案
面试金句:

"Vitamin 的 AI 管家使用 ReAct 框架 — Agent 每次回答前先推理需要什么数据, 然后调用工具获取真实数据, 再基于真实数据生成回复. 这比纯 LLM 回答减少了 80% 的幻觉."
