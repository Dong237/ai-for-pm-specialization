第一对: Precision & Recall ↓
(3)

Precision & Recall — 最核心的一对

Precision (精确率)
模型说"是"的里面, 真的有多少是"是"

Precision 高 = 误报少. 模型很谨慎, 说"是"的基本都对.

类比: 网购退货审核. Precision 高 = 批准退货的基本都是合理请求, 很少批错. 但可能有些合理退货没批到.
Recall (召回率)
所有真的"是"里面, 模型找到了多少

Recall 高 = 漏报少. 模型很积极, 尽量不遗漏.

类比: 安检. Recall 高 = 危险物品几乎都能检出, 但可能会误报一些无害物品.
PM 关键判断: Precision 优先还是 Recall 优先?
  • Precision 优先 推送通知 (误报 = 骚扰用户)
  • Recall 优先 异常检测 (漏报 = 错过危险)
  • Vitamin: 情绪分类用 Precision 优先 (别把开心判成焦虑), 异常检测用 Recall 优先 (别漏掉异常)
Precision + Recall 怎么平衡? ↓
(4)

F1 Score & AUC — 综合评估

F1 Score
Precision 和 Recall 的调和平均值

当 Precision 和 Recall 无法两全时, F1 给你一个综合分. 范围 0-1, 越高越好.

类比: 高考总分. 语文和数学不能只看一科, F1 = 两科的综合表现.
AUC (Area Under Curve)
ROC 曲线下的面积, 衡量模型区分正负样本的综合能力

AUC = 0.5 意味着模型跟随机猜一样烂. AUC = 1.0 意味着完美区分. 一般 0.8+ 算不错.

类比: 品酒师区分红酒好坏的综合能力. AUC 0.9 = 这个品酒师 90% 的情况能判断对.
PM 使用场景:

写 PRD 时: "情绪分类模型要求 F1 ≥ 0.8, AUC ≥ 0.85". 这样算法工程师就知道你要什么了.

下一对: Embedding & Token ↓
(5)

Embedding & Token — 模型怎么"看"文字

Embedding (向量嵌入)
把任何东西变成一组数字, 相似的东西距离近

文字、图片、用户行为, 都可以变成向量. 向量之间的距离 = 相似度.

类比: 把城市标在地图上. 北京和天津离得近(相似), 北京和纽约离得远(不相似). Embedding = 把概念标在"意义地图"上.
Token (词元)
LLM 处理文本的最小单位

不是字, 不是词, 是模型自己切的"块". 中文通常 1 字 ≈ 1-2 个 token.

类比: 乐高积木. 整篇文章 = 一个乐高城堡, token = 一块块积木. 不同模型用不同大小的积木.
PM 实操: Token 数 = 成本. Vitamin 每次对话消耗 ~500 token, API 按 token 计费. 你要算: 月活 × 每日对话次数 × 500 token = 月成本.
Context Window & Fine-tuning ↓
(6)

Context Window & Fine-tuning — 记忆与定制

Context Window (上下文窗口)
模型一次能"看到"的最大 token 数

超出窗口 = 模型忘记前面的内容. GPT-4o 是 128K, Claude 是 200K.

类比: 办公桌面积. 桌子越大, 能同时摊开的文件越多. Context Window = 模型的"桌子大小".
Fine-tuning (微调)
在预训练模型上, 用你的数据再训练一轮

让通用模型变成你的专属模型. 需要标注数据 + 训练资源.

类比: 请一个通用翻译, 让他专门学你们行业的术语. Fine-tuning = "岗前培训".
PM 决策点: 什么时候需要 Fine-tuning?
  • 通用模型 + Prompt 工程搞不定你的场景
  • 你有足够的标注数据 (通常 1000+ 条)
  • 需要稳定的、可复现的输出风格
最后两个: RLHF & Emergence ↓
(7)

RLHF & Emergence — 模型是怎么变"聪明"的

RLHF (人类反馈强化学习)
让人类给模型的回答打分, 模型学会"什么回答更好"

ChatGPT 为什么比原始 GPT-3 好用这么多? 核心就是 RLHF — 人类标注员训练它"懂事".

类比: 新员工入职. 导师看他的工作成果, 给反馈"这样做更好". 几百轮之后, 他就变成老手了.
Emergence (涌现)
模型规模大到一定程度, 突然出现没专门训练过的能力

没人教 GPT-3 做数学, 但 1750 亿参数突然就能做简单加法了. 这种"突然会了"就是涌现.

类比: 蚂蚁群体. 单只蚂蚁很蠢, 但上万只蚂蚁形成群体后, 突然展现出复杂的建筑和协作能力.
PM 认知: 涌现意味着你不能精确预测大模型能做什么. 新版本发布时, 先测试再承诺. 别看了 demo 就写进 PRD.

(9)

Vitamin 的 10 个术语实战

每个术语在 Vitamin 里都有具体落地场景. 面试时直接举例.

  • Precision: 情绪分类 — 判定"焦虑"的准确率, 误报会吓到用户
  • Recall: 异常检测 — 不能漏掉经期异常, 漏报有健康风险
  • F1: 营养建议分类 — Precision/Recall 需要平衡
  • AUC: 用户流失预测 — 模型区分"会流失"和"不会流失"的能力
  • Embedding: 食物相似度 — "苹果"和"梨"的向量距离近
  • Token: 成本控制 — 每次对话 500 token × API 单价 = 月成本
  • Context Window: 对话记忆 — 用户一周聊天不超过 5K token, 够用
  • Fine-tuning: 个性化 — 用 Vitamin 对话数据微调, 让回复更懂女性用户
  • RLHF: 语气优化 — 用用户评分训练模型, 让回复更温柔
  • Emergence: 新能力发现 — 更新模型版本后, 测试是否有新能力可用
面试金句:

"我在 Vitamin 项目中, 情绪分类模块我选择优先优化 Precision — 因为误判用户情绪的代价远大于漏判. 而异常检测模块我优先 Recall — 宁可多提醒, 也不能漏掉潜在健康风险."
