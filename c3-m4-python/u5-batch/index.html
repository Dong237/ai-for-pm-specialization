(1)

直接 for 循环会怎样?

你写了个循环, 100 条数据不带任何等待直接冲:

# 天真版: 不做任何限速 results = [] for q in questions: # 100 条 answer = ask_ai(q) results.append(answer) # 跑到第 7 条: 429 RateLimitError! # 每分钟只能请求 60 次, 你 1 秒请求了 7 次

问题: API 有频率限制 (比如每分钟 60 次). 你不加限制地连续调用, 几秒钟就触发 429 限频.

Rate Limit 到底是什么? ↓
(2)

Rate Limit -- API 的 "限速牌"

类比

Rate Limit 就像高速公路的限速牌:

路面能跑 200km/h, 但限速 120km/h. 超速了就罚款 (429 错误).

API 也一样: 服务器有容量上限, 给每个用户设了 "每分钟最多 N 次" 的限制.

模型RPM (每分钟请求数)TPM (每分钟 token 数)
gpt-4o-mini500200,000
gpt-4o50030,000
gpt-4.1500200,000

PM 须知: 新注册账号的 Rate Limit 更低 (Tier 1 只有 60 RPM). 充值 $50 以上才能提到 500 RPM.

怎么解决? 最简单的一行代码 ↓
(3)

time.sleep() -- 最简单的限速方案

import time results = [] for i, q in enumerate(questions): answer = ask_ai(q) results.append(answer) print(f"[{i+1}/{len(questions)}] Done") time.sleep(1) # 每次调用后等 1 秒 — 就这么简单!

time.sleep(1) 表示 "暂停 1 秒". 100 条数据就是 100 秒 (不到 2 分钟), 完全可以接受. 远比手动复制粘贴快.

加上进度显示, 让等待不那么无聊 ↓
(4)

加进度显示 -- 知道跑到哪了

import time questions = ["问题1", "问题2", ...] # 100 条 results = [] total = len(questions) for i, q in enumerate(questions): answer = ask_ai(q) results.append(answer) # 进度显示 pct = (i + 1) / total * 100 print(f"[{i+1}/{total}] {pct:.0f}% {q[:20]}... done") time.sleep(1) print(f"\n全部完成! 共 {len(results)} 条")
进阶: 结合错误处理 + 限速 ↓
(5)

完整版: 错误处理 + 限速 + 进度

import time def batch_process(questions, delay=1.0): """批量处理问题列表, 带限速和错误处理.""" results = [] total = len(questions) for i, q in enumerate(questions): answer = ask_ai(q) # ask_ai 已经有 try/except results.append({ "index": i + 1, "question": q, "answer": answer, "status": "error" if "ERROR" in answer else "ok" }) pct = (i + 1) / total * 100 status = "OK" if "ERROR" not in answer else "FAIL" print(f"[{i+1}/{total}] {pct:.0f}% {status}") if i < total - 1: # 最后一条不用等 time.sleep(delay) ok = sum(1 for r in results if r["status"] == "ok") print(f"\n完成: {ok}/{total} 成功") return results
来看两种延迟策略的对比 ↓
(6)

固定延迟 vs 指数退避 -- 哪个好?

策略代码适用场景
固定延迟 time.sleep(1) 数据量小 (<200 条), 简单可靠
指数退避 time.sleep(2 ** n) 遇到 429 时重试, 越等越久
自适应 正常时 sleep(1), 429 时退避 最佳: 快时快, 慢时慢

PM 建议: 100 条数据用 time.sleep(1) 就够了. 等到你需要跑 10,000 条时, 再考虑更复杂的策略.

还有一个实用技巧: 断点续跑 ↓
(7)

断点续跑 -- 跑一半断了也不怕

批量处理的终极保险: 每条结果即时保存, 崩了从断点继续.

import json def batch_with_checkpoint(questions, output_file="results.json"): # 加载已有结果 (如果之前跑过一部分) try: with open(output_file) as f: results = json.load(f) start_from = len(results) print(f"从第 {start_from + 1} 条继续...") except FileNotFoundError: results = [] start_from = 0 for i in range(start_from, len(questions)): answer = ask_ai(questions[i]) results.append({"q": questions[i], "a": answer}) # 每条都保存 — 即使崩了, 已跑的都在 with open(output_file, "w") as f: json.dump(results, f, ensure_ascii=False, indent=2) print(f"[{i+1}/{len(questions)}] saved") time.sleep(1)

PM 技巧: 跑到一半电脑没电了? 重新运行脚本, 自动从上次断点继续. 不重复调用, 不浪费钱.
