API 怎么收费? 按 Token, 不按次数

API 不是 "每次调用收 X 元", 而是 "你发了多少 token + AI 回了多少 token, 分别计费".

模型输入价格 (每百万 token)输出价格 (每百万 token)
gpt-4o-mini$0.15$0.60
gpt-4o$2.50$10.00
gpt-4.1$2.00$8.00
gpt-4.1-mini$0.40$1.60

gpt-4o-mini 有多便宜? 一次典型调用 (50 token 输入 + 100 token 输出) = $0.000068. 跑 100 条 = 不到 1 分钱. PM 做测试完全 OK.

怎么获取每次调用的 token 数? ↓
(2)

response.usage -- API 自带的 token 计数

response = client.chat.completions.create( model="gpt-4o-mini", messages=[...] ) # API 返回里自带 token 统计 usage = response.usage print(f"输入: {usage.prompt_tokens} tokens") print(f"输出: {usage.completion_tokens} tokens") print(f"总计: {usage.total_tokens} tokens") # 输出: # 输入: 42 tokens # 输出: 98 tokens # 总计: 140 tokens

注意: response.usage 是 API 返回的精确值, 不是估算. 用这个数算费用最准.

有了 token 数, 怎么算费用? ↓
(3)

算钱公式 -- 一行代码

# gpt-4o-mini 价格 (每个 token) INPUT_PRICE = 0.15 / 1_000_000 # $0.00000015 OUTPUT_PRICE = 0.60 / 1_000_000 # $0.0000006 def calc_cost(usage): """根据 usage 对象算费用.""" input_cost = usage.prompt_tokens * INPUT_PRICE output_cost = usage.completion_tokens * OUTPUT_PRICE return input_cost + output_cost # 例: 42 输入 + 98 输出 # = 42 * 0.00000015 + 98 * 0.0000006 # = $0.0000063 + $0.0000588 # = $0.0000651 (约 0.007 分人民币)
怎么在调用前就预估 token 数? ↓
(4)

tiktoken -- 调用前预估 Token 数

import tiktoken # 加载 gpt-4o-mini 的 tokenizer enc = tiktoken.encoding_for_model("gpt-4o-mini") # 计算文本的 token 数 text = "经期可以运动吗?" tokens = enc.encode(text) print(f"'{text}' = {len(tokens)} tokens") # 输出: '经期可以运动吗?' = 7 tokens # 看看 token 长什么样 print(tokens) # [12345, 23456, ...] print([enc.decode([t]) for t in tokens]) # ['经', '期', '可以', '运动', '吗', '?']

用途: 调用前预估 token 数, 就能在发请求之前算出大概费用. 比如你要批量处理 1000 条, 先抽 10 条算一算, 估算总费用.

来建一个完整的成本追踪系统 ↓
(5)

成本追踪系统 -- 每次调用都记账

# 成本追踪器 class CostTracker: def __init__(self, model="gpt-4o-mini"): self.calls = [] self.total_cost = 0 self.total_tokens = 0 # 价格表 (每个 token) self.prices = { "gpt-4o-mini": (0.15/1e6, 0.60/1e6), "gpt-4o": (2.50/1e6, 10.0/1e6), } self.model = model def log(self, question, usage): ip, op = self.prices[self.model] cost = usage.prompt_tokens * ip + usage.completion_tokens * op self.total_cost += cost self.total_tokens += usage.total_tokens self.calls.append({ "question": question[:30], "prompt_tokens": usage.prompt_tokens, "completion_tokens": usage.completion_tokens, "cost": cost }) def summary(self): print(f"总调用: {len(self.calls)} 次") print(f"总 Token: {self.total_tokens:,}") print(f"总费用: ${self.total_cost:.5f}") print(f"均费用: ${self.total_cost/len(self.calls):.6f}/call")
把追踪器整合进 ask_ai() 函数 ↓
(6)

整合: ask_ai() + 成本追踪

tracker = CostTracker("gpt-4o-mini") def ask_ai_tracked(question, system_prompt="你是 Vitamin 健康助手."): resp = client.chat.completions.create( model="gpt-4o-mini", messages=[ {"role": "system", "content": system_prompt}, {"role": "user", "content": question} ], temperature=0.3, max_tokens=500 ) # 记录成本 tracker.log(question, resp.usage) return resp.choices[0].message.content # 使用 for q in questions: ask_ai_tracked(q) time.sleep(1) # 最后打印汇总 tracker.summary()
来看预估月成本的方法 ↓
(7)

预估月成本 -- PM 必须会的计算

面试题: "Vitamin 上线后一个月 API 费用多少?" 你要能答:

# Vitamin 月成本预估 daily_users = 1000 # 日活用户 msgs_per_user = 3 # 每人每天平均对话数 avg_input_tokens = 80 # 每次输入 ~80 token avg_output_tokens = 150 # 每次输出 ~150 token daily_calls = daily_users * msgs_per_user # 3,000 次/天 monthly_calls = daily_calls * 30 # 90,000 次/月 # gpt-4o-mini 定价 input_cost = monthly_calls * avg_input_tokens * 0.15 / 1e6 output_cost = monthly_calls * avg_output_tokens * 0.60 / 1e6 total = input_cost + output_cost print(f"月调用: {monthly_calls:,} 次") print(f"月费用: ${total:.2f}") # 月调用: 90,000 次 # 月费用: $9.18 (用 gpt-4o-mini 非常便宜!)

如果用 gpt-4o? 同样的量, 月费用 = $153. 差 16 倍. 这就是 PM 选模型的成本视角.

Vitamin 案例 -- 成本控制策略

✓ 省钱技巧

  • 默认用 gpt-4o-mini (便宜 16 倍)
  • 设 max_tokens=200 (限制回复长度)
  • system prompt 尽量精简
  • 缓存常见问题的回复
  • 设月预算上限 ($20)

✗ 烧钱陷阱

  • 用 gpt-4o 跑大量测试
  • system prompt 写 2000 字
  • 不限 max_tokens, AI 回 2000 字
  • 重复调用不缓存
  • 不看 Usage Dashboard
