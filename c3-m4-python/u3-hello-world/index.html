(1)

网页版 vs API -- 一张图说清

你在 ChatGPT 网页上打字聊天时, 背后发生的事跟 API 调用是完全一样的:

你的 Python 脚本
(替代网页)
→
OpenAI API
(同一个服务器)
→
GPT 模型回复
(同样的模型)

区别在哪?

  • 网页版: 手动输入, 一次一条, 不能批量
  • API: 代码调用, 可批量 100 条, 可自动化, 可定制参数

API 就是 "把网页聊天变成代码能操控的接口".

理解了? 来看最简单的 API 调用代码 ↓
(2)

10 行代码 -- 你的第一个 API 调用

这就是完整代码. 10 行, 不多也不少:

# first_call.py from openai import OpenAI from dotenv import load_dotenv load_dotenv() # 从 .env 读取 OPENAI_API_KEY client = OpenAI() # 自动使用环境变量中的 Key response = client.chat.completions.create( model="gpt-4o-mini", messages=[ {"role": "system", "content": "你是 Vitamin 健康助手, 用简洁友好的语气回答."}, {"role": "user", "content": "经期可以运动吗?"} ] ) print(response.choices[0].message.content)

为什么选 gpt-4o-mini? 它是目前性价比最高的模型: 质量接近 GPT-4o, 但价格只有 1/15. 非常适合 PM 做原型和测试.

来逐行拆解这 10 行代码 ↓
(3)

逐行拆解 -- 每一行在干嘛

行号代码干嘛用
1-2from openai import OpenAI导入 OpenAI 库
3from dotenv import load_dotenv导入 .env 读取工具
4load_dotenv()读取 .env 文件里的 Key
5client = OpenAI()创建 API 客户端 (自动用环境变量的 Key)
6-11client.chat.completions.create(...)发送请求: 指定模型 + 消息
12print(...message.content)打印 AI 的回复文本
最关键的是 messages 这个参数 ↓
(4)

三种角色: system / user / assistant

messages 是一个列表, 每条消息有一个 "角色". 就像一个聊天群:

system
→
设定角色 -- 告诉 AI "你是谁", 定义行为边界. 用户看不到.
user
→
用户输入 -- 就是你要问 AI 的问题.
assistant
→
AI 回复 -- AI 之前的回答. 多轮对话时用.
# 多轮对话示例 messages = [ {"role": "system", "content": "你是 Vitamin 健康助手."}, {"role": "user", "content": "经期可以运动吗?"}, {"role": "assistant", "content": "可以! 建议低强度运动..."}, {"role": "user", "content": "那瑜伽呢?"} # 接着聊 ]

PM 要点: system prompt 就是你的 "产品说明书". 它决定 AI 的人格、语气、知识边界. 你写好 system prompt = 你定义了产品.

代码理解了, 来看 API 返回的数据长什么样 ↓
(5)

API 返回的 JSON -- 不只是文字

API 返回的不是纯文字, 是一个结构化的 JSON 对象:

# 打印完整返回 print(response) # 输出类似: { "id": "chatcmpl-abc123", "model": "gpt-4o-mini", "choices": [{ "message": { "role": "assistant", "content": "经期可以适当运动..." } }], "usage": { "prompt_tokens": 42, # 你发的 token 数 "completion_tokens": 98, # AI 回复的 token 数 "total_tokens": 140 # 总计 (这个数决定费用) } }

usage 字段是 PM 的好朋友! 它告诉你每次调用花了多少 token. 后面 Unit 7 会教你用它算成本.

来看更多参数: temperature, max_tokens ↓
(6)

三个最常用的参数

除了 model 和 messages, 你还会经常用到这三个:

response = client.chat.completions.create( model="gpt-4o-mini", messages=[...], temperature=0.3, # 控制创意度 (0=严谨, 1=创意) max_tokens=500, # 限制回复最大长度 n=1 # 生成几条回复 (默认 1) )
参数作用Vitamin 推荐值
temperature创意度 (0-2)健康建议: 0.2-0.3, 聊天: 0.7
max_tokens回复最大 token 数200-500 (防止长篇大论)
n生成几条候选1 (省钱, 除非做 A/B 测试)
参数都清楚了, 来看完整的可复制模板 ↓
(7)

可复制模板 -- 你以后每次调 API 都从这里开始

把这段代码存为 template.py, 以后每个脚本都从它复制:

# template.py — PM 的 API 调用模板 from openai import OpenAI from dotenv import load_dotenv load_dotenv() client = OpenAI() def ask_ai(question, system_prompt="你是一个有帮助的助手."): """给 AI 发一个问题, 返回回答文本.""" response = client.chat.completions.create( model="gpt-4o-mini", messages=[ {"role": "system", "content": system_prompt}, {"role": "user", "content": question} ], temperature=0.3, max_tokens=500 ) return response.choices[0].message.content # 使用 answer = ask_ai("经期可以运动吗?", "你是 Vitamin 健康助手.") print(answer)

PM 技巧: 把 API 调用封装成 ask_ai() 函数后, 你每次调用只需要一行代码. 后面批量处理 100 条数据, 就是循环调这个函数.

代码能跑了, 来看 Vitamin 的 API 调用设计 ↓
(9)

Vitamin 案例 -- system prompt 设计

Vitamin 的 system prompt 就是你这个产品的 "灵魂". 初版长这样:

system_prompt = """你是 Vitamin, 一个女性健康 AI 助手. 规则: 1. 用温暖、亲切的语气, 像好朋友一样说话 2. 回复控制在 200 字以内 3. 涉及具体医疗诊断时, 提醒用户咨询医生 4. 不要使用专业术语, 用大白话解释 5. 根据用户当前的身体状态 (5 态之一) 给出建议 你知道的 5 种身体状态: - 经期 (menstrual) - 卵泡期 (follicular) - 排卵期 (ovulatory) - 黄体期 (luteal) - 无特殊状态 (normal) """

PM 产品思维: system prompt 就是你写给 AI 的 PRD. 它定义了:

  • 人格 -- 温暖好朋友 (不是冷冰冰的医疗机器人)
  • 边界 -- 不做诊断, 提醒就医
  • 格式 -- 200 字以内, 大白话
  • 知识 -- 5 种身体状态
