为什么 PM 要碰 Python?

你不是要转行当工程师. 你只需要 Python 做一件事: 写个脚本, 自动批量调 LLM API.

没有 Python 的 PM: 打开 ChatGPT 网页, 一条一条复制粘贴, 100 条用例测 2 小时.

有 Python 的 PM: 跑一个脚本, 100 条用例 3 分钟跑完, 自动输出 CSV 报告.

-- 这不是"会编程", 这是"会自动化". 跟你用 Excel 公式批量算数据一个道理.

那第一步, 先把 Python 装上 ↓
(2)

你的电脑上可能已经有 Python 了

先别急着下载, 打开终端 (Terminal) 试一下:

# Mac: 打开 "终端" 应用 # Windows: 打开 "命令提示符" 或 "PowerShell" python3 --version # 如果输出 Python 3.11.x 或 3.12.x, 恭喜, 已经装好了 # 如果报错 "command not found", 往下看怎么装
✓

Mac 用户 (最常见)

装 Homebrew, 然后 brew install python. 或者直接去 python.org 下载 .pkg 安装包.

✓

Windows 用户

去 python.org 下载安装包, 安装时勾选 "Add Python to PATH" (很重要!).

版本要求: Python 3.9 以上就行, 推荐 3.11 或 3.12. 不要装 Python 2.x, 那是上古版本.

装好了, 下一步理解一个重要概念: 虚拟环境 ↓
(3)

虚拟环境 (venv) -- 你需要知道的唯一概念

一句话: 虚拟环境 = 给每个项目一个独立的 "安装包文件夹".

类比

想象你有两个客户, 一个要求你用蓝色主题, 一个要红色主题.

如果你只有一个 PPT 模板, 每次切换就得全改. 虚拟环境就是给每个客户建一个独立的模板文件夹, 互不干扰.

# 创建虚拟环境 (只需要做一次) python3 -m venv .venv # 激活虚拟环境 # Mac / Linux: source .venv/bin/activate # Windows: .venv\Scripts\activate # 激活后, 终端前面会出现 (.venv) 标记 # (.venv) $ _ <-- 看到这个就对了

.venv 这个名字是约定俗成. . 开头表示隐藏文件夹, 不会碍眼. 你也可以叫 env, 但 .venv 是社区标准.

环境有了, 下一步装我们需要的包 ↓
(4)

pip install -- 安装你需要的包

pip 就是 Python 的 "应用商店". 我们这个模块一共需要 3 个包:

包名干什么用安装命令
openai调 OpenAI / 兼容 APIpip install openai
python-dotenv安全读取 .env 文件里的 API Keypip install python-dotenv
tiktoken计算 token 数量, 估算成本pip install tiktoken
# 一条命令装完 pip install openai python-dotenv tiktoken # 验证安装成功 python3 -c "import openai; print(openai.__version__)" # 输出类似: 1.x.x 就是成功了
三个包装好了, 接下来创建项目文件夹 ↓
(5)

项目文件夹结构 -- 5 个文件就够

一个 PM 自动化脚本, 不需要复杂的工程结构. 这就是你最终要建的:

vitamin-api/ ├── .env # API Key (绝对不能提交到 Git!) ├── .gitignore # 告诉 Git 忽略 .env 和 .venv ├── .venv/ # 虚拟环境 (自动生成) ├── input.csv # 输入数据 (用例列表) ├── main.py # 你的脚本 └── requirements.txt # 记录安装了哪些包
# 生成 requirements.txt (记录当前环境的包) pip freeze > requirements.txt # 以后别人拿到你的项目, 一条命令恢复环境 pip install -r requirements.txt

PM 生存技巧: requirements.txt 就像菜单, 记录了这个项目 "点了哪些菜". 别人拿到菜单, 就能复现一模一样的环境.

结构清楚了, 来写你人生中第一个 Python 文件 ↓
(6)

hello.py -- 你的第一个 Python 文件

打开任意文本编辑器 (推荐 VS Code), 新建一个文件叫 hello.py:

# hello.py — 你的第一个 Python 文件 name = "Vitamin PM" print(f"Hello, {name}! 你的 Python 环境搭好了.") # 小计算 users = 100 cost_per_call = 0.002 total = users * cost_per_call print(f"100 个用户, 每次 API 调用 ${cost_per_call}, 总成本: ${total}")
# 在终端运行 python3 hello.py # 输出: # Hello, Vitamin PM! 你的 Python 环境搭好了. # 100 个用户, 每次 API 调用 $0.002, 总成本: $0.2

恭喜! 如果你看到了输出, 说明 Python 环境 100% 没问题. 从现在开始, 你就是一个 "会跑 Python 脚本的 PM".

环境搞定, 来看 PM 为什么选 VS Code ↓
(7)

编辑器选择 -- VS Code 就够了

不需要 PyCharm, 不需要 Jupyter. 对 PM 来说, VS Code 是最佳平衡点:

工具优点缺点PM 推荐度
VS Code免费, 轻量, 插件多, AI 补全需要简单配置首选
PyCharm功能强大, 自动补全好太重, PM 用不上 80% 功能不必要
Jupyter交互式, 看数据好不适合写脚本数据分析时用
CursorAI 原生编辑器基于 VS Code, 可替代进阶选择

VS Code 必装插件:

  • Python (Microsoft 官方) -- 语法高亮 + 运行按钮
  • GitHub Copilot -- AI 帮你补全代码, PM 写脚本效率翻倍
