API Key 不就是个密码吗? 存哪儿有啥讲究?

(1)

一个真实的恐怖故事

2024 年, 一个开发者把 OpenAI API Key 写在代码里, push 到了 GitHub 公开仓库.

结果:

  • 15 分钟内, 有人用他的 Key 发了 10 万条请求
  • 一夜之间产生了 $2,300 的账单
  • OpenAI 没有退款 -- "Key 管理是用户责任"

GitHub 上有专门的扫描机器人, 实时抓取所有新提交中的 API Key. 从你 push 到被盗用, 平均不到 1 分钟.

所以, Key 绝对不能出现在代码里. 那放哪? ↓
(2)

错误做法 vs 正确做法

✗ 错误: 硬编码在代码里

import openai client = openai.OpenAI(api_key="sk-abc123...") # 千万别这样!

Key 直接暴露在源代码中, 任何看到这个文件的人都能拿走.

✓ 正确: 放在 .env 文件里

# .env 文件 (跟代码同目录, 但不提交到 Git) OPENAI_API_KEY=sk-abc123...
# main.py — 从环境变量读取 from dotenv import load_dotenv import os load_dotenv() api_key = os.getenv("OPENAI_API_KEY")

代码里看不到 Key, .env 文件只存在你本地电脑.

理解了原理, 来看具体操作步骤 ↓
(3)

Step-by-step: 创建 .env 文件

三步操作, 30 秒搞定:

# Step 1: 在项目根目录创建 .env 文件 # 用编辑器新建, 或用命令行: touch .env # Step 2: 在 .env 里写入你的 Key # 打开 .env, 写一行: OPENAI_API_KEY=sk-proj-你的key粘贴在这里 # Step 3: 在 Python 里读取 from dotenv import load_dotenv import os load_dotenv() # 读取 .env 文件 key = os.getenv("OPENAI_API_KEY") print("Key loaded:", key[:8] + "...") # 只打印前 8 位, 安全

注意: .env 文件里 = 两边不要加空格, 值不要加引号 (除非值里有特殊字符).

文件创建好了, 但还要防止它被 Git 提交 ↓
(4)

.gitignore -- 防止意外提交

.gitignore 就是一个"黑名单", 告诉 Git: 这些文件永远不要追踪.

# .gitignore 文件内容 # 环境变量 (API Key 在这里) .env # 虚拟环境 .venv/ # Python 缓存 __pycache__/ *.pyc # IDE 配置 .vscode/ .idea/

重要: .gitignore 必须在 git add .env 之前创建! 如果你已经不小心 add 了 .env, 要用 git rm --cached .env 把它从 Git 追踪里移除.

好了, 你的 Key 现在受到双重保护. 来看验证方法 ↓
(5)

验证: 确认 Key 没有被追踪

设置好之后, 用这两条命令确认:

# 检查 Git 是否追踪了 .env git status # 如果 .env 出现在 "Untracked files" 或者完全不出现, 就是安全的 # 如果出现在 "Changes to be committed", 赶紧 git rm --cached .env # 另一个验证方法: 搜索代码里有没有硬编码的 Key grep -r "sk-" *.py # 如果没有输出, 说明代码里没有写死 Key

好习惯: 每次 git push 之前, 跑一遍 git diff --cached 看看即将提交的内容, 确认没有敏感信息.

来看安全管理的完整流程图 ↓
(6)

Key 管理的三层防线

生产级的 Key 管理有三层, PM 需要做到前两层:

Layer 1
→
.env 文件 -- Key 只存本地, 不进代码
Layer 2
→
.gitignore -- 防止 .env 被提交到 Git
Layer 3
→
密钥管理服务 (AWS Secrets / 1Password) -- 生产环境用, PM 了解即可

面试加分句: "我开发脚本时用 .env + .gitignore 管理 Key, 知道生产环境应该用 Secrets Manager, 但 PM 做原型阶段 .env 已经足够安全."

如果 Key 真的泄露了怎么办? ↓
(7)

泄露了怎么办? 30 秒应急

万一 Key 泄露 (推到了公开仓库), 立刻做这三件事:

紧急 3 步 (30 秒内):

  1. Revoke -- 去 OpenAI Dashboard → API Keys → 点 "Revoke" 废掉这个 Key
  2. Regenerate -- 生成一个新 Key, 更新你本地的 .env 文件
  3. Audit -- 查看 OpenAI Dashboard 的 Usage 页, 看有没有异常调用

Revoke 是最重要的一步. 旧 Key 废掉了, 机器人拿到也没用.

设置消费上限: 在 OpenAI Dashboard → Billing → Usage limits, 设一个月消费上限 (比如 $20). 即使 Key 泄露, 损失也有上限.

Vitamin 案例 -- Key 管理方案

Vitamin 作为健康类 App, 安全尤其重要. 你的 Key 管理方案:

✓ 你现在要做的

  • 本地开发: .env + .gitignore
  • 设 OpenAI 月消费上限 $20
  • GitHub 仓库设为 Private
  • 不在微信/飞书分享 Key

未来上线时要做的

  • 用云平台的 Secrets Manager
