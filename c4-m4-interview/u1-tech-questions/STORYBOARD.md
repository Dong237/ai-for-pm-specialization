## (1) 为什么面试官爱问技术题
AI PM 面试不考你写代码, 但考你能不能跟工程师对话. 7 道最高频的技术追问.

## (2) 错误回答 vs 正确回答
"我不懂技术" 是死刑. "我全懂" 也是吹牛. 正确: 用产品视角讲技术 trade-off.

## (3) Q1: RAG vs Fine-tune 怎么选?
决策树: 数据是否频繁更新? → RAG. 要风格统一? → Fine-tune. 预算有限? → 先 Prompt.

## (4) Q2: Token 成本怎么控?
输入/输出 token 定价, 缓存策略, prompt 压缩, 模型选型.

## (5) Q3: 幻觉怎么处理?
检测 + 缓解 + 产品层面兜底. Grounding, citation, confidence score.

## (6) Q4: Prompt Engineering 和代码有什么区别?
Prompt 是产品规格说明, 不是代码. 版本管理, A/B 测试, 评测集.

## (7) Q5-Q7: 评测/延迟/安全
Q5: 怎么评测模型好不好? Q6: 用户等太久怎么办? Q7: 数据安全怎么保障?
