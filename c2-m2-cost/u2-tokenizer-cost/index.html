为什么差这么多? 答案在训练数据里 ↓
(2)

BPE: 训练数据决定 token 效率

所有主流模型用的分词算法叫 BPE (Byte Pair Encoding). 它的原理很简单:

BPE 原理

BPE 统计训练文本中哪些字符对出现最频繁, 然后把它们合并成一个 token.

训练数据中英文多 → "the", "ing", "tion" 被合并成单个 token → 英文 token 效率高

训练数据中中文少 → 中文汉字没机会合并 → 每个字都是单独 token → 效率低

→ 这就是中文 "税" 的根源: 不是中文天生低效, 是 tokenizer 偏心了.

✗

GPT-2 时代 (英文至上)

训练数据 90%+ 英文, 中文汉字被拆成 UTF-8 字节, 一个汉字 = 3 个 token!

✓

Qwen / DeepSeek (中文优化)

训练数据包含大量中文, "人工智能" "信息基础设施" 整体成为一个 token. 效率反超英文!

GPT 后来改善了, 但中文依然比英文贵 ↓

这种差异带来的 "中文税" 有多大? ↓
(4)

中文 "税": 同样意思, 中文比英文贵 30-50%

在 GPT 模型上, 中文文本比等价英文文本多消耗 30-50% 的 token. 这就是所谓的 "中文税".

对比实验:

"人工智能正在改变世界" vs "AI is changing the world"

  • 中文在 GPT-5.5: ~8 tokens
  • 英文在 GPT-5.5: ~5 tokens
  • 中文多了 60%

→ 但在 Qwen / DeepSeek 上, 中文反而可能比英文 少 token — 中/英比率低至 0.65x!

PM 决策影响:

如果你的产品是中文为主 (Vitamin 就是), 选择 中文友好的 tokenizer (Qwen / DeepSeek) 能直接省 30-50% 的 token 费用. 这还没算上价格差, 只是 token 数量的差异.

Token 差异 x 价格差异 = 复合效应, 有多可怕? ↓
(5)

复合效应: Token 差异 × 价格差异

Tokenizer 差异和价格差异叠加, 产生惊人的复合效应.

算个账 — Vitamin 一天 1000 用户, 每人 5 次对话, 每次 ~500 token:

模型 Token/次 日 Token 月费用
GPT-5.5 ~575 2.875M $3,019
Claude Sonnet 4.6 ~520 2.6M $1,404
DeepSeek V4 Flash ~400 2.0M $25

GPT-5.5 是 DeepSeek 的 121 倍. 其中 ~35 倍来自价格差, ~3.5 倍来自 tokenizer 差异.

(7)

选模型的新维度: Tokenizer 友好度

除了 "模型能力" 和 "价格", 你还需要考虑第三个维度: Tokenizer 对你语言的友好度.

中文产品的 Tokenizer 友好度排名:

  1. 极好 Qwen3 系列 / DeepSeek V4 — 中文 token 效率比英文还高
  2. 好 GLM / 文心 — 国产模型, 中文优化
  3. 中等 GPT-5.5 (o200k) — 比 GPT-4 大幅改善, 但仍偏英文
  4. 中等 Claude (cl200k) — 类似 GPT-5.5 水平
  5. 一般 Gemini — 支持多语言但中文不突出

PM 决策框架:

选模型 = 能力够用 × 价格可承受 × tokenizer 友好

→ 对中文产品来说, tokenizer 友好度直接影响 30-50% 的成本.
