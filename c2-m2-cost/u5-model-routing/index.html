不是所有问题都需要 "最强大脑"

看看 Vitamin 一天收到的消息, 按复杂度分:

60% 简单 — "你好" / "帮我记一下早餐" / "今天心情还不错"

15% 复杂 — "我有多囊, 在吃达英-35, 想了解副作用" / 需要深度推理的问题

如果全用 Claude Opus 4.7: 60% 的 "你好" 也花 $0.0045/次. 这钱花得冤!

用 DeepSeek V4 Flash 处理 "你好": $0.00006/次. 质量完全够用. 省 75 倍.

路由的核心是什么? ↓
(2)

路由的核心: 复杂度 → 模型档位

路由公式

判断复杂度 → 选模型

简单 → 便宜模型 (DeepSeek V4 Flash)

中等 → 中档模型 (Claude Sonnet / GPT-5.4)

复杂 → 贵模型 (Claude Opus / GPT-5.5)

→ 像分诊台: 感冒去社区诊所, 骨折去骨科, 疑难杂症去专家门诊.

路由器怎么判断复杂度? ↓
(3)

路由器: 4 种判断复杂度的方式

1

关键词规则

包含 "你好" / "记录" → 简单. 包含 "副作用" / "诊断" → 复杂. 最简单, 0 成本.

2

输入长度

短消息 (<20 token) 大概率简单. 长消息 (>100 token) 可能复杂.

3

便宜模型先分类

用 DeepSeek V4 Flash 做一次 "这个问题是简单/中等/复杂?" 的分类, 成本 $0.00003.

4

功能级路由

不按消息分, 按功能分: 闲聊功能 = 便宜, 健康咨询 = 中档, 医疗问答 = 贵.

PM 推荐: 先用 方式 4 (功能级路由), 最简单最稳定. 后期再加 方式 3 (模型分类) 做精细化.

路由能省多少钱? ↓
(4)

路由的省钱数学

假设 Vitamin 每天 5000 次调用, 平均每次 $0.003 (用 Sonnet).

无路由 (全 Sonnet)
$450/月
5000 × 30 × $0.003
→
有路由 (三档)
$112/月
60% × $0.00006 + 25% × $0.003 + 15% × $0.005

路由省了 75% 的成本.

$450 → $112, 每月省 $338.

路由失误怎么办? ↓
(5)

路由失误 vs 不路由 — 哪个更贵?

!

路由错误: 复杂问题走了便宜模型

回复质量差, 用户不满意, 可能需要重试 → 花 2 倍钱

解决: 加 "兜底机制" — 如果便宜模型回复不确定度高, 自动升级到贵模型.
!

不路由: 全用贵模型

"你好" 也花 $0.005 → 60% 的钱都浪费了

结论: 路由偶尔失误 (5-10%), 总成本仍然远低于不路由. 不路由才是最大的浪费.

兜底机制: 不确定就升级 ↓
(7)

兜底机制: 便宜模型不行就升级

路由不是一锤子买卖. 加一个 "不确定就升级" 的兜底机制:

  1. 先用便宜模型回答
  2. 检查回复: 是否包含 "我不确定" / "建议咨询医生" / 置信度低?
  3. 如果是 → 自动升级到中/贵档重新回答
  4. 如果不是 → 直接返回便宜模型的回复

额外成本: 大概 5-10% 的请求会触发升级, 花了两次钱. 但总成本仍然远低于全用贵模型.

→ 这是 "保底质量" 和 "控制成本" 的最佳平衡点.

Vitamin 的完整路由方案

Vitamin 路由架构:

  1. 入口层: 按功能模块直接路由 (闲聊/记录 → 便宜, 健康问答 → 中档)
  2. 判断层: 中档模型回复后检查置信度, 低于阈值 → 升级贵档
  3. 兜底层: 贵档仍然不确定 → 返回 "建议咨询医生" + 附近医院推荐
