(1)

一条 API 调用的账单长什么样

每次你调 LLM API, 返回的 JSON 里都有一个 usage 字段. 这就是计费的依据.

{
  "model": "deepseek-v4-flash",
  "usage": {
    "prompt_tokens": 328,   // 你发的 (输入)
    "completion_tokens": 156,  // AI 回的 (输出)
    "total_tokens": 484
  }
}

三个数字决定你的账单:

  1. prompt_tokens — 你发给模型的内容 (system prompt + 用户输入 + 历史对话)
  2. completion_tokens — 模型生成的回复
  3. 费用 = prompt_tokens × 输入单价 + completion_tokens × 输出单价
那 "单价" 到底是多少? 答案是: 看你选的模型 ↓
(2)

不是 "按字收费" — 是按 Token 收费

✗

错觉: "一个字一分钱"

以为费用 = 字数 × 单价, 像短信一样.

为啥不对: Token ≠ 字. "unhappiness" 是 1 个英文词, 但 2 个 token (un + happiness). "你好" 可能是 1 个 token, 也可能是 2 个 — 取决于模型的 tokenizer.
✗

错觉: "每家收费差不多"

以为 LLM API 像云服务器, 价格差 2-3 倍封顶.

为啥不对: 最便宜的 DeepSeek V4 Flash 输入 $0.14/M, 最贵的 GPT-5.5 Pro 输入 $30/M — 差了 214 倍.
真相

LLM 计费的核心单位是 Token, 不是字、不是词、不是句.

1 token ≈ 4 英文字母 ≈ 1-1.5 中文字

→ 所有 API 的价格都标 "每百万 token" (per 1M tokens). 你需要先知道你的文本有多少 token, 才能算账.

价格标的 "per 1M tokens" — 这个单位是啥意思? ↓
(5)

计费单位: 每百万 Token (per 1M tokens)

别被 "百万" 吓到 — 你的一次调用通常只有几百到几千 token.

换算公式:

单次费用 = token 数 ÷ 1,000,000 × 单价

举个例子:

Vitamin 用户问: "经期可以跑步吗?"

  • 输入 ~350 tokens (system prompt 200 + 用户输入 50 + 历史 100)
  • 输出 ~200 tokens (AI 回复)

用 DeepSeek V4 Flash:

350 ÷ 1M × $0.14 + 200 ÷ 1M × $0.28 = $0.0001

用 Claude Opus 4.7:

350 ÷ 1M × $5.00 + 200 ÷ 1M × $25.00 = $0.0068

→ 同一个问答, Opus 是 DeepSeek 的 68 倍.

有些模型的账单里, 还有你看不见的 "隐藏 token" ↓
(6)

隐藏成本: Reasoning Tokens (思考 Token)

用 o3 / DeepSeek R1 这类 "推理模型", 你看到的输出是 500 token, 但它可能在背后 "想" 了 3000 token. 你全得付钱.

普通模型 (DeepSeek V4 Flash):

500 output tokens (全可见)
可见 output

推理模型 (o3 / DeepSeek R1):

2500 reasoning tokens (不可见, 但收费!)
500
隐藏 reasoning tokens 可见 output

这意味着: 推理模型的实际花费可能是你预估的 3-6 倍. 看到 500 output tokens, 实际账单可能按 3000 tokens 收.

→ 这就是为什么 Vitamin 不需要给每个问题都用推理模型. 简单问题用普通模型即可.

还有一种省钱方式: 不着急的任务, 用 Batch API ↓
(7)

Batch API: 延时换折扣, 省 50%

OpenAI / Anthropic 都提供 Batch API: 你把请求打包发过去, 24 小时内返回结果, 但价格打五折.

实时 API (Real-time)
  • 即时返回 (~1-5 秒)
  • 原价: 100%
  • 适合: 用户对话、实时交互
Batch API (批量)
  • 24 小时内返回
  • 原价的 50%
  • 适合: 内容生成、分析报告、数据标注

Vitamin 能用 Batch API 的场景:

  • 每晚生成次日的 "每日健康小贴士" (不需要实时)
  • 批量分析用户周报 / 月报 (可以等几小时)
  • 内容审核 (先存后审)

→ 把能延迟的任务移到 Batch, 立省 50%.

回到 Vitamin — 有了这些知识, 选什么模型? ↓
(9)

Vitamin 案例: 选模型的逻辑

不同功能用不同模型, 这叫 "模型分级". Vitamin 至少需要三档:

✓ 便宜档: DeepSeek V4 Flash

  • 日常对话 / 闲聊
  • 简单分类 (情绪 5 状态)
  • 内容改写 / 摘要

$0.14/$0.28 — 占 Vitamin 80% 的调用量

◆ 中档: Claude Sonnet 4.6

  • 健康建议生成
  • 多轮深度对话
  • 用户周报 / 月报

$3.00/$15.00 — 占 15% 调用量

◆ 贵档: Claude Opus 4.7 / GPT-5.4

  • 复杂健康问题需要深度推理
  • 敏感场景需要最高质量回复

$5.00/$25.00 — 占 5% 调用量, 但贡献 30%+ 的账单

面试金句:

"AI 产品选模型不是选最好的, 是按场景选够用的. 80% 的请求走便宜模型, 20% 的高价值请求走贵模型 — 这能省 60-80% 成本."
