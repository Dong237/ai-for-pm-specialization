(1)

Vitamin 的起点: ¥5,000/月

假设 Vitamin 有 1000 DAU, 每人每天 5 次对话, 全部使用 Claude Sonnet 4.6, 无任何优化.

基准成本 (无优化):

  • 模型: Claude Sonnet 4.6 ($3/$15)
  • 1000 DAU × 5 次/天 × 30 天 = 150,000 次/月
  • 每次: 输入 400 token + 输出 300 token
  • 月费: 约 $700 (¥5,040)
优化第 1 刀: 控制输出量 ↓
(2)

第 1 刀: 控制输出量 (省 20%)

输出从 300 token 降到 150 token

加 prompt 约束 "请用 2-3 句话回答" + 设 max_tokens + 用 JSON 做分类任务

¥5,040 → ¥4,000 (省 ¥1,040)

为什么先砍输出: 输出 token 比输入贵 5 倍. 砍 50% 输出, 总成本降 ~20%. 最简单, 0 工程成本.

第 2 刀: Prompt Caching ↓
(3)

第 2 刀: Prompt Caching (再省 38%)

缓存 system prompt (70% 命中率)

把 system prompt 固定在前, 动态内容放在 user message 里. Claude 缓存命中 90% 折扣.

¥4,000 → ¥2,500 (省 ¥1,500)

缓存效果: 400 token 输入中 280 token 是固定前缀. 缓存命中后, 这部分从 $3/M 降到 $0.30/M — 省 90%.

第 3 刀: 模型路由 ↓
(4)

第 3 刀: 模型路由 (再省 52%)

60% 走 DeepSeek V4 Flash, 25% 走 Sonnet, 15% 走 Opus

简单对话/记录/分类 → DS Flash ($0.14/$0.28). 健康问答 → Sonnet. 复杂推理 → Opus.

¥2,500 → ¥1,200 (省 ¥1,300)
还能更省 — 第 4 刀 ↓
(5)

第 4 刀: 选中文友好 Tokenizer + Batch API

两个额外优化:

  1. Tokenizer 优化: 中文内容在 DS Flash 上 token 更少 (省 ~20% token)
  2. Batch API: 非实时任务 (每日总结 / 内容审核) 走 Batch, 再省 50%
¥1,200 → ¥750 (省 ¥450)
看看完整的瀑布 ↓
(6)

起点 (无优化)¥5,040
+输出控制¥4,000

¥5,040 → ¥750. 每月省 ¥4,290.

优化的顺序很重要 ↓
(7)

优化顺序: 先做效果大、成本低的

优先级优化项省多少工程成本难度
1控制输出量 (prompt 约束)~20%0 (改 prompt 就行)低
2Prompt Caching (结构调整)~38%低 (调 prompt 顺序)低
3模型路由~52%中 (需要路由逻辑)中
4Tokenizer 优化 + Batch~38%中 (需要改模型选型)中

PM 规则: 先做改 prompt 就能生效的优化 (第 1、2 步), 再做需要工程改动的 (第 3、4 步). 从 ROI 最高的开始.
