PM 的终极产出物: 优先级排序

面试官不会问你 "学了什么". 他们会问: "给你一个产品, 你先做哪个功能, 为什么?"

一个好的功能优先级排序需要 4 个东西:

  1. 评估框架 — 用什么维度打分 (不是拍脑袋)
  2. 数据支撑 — 每个维度的分数怎么来的
  3. 视觉呈现 — 矩阵图, 一眼看出先后
  4. 行动路线 — 分阶段的 roadmap, 有时间有里程碑

今天你要做的就是: 把 M1.4 学过的所有框架, 用在一个真实决策上.

你的武器库:

U1 Augmentation 谱系 + U2-4 R/W/C 分类
+ U5 ROI + U6 Build/Buy/API + U7 TTV

全部汇聚到一个 2×2 矩阵里.

先认识一下 6 个候选功能 ↓

6 个候选都认识了. 先看第一个评分维度: ROI ↓
(3)

评分维度 1: ROI — 这个功能值不值得做?

回顾 U5 的 ROI 框架: 价值 = 用户量 × 使用频率 × 单次价值

功能用户量频率单次价值ROI 评分
🔍 症状关键词提取 全部用户 每天 高 (节省人工分诊) 9/10
💊 个性化健康建议 全部用户 每天 高 (核心价值主张) 9/10
💬 多轮对话 活跃用户 每周 中高 (提升留存) 7/10
📷 饮食拍照 部分用户 每天 中 (nice-to-have) 5/10
📝 术语翻译 全部用户 偶尔 中 (改善体验) 5/10
🧠 情绪分类 全部用户 每天 中高 (5状态核心) 7/10
关键观察:

"症状提取" 和 "健康建议" 得分最高 — 因为它们是 Vitamin 的核心价值主张, 每个用户每天都会用到.

"饮食拍照" 和 "术语翻译" 得分最低 — 一个是 "锦上添花", 一个用得太少.

再看第二个维度: Time-to-value ↓
(4)

评分维度 2: TTV — 多快能上线?

回顾 U7: TTV = 乐观估计 × 现实乘数. 速度越快, 得分越高.

功能路径乐观估计×1.5 实际TTV 评分
🔍 症状关键词提取 API 2 周 3 周 9/10
💊 个性化健康建议 API + 人审 3 周 4.5 周 7/10
💬 多轮对话 API 4-6 周 6-9 周 4/10
📷 饮食拍照 API + Vision 6-8 周 9-12 周 3/10
📝 术语翻译 API (简单) 1 周 1.5 周 10/10
🧠 情绪分类 API 2 周 3 周 9/10
关键观察:

"术语翻译" 最快 (1 周) — 因为任务简单, 不需要复杂的 prompt 工程.

"饮食拍照" 最慢 (9-12 周) — 多模态 + 中式菜品识别 + 营养数据库对接, 是最复杂的功能.

两个维度都有了, 放进矩阵里! ↓
(5)

2×2 矩阵: PM 最趁手的决策工具

把 ROI 放 Y 轴 (高→低), TTV 放 X 轴 (快→慢), 得到 4 个象限:

ROI 高 ↑
🟢 立刻做 ROI 高 + TTV 快 Phase 1 (1-3 周)
🔵 规划好再做 ROI 高 + TTV 慢 Phase 2 (月 2-3)
🟡 顺手做 ROI 低 + TTV 快 Phase 1 搭车
🔴 先不做 ROI 低 + TTV 慢 Phase 3+ 或放弃
← TTV 快     TTV 慢 →
为什么这个框架特别适合 AI 功能决策?
  • 直觉对齐: "先做容易的高回报的" — 这是人人都认同的逻辑
  • 对话工具: 跟工程师、老板沟通时, 一张图胜过千言万语
  • 迭代友好: AI 功能效果有不确定性, 快上线 → 快验证 → 快迭代
好, 把 6 个功能放进去 ↓
(6)

6 个功能, 放在哪个象限?

根据前两步的评分, 我们可以推导出每个功能的位置:

🟢 立刻做
🔍 症状关键词提取 ROI 9 + TTV 9 = 最完美的起步功能
🧠 情绪分类识别 ROI 7 + TTV 9 = 5 状态体系的核心支撑
🔵 规划好再做
💊 个性化健康建议 ROI 9 + TTV 7 = 高价值但需要合规审查
💬 多轮身体状态对话 ROI 7 + TTV 4 = 技术复杂, 需要稳扎稳打
🟡 顺手做
📝 医学术语翻译 ROI 5 + TTV 10 = 一周就能搞定, 搭车上线
🔴 先不做
📷 饮食拍照识别 ROI 5 + TTV 3 = 投入大回报低, 等技术成熟

"先不做" ≠ "永远不做"

饮食拍照识别是个好功能, 但在 Vitamin 的早期阶段, 资源应该集中在核心的 "身体状态追踪" 上. 等 Phase 1-2 跑通了, 再回来做这个. 到那时, Vision API 的价格和识别精度可能都更好了.

Vitamin AI 能力全景图

Vitamin · AI 功能优先级文档 (v1.0)

产品定位

女性健康 AI App, 核心能力: 5 种身体状态追踪 + 个性化健康陪伴

AI 功能矩阵

Phase 1 🔍 症状提取 🧠 情绪分类 📝 术语翻译
Phase 2 💊 健康建议 💬 多轮对话
Phase 3 📷 饮食拍照

关键决策依据

  • Phase 1 选择 ROI 最高 + TTV 最快的功能, 用 4 周跑通核心闭环
  • Phase 2 投入高价值但需要合规和技术攻关的功能
  • Phase 3 根据用户反馈决定是否启动多模态功能

技术选型

Phase 1-2 全部采用 API 路径 (GPT-4o / Claude), 不做 fine-tune. 原因: 团队首次 AI 项目, 优先速度和灵活性.

面试金句, 抄走:

"我用 ROI×TTV 矩阵给 Vitamin 的 6 个候选 AI 功能排了优先级. Phase 1 做了 3 个快速验证功能, 4 周上线; Phase 2 做合规要求高的个性化建议和多轮对话; Phase 3 视用户反馈决定是否做多模态. 全部 API 路径, 不做 fine-tune, 因为团队首次做 AI 功能."

→ 这段话涵盖了: 分类能力 + 评估能力 + 排序能力 + 技术选型能力 + 风险管理能力. 5 个 AI PM 核心能力, 一段话全部展现.
