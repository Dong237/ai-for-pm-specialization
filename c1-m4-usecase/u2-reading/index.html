(2)

一个常见误解 -- 和真相

×

误解: "Reading 就是让 AI 理解文本"

AI 读懂了内容, 然后什么都能做.

问题: "理解" 太模糊了. 同一段文字, Summarize 只关注 "重点是什么", Classify 只关注 "属于哪类", Extract 只关注 "有哪些实体". 它们需要不同的 prompt、不同的输出格式、不同的评估标准.

PM 不能笼统地说 "让 AI 读一下用户日记" -- 你必须指定: 是要压缩? 贴标签? 还是挖字段?
真相

Reading 不是 "理解" -- 是 3 种不同的信息变换:

📜
Summarize
长 → 短
🏷
Classify
文本 → 标签
⚒️
Extract
散乱 → 结构

→ 每种变换都有自己的 prompt 模板、输出格式、质量指标. 下面一个个拆.

先从第一个开始: Summarize ↓
(3)

Pattern 1: Summarize -- 长变短

Summarize 的本质: 保留意思, 压缩体积. 像把一杯果汁浓缩成一颗糖.

📝 输入: 用户日记 (87 字)
最近一周一直感觉头疼, 晚上老是睡不好, 凌晨三四点就醒了. 食欲也不太好, 早上基本不想吃东西. 心情有点低落, 可能跟工作压力大有关系. 上周开始吃了布洛芬, 头疼稍微好一点了.
📄 输出: 3 句摘要

• 头疼 + 失眠持续一周, 凌晨早醒

• 食欲差、情绪低, 疑似工作压力导致

• 已服布洛芬, 头疼有缓解

87 字 → 42 字. 压缩率 52%.

Vitamin 怎么用 Summarize?

  • 每周健康周报: 7 天的日记自动浓缩成 3 条核心
  • 医生端: 用户 3 个月的记录 → 一页摘要
  • 用户自己: "帮我看看这周身体整体怎么样"
第二种 Reading: Classify ↓
(4)

Pattern 2: Classify -- 贴标签

Classify 的本质: 把文本放进预定义的几个桶里, 并说出 AI 有多确信.

📝 输入: 用户消息
最近老是睡不好, 心烦, 不知道为什么就是提不起劲. 工作上也焦虑, 总觉得自己做得不够好.
🏷 输出: 情绪分类 + 置信度
焦虑
82%
疲惫
71%
沮丧
58%
平静
12%
愉快
5%
主要情绪: 焦虑 次要: 疲惫

Vitamin 怎么用 Classify?

  • 情绪日记分类: 用户写完 → AI 自动贴情绪标签 (可让用户修正, L3 增强)
  • 消息意图分类: 用户输入 → 是提问 / 倾诉 / 记录 / 求助?
  • 症状严重度分级: 轻微 / 中等 / 严重 / 紧急
第三种 Reading: Extract ↓
(5)

Pattern 3: Extract -- 挖结构

Extract 的本质: 从散乱的自然语言中, 挖出有名字的字段. 像考古一样, 从泥土里刨出宝贝.

📝 输入: 用户消息
头疼、失眠、食欲不好, 上周开始吃了布洛芬, 昨天量了体温37.2. 经期第三天, 量比较大.
📋 输出: 结构化数据
symptoms: [头疼, 失眠, 食欲不好]
medication: [布洛芬]
vitals: {体温: 37.2}
period: {day: 3, flow: heavy}
timeline: 上周 (布洛芬), 昨天 (体温)

Vitamin 怎么用 Extract?

  • 症状提取: 用户自由输入 → 自动填入症状清单
  • 用药信息提取: 识别药品名 + 剂量 + 开始时间
  • 生理数据提取: 从文字中抓出体温、体重、血压等数值

Extract 是最 "结构化" 的 Reading 模式 -- 输出可以直接存数据库, 供后续分析使用.

选对了模式, 接下来 PM 要关注什么? ↓
(7)

PM 需要关注的 3 个质量问题

AI 不是 100% 准确的. 每种 Reading 模式都有自己的 "出错方式", PM 要提前设计兜底:

📜 Summarize 的风险

💣 最怕什么?

漏掉关键信息. 用户提到 "吃了布洛芬", 但摘要里没体现.

🛡️ PM 怎么兜底?

  • 摘要下方展示 "查看原文" 链接
  • 涉及用药/症状, 用 Extract 辅助校验
  • 加 "这个摘要准确吗?" 用户反馈按钮

🏷 Classify 的风险

💣 最怕什么?

分错类. 用户其实很焦虑, AI 说 "平静". 导致下游推荐完全错位.

🛡️ PM 怎么兜底?

  • 展示 top-3 类别让用户选 (L3 增强)
  • 置信度低于阈值时标记 "不确定"
  • 收集用户修正数据, 持续优化分类模型

⚒️ Extract 的风险

💣 最怕什么?

两种错: 漏提取 ("布洛芬" 没抓到) 或多提取 (把 "不头疼" 提取成 "头疼").

🛡️ PM 怎么兜底?

  • 提取结果高亮显示原文中的对应部分, 让用户确认
  • 关键字段 (用药) 缺失时主动追问
  • 不依赖 Extract 做医疗决策, 仅用于辅助记录

PM 的通用原则: 无论哪种 Reading 模式, AI 读完之后, 都要给用户一个修正的机会. 这就是 U1 说的 Augmentation -- AI 给初稿, 人来确认.

Vitamin 的 5 个状态, 各用什么 Reading?

每个身体状态的用户数据不同, 需要的 Reading 模式也不同:

经期
⚒️ Extract 为主

提取: 经期天数、经量、症状 (痛经/头疼/情绪波动)

🏷 Classify 辅助

痛经严重度分级: 轻微/中等/严重

备孕
📜 Summarize 为主

每周排卵追踪数据汇总: 基础体温、LH试纸、白带变化

⚒️ Extract 辅助

提取: 排卵日预测依据、同房记录

孕期
⚒️ Extract 为主

提取: 产检数据 (血压/体重/胎心率)、不适症状

🏷 Classify 关键

症状严重度分级: 正常/需注意/立即就医

产后
🏷 Classify 为主

产后情绪筛查: 正常波动 vs 产后抑郁风险

📜 Summarize 辅助

恢复进度周报: 喂养记录 + 身体恢复 + 情绪变化

更年期
⚒️ Extract 为主

提取: 潮热频次、失眠天数、情绪变化、HRT 用药

📜 Summarize 辅助

月度症状变化趋势总结

核心洞察: Extract 在每个状态都用得到 -- 因为 Vitamin 的核心价值就是 "把用户的自由表达变成可追踪的结构化数据". 而 Classify 在高风险状态 (孕期/产后) 尤其关键, 因为它决定了 AI 后续的响应策略.
