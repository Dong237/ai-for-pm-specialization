Unit 6·
Build vs Buy vs API

三条路通向同一个目标. 选错了, 浪费几个月.

上一课用 ROI 选出了 Vitamin 最值得做的功能.

现在问题是: 怎么做?

自己训模型? 用 Coze 搭一个? 还是让开发调 API?

—— 5 个问题, 帮你选路.

10 步学完, 你会:
  • 知道 Build / Buy / API 三条路的成本、速度、控制权差异
  • 用 5 个问题快速定位"你该走哪条路"
  • 了解 2026 年的主流平台: Coze, Dify, 各大 API
  • 给 Vitamin 的每个功能选定实施路径
走起 →
(1)

同一个功能, 三个团队做法完全不同

假设三个健康 App 都想做"每日身体总结" 功能:

🏗️ Build

A 团队: 自训模型

招了 3 个 ML 工程师, 收集了 10 万条健康记录, Fine-tune 了一个专属模型, 私有化部署.

花了
¥50 万 + 6 个月
效果
极度定制, 数据隐私最强
🛒 Buy

B 团队: 用 Coze 搭

PM 自己在 Coze 上搭了一个 Bot, 接了知识库, 写了 Prompt, 发布到微信小程序.

花了
¥0 + 3 天
效果
能用, 但定制有限
🔌 API

C 团队: 调 API

1 个后端开发, 调 DeepSeek API, 写 Prompt + 接用户数据, 嵌入 App 原生功能.

花了
¥4 万 + 3 周
效果
产品级体验, 可定制

三个团队都做出了"每日身体总结". 但成本从 ¥0 到 ¥50 万, 时间从 3 天到 6 个月. 选错路, 就是浪费.

很多人有一个错误假设 ↓
(2)

两个错误假设, 都会让你踩坑

✗

"自己训模型 = 最专业"

"我们要有自己的模型, 这才显得技术实力强."

为啥不对: 2026 年, 42% 的企业 AI 项目被砍掉, 原因不是技术差, 而是选了太重的路. 大多数产品功能, 调 API 就够了. 自训模型应该是最后的选择, 不是第一选择.
✗

"用平台 = 不专业 / 玩具"

"Coze 搭的东西不算真产品, 上不了台面."

为啥不对: Coze 2025 年开源了核心组件, Dify 已经有 5 万+ GitHub Stars. 平台做 MVP 验证是最高效的选择. 先验证 idea, 再升级技术栈 — 这才是专业做法.
真相

没有"最好的路", 只有最匹配你当前阶段的路.

先 Buy 验证 → 再 API 上线 → 最后考虑 Build

→ 这个顺序让你用最低成本验证想法, 用最合理的投入做出产品级功能.

先看第一条路: Build ↓
(3)

路 1: Build (自训/自研)

自己训练或微调一个模型, 部署在自己的服务器上. 控制权最大, 但门槛也最高.

🏗️ Build 路线详解

优势
  • 数据完全掌控, 不经过第三方
  • 模型完全定制, 可以针对特定场景优化
  • 长期来看, 边际成本最低
  • 竞争壁垒最强 (别人抄不了你的模型)
风险
  • 初始投入 ¥10 万+ (算力 + 人力)
  • 需要 ML 工程师 (年薪 40-80 万)
  • 周期 3-6 个月, 不确定性大
  • 维护成本 = 初始成本的 5 倍 (长期)
什么时候走 Build 路:
  • AI 是你产品的核心竞争力 (不是辅助功能)
  • 你有独家数据可以训出别人训不出的模型
  • 涉及极度敏感的数据, 不能走第三方 API
  • 预算 > ¥10 万, 时间 > 3 个月, 团队有 ML 人才
Vitamin 适合 Build 吗?

目前不适合. Vitamin 的 AI 功能 (总结、Q&A、建议) 用通用大模型 + Prompt 就能做到 80% 的效果. 不需要自训模型.

→ 除非: 将来 Vitamin 积累了百万级用户的经期数据, 想训一个"最准的经期预测模型" — 那时候可以考虑.

下一条路: Buy ↓
(4)

路 2: Buy (SaaS/平台)

用 Coze、Dify 这样的无代码/低代码平台搭建 AI 功能. PM 自己就能做, 不需要开发.

🛒 Buy 路线详解

优势
  • 零代码, PM 自己就能搭
  • 几天就能上线 MVP
  • 成本极低 (免费 / ¥0-5K/月)
  • 内置知识库、对话流、多平台发布
风险
  • 定制能力有限 (平台给什么你用什么)
  • 用户体验受平台限制, 难以嵌入自有 App
  • 数据在第三方 (合规风险)
  • 依赖平台, 如果平台停服你就停服
什么时候走 Buy 路:
  • 你想快速验证一个 AI 功能 idea, 看用户是否真的需要
  • 团队没有开发者, PM 要自己搞定
  • 预算 < ¥1 万, 时间 < 2 周
  • 功能可以独立存在 (比如一个微信 Bot), 不需要嵌入 App
2026 年主流平台:
Coze (扣子)

字节跳动出品. 最低门槛, 模板丰富, 支持发布到微信/飞书/抖音.

免费额度 已开源
Dify

开源 LLM 应用平台. RAG 支持强, 多模型切换, 适合稍有技术基础的团队.

开源 5 万+ Stars
FastGPT

国产开源, 知识库问答为主. 适合做企业内部知识库 Bot.

开源
第三条路: API ↓
(5)

路 3: API (调 API)

让开发者直接调用大模型的 API, 在你自己的产品里实现 AI 功能. 这是大多数产品级 AI 功能的选择.

🔌 API 路线详解

优势
  • 完全嵌入自有产品, 用户体验最好
  • Prompt 和流程可深度定制
  • 可切换模型 (今天用 DeepSeek, 明天换 GPT)
  • 按量付费, 成本可控 (¥150-3K/月)
风险
  • 需要至少 1 个后端开发
  • 2-4 周开发周期
  • API 供应商可能涨价 / 改接口
  • 需要自己做异常处理和监控
什么时候走 API 路:
  • 你要把 AI 功能嵌入自己的 App, 不是独立 Bot
  • 团队有 1 个以上开发者
  • 预算 ¥1-10 万, 时间 2-8 周
  • 需要定制对话流程和 UI, 但不需要训自己的模型
主流 API 供应商 (2026):
DeepSeek API

性价比最高. 中文能力强, 价格是 GPT-4 的 1/10.

按量付费
OpenAI / GPT API

综合能力最强. 全球生态最完善, 但价格较高.

按量付费
豆包 / 通义 API

国产主力. 合规无忧, 中文优化好, 价格适中.

按量付费

最佳实践路径:

Buy (验证) → API (上线) → Build (壁垒)

第一步用 Coze 花 3 天搭个 MVP, 看用户反馈. 反馈好 → 用 API 做产品级功能. 数据量够大、想建壁垒 → 再考虑 Build.

怎么快速判断该走哪条路? 5 个问题 ↓
(7)

5 个问题, 定你的路线

不用纠结, 回答这 5 个问题, 答案自动浮现:

  1. 预算多少?
    < ¥1 万 → Buy  |  ¥1-10 万 → API  |  > ¥10 万 → 可以考虑 Build
  2. 多久要上线?
    < 2 周 → Buy  |  2-8 周 → API  |  > 2 个月 → Build 可接受
  3. 团队有开发者吗?
    没有 → Buy (PM 自己来)  |  有 → API 或 Build
  4. 需要深度定制吗?
    通用能力够 → Buy/API  |  需要定制模型 → Build
  5. 涉及敏感数据吗?
    是 → Build (私有部署) 或 API + 加密  |  否 → Buy/API 都行

规律: 大多数答案指向 Buy (验证期) 或 API (上线期). Build 只在特殊条件下才值得.

总结 + 作业 ↓
(10)

一句话规则 + 作业

一句话:

先 Buy 验证, 再 API 上线,
最后考虑 Build.

不要一上来就训模型. 用最低成本验证想法, 用最合理的投入做产品.
