## (1) Observable: 6 Vitamin features, limited budget
Can't build them all. Need a filter. That filter is ROI.

## (2) Wrong intuition vs truth
Wrong: "Just build what users ask for most"
Truth: Frequency ≠ value. A feature used 1000x at ¥0.01/use < a feature used 10x at ¥100/use.

## (3) ROI formula: 4 building blocks
ROI = (Time saved × Users affected × Value per save) / Implementation cost.
Animate each block.

## (4) Ongoing costs: the hidden drain
API bills, maintenance (prompt tuning, monitoring), opportunity cost.

## (5) Sensitivity analysis
Which variable has biggest impact? Usually user count.
Spider chart.

## (6) Break-even timeline
At what month does the feature pay for itself?

Rule: "ROI > 3x = green light. ROI < 1x = red flag. 1-3x = dig deeper."
4 tasks + resource table.
