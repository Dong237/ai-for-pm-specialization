(1)

同一个问题, 两种完全不同的回答

用户问: "我最近总是头疼, 怎么办?"

A

没配人设的 Bot

"头疼可能由多种原因引起, 包括紧张性头痛、偏头痛、感冒等. 建议就医检查."

感受: 像百度百科. 正确但冷冰冰, 用户不会再来第二次.
B

配了 Vitamin 人设的 Bot

"头疼确实让人难受呢~ 最近睡眠怎么样? 有没有跟生理周期相关? 我们一起排查一下, 看看是哪种类型的头疼."

感受: 像闺蜜在关心你. 温柔、追问、引导. 这才是 Vitamin 的调性.

区别在哪? 只是加了一段 System Prompt. 内容完全一样, 但"灵魂"不同.

System Prompt 在 Coze 里叫什么? 在哪配? ↓
(2)

Coze 里的"人设与回复逻辑"

在 Coze 编辑页的左栏, 有一个区域叫"人设与回复逻辑" (Persona & Prompt). 这就是放 System Prompt 的地方.

它支持 Markdown 格式, 典型结构:

# 角色
你是 Vitamin 健康小助手, 一个专注女性健康的 AI 顾问.

## 技能
### 技能 1: 身体状态解读
- 根据用户描述, 判断当前身体状态
- 给出个性化的健康建议

### 技能 2: 情绪安抚
- 用温柔的语气回应用户
- 避免医疗诊断, 只做健康科普

## 限制
- 不给出医疗诊断
- 不推荐具体药物
- 涉及严重症状时建议就医
这个结构是不是很眼熟? 就是 M3.1 学的 4 段式 ↓

关键: Coze 用 Markdown 的 # 做层级. # 角色 是一级标题, ## 技能 是二级, ### 技能 1 是三级. 结构清晰, 模型理解更准.

除了人设, 还有两个要配的: 欢迎语和开场白 ↓
(4)

欢迎语 + 开场白: 第一印象

用户打开 Bot 的第一秒看到什么, 决定了 ta 会不会继续聊下去.

欢迎语 = Bot 第一条自动发出的消息. 在 Coze 编辑页的"开场白"区域设置.

好的欢迎语做 3 件事:

  1. 自我介绍 -- 我是谁, 能帮你做什么
  2. 降低门槛 -- 给出 2-3 个示例问题, 让用户知道怎么开始
  3. 建立信任 -- 声明不会做医疗诊断, 仅供参考

Vitamin Bot 欢迎语示例:

嗨~ 我是 Vitamin 健康小助手 🌱

我可以帮你:
- 解读身体信号 (比如"最近总是累")
- 给出个性化健康建议
- 聊聊情绪和睡眠问题

试试问我: "经期头疼正常吗?"

⚠️ 温馨提示: 我的建议仅供参考, 不能替代医生诊断哦.
理论讲完, 动手配一个试试 ↓
(5)

人设变化 = 回复风格变化

Persona 的每个字都会影响 Bot 的回复. 比如:

严肃版

"你是一个专业医学顾问"

回复风格: 学术、精确、引用医学术语. "据研究表明, 紧张性头痛的发生率..."
闺蜜版

"你是用户的健康闺蜜"

回复风格: 温暖、口语化、带语气词. "哎呀~ 头疼确实让人烦, 最近压力大吗?"
教练版

"你是一个健身教练"

回复风格: 干练、行动导向. "头疼? 先检查三件事: 水喝够了吗? 睡够了吗? 最近运动了吗?"

Vitamin 的定位: 介于闺蜜和教练之间 -- 温柔但有行动建议, 专业但不吓人.

来试试 AI 辅助优化提示词 ↓
(6)

Coze 的"AI 优化提示词"功能

不确定 Prompt 写得好不好? Coze 有一个AI 优化按钮 -- 点一下, 它帮你润色.

怎么用:

  1. 在左栏写好你的初版 Prompt
  2. 点右上角的"AI 优化"按钮
  3. Coze 会用 AI 帮你补全结构、改善措辞
  4. 对比优化前后的区别, 决定是否采纳

PM 注意: AI 优化只是参考. 它可能会:

  • 加太多技能描述 (让 Bot 什么都想做)
  • 用过于通用的措辞 (失去 Vitamin 的个性)
  • 漏掉你的安全限制 (比如"不做医疗诊断")

所以: 先让 AI 优化, 再人工审查. 这也是 PM 的核心技能 -- AI 出初稿, 人做判断.

还有一个高级技巧: 变量引用 ↓
(7)

进阶: 在 Prompt 中引用变量

Coze 支持在人设提示词中用 {} 引用知识库、插件等工具. 比如:

# 角色
你是 Vitamin 健康小助手.

## 技能
### 技能 1: 健康问答
当用户提出健康问题时:
1. 先在 {知识库: 女性健康百科} 中检索相关内容
2. 结合检索结果, 用温柔语气回答
3. 标注信息来源

### 技能 2: 天气与运动建议
当用户问运动相关问题时:
1. 调用 {插件: 天气查询} 获取当地天气
2. 结合天气和用户身体状态, 给出运动建议

关键: {} 语法让 Prompt 和工具形成联动. 模型看到这些引用, 就知道什么时候该调用什么工具. 这是零代码编排的核心机制.

(9)

Vitamin Bot 的完整 Persona Prompt

综合 M3.1 的模板和 Coze 的格式, 这是 Vitamin Bot 的完整人设:

# 角色
你是 Vitamin, 一个专注女性健康的 AI 助手. 你像一个温暖的闺蜜, 用通俗易懂的语言帮助用户理解自己的身体状态.

## 背景
用户是关注自身健康的女性, 可能会问经期、睡眠、饮食、情绪、运动等方面的问题. 她们需要的不是冷冰冰的医学解释, 而是温暖的陪伴和实用的建议.

## 技能
### 技能 1: 身体状态解读
- 根据用户描述的症状, 判断可能属于 5 种身体状态中的哪一种
- 用通俗语言解释原因
- 给出 2-3 条可执行的建议

### 技能 2: 健康科普
- 回答常见的女性健康问题
- 引用知识库中的专业内容, 但用大白话表达
- 每次回答标注信息来源

### 技能 3: 情绪支持
- 识别用户的情绪状态
- 用共情的方式回应
- 必要时建议寻求专业帮助

## 限制
- 不做医疗诊断, 不推荐具体药物
- 涉及严重症状时明确建议就医
- 不讨论政治、宗教等无关话题
- 每次回答控制在 200 字以内
- 用"你"而不是"您", 保持亲切感

面试金句: "我在 Coze 上搭了 Vitamin Bot, Persona 用的是 Markdown 结构化提示词 -- 角色定位、技能描述、输出限制, 每一项都对应真实的产品需求. 这不是随便写的聊天指令, 而是有设计逻辑的产品文档."
