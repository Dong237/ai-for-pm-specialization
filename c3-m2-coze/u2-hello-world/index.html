程序员的第一课叫 Hello World

每个程序员入门的第一件事, 就是让电脑输出 "Hello World". 不是因为这句话有什么意义 -- 是因为跑通第一个端到端流程比什么都重要.

对 PM 来说, Hello World 就是:

  1. 打开 Coze

就这 4 步. 完成后, 你已经超过了 90% "只看不动手" 的 PM 学员.

这三项都可以后改, 不用纠结. 先填了再说.

你的 Bot 现在是一张白纸

刚创建的 Bot 能聊天, 但它只是在调用通用大模型回答问题. 它不知道 Vitamin 是什么, 也不会查任何数据.

✓ Hello World Bot 能做的

  • 通用闲聊 (聊天气、讲笑话)
  • 简单问答 (百科知识)
  • 文本生成 (写文案、翻译)

✗ 它还做不到的

  • 回答 Vitamin 相关的专业问题
  • 按你的风格和规则回答 (没配人设)
  • 查天气、查数据库 (没接插件)
  • 按流程处理复杂任务 (没建工作流)

这就是为什么我们需要后面 6 个单元:

U3 加人设 → U4 加知识库 → U5 加工作流 → U6 加插件 → U7 调试发布 → U8 组装成 Vitamin Bot

在加东西之前, 先了解两个重要概念 ↓
(7)

两个重要概念: 运行模式 & 模型选择

概念 1: 运行模式

  • 单 Agent 一个 Bot 处理所有问题 -- 大部分场景够用
  • 多 Agent 多个 Agent 分工协作 -- 复杂场景 (如 Vitamin 终版)

新手先用单 Agent. 等 U8 搭 Vitamin 时再考虑多 Agent.

概念 2: 模型选择

  • 豆包 -- 字节自家模型, 免费额度最大, 中文好. 首选
  • 通义千问 -- 阿里模型, 推理能力强
  • DeepSeek -- 性价比之王, 代码/推理能力好
  • Moonshot -- 长上下文场景好用

Vitamin Bot 建议用豆包 -- 成本低、中文好, 和 Coze 平台原生集成最好.

来看 3 个常见新手踩坑 ↓
(8)

新手常见 3 个坑

坑 1

Prompt 写在了错误的位置

System Prompt 应该写在左栏的"人设与回复逻辑"里, 不是在对话框里跟 Bot 说.
对话框里说的 = User Prompt, 每次对话都会变. 人设 = System Prompt, 是固定的.
坑 2

忘了选对话记忆长度

默认只记住最近几轮对话. 如果你要做多轮问诊, 记得调大"对话轮数"设置.
但也别无限大 -- 上下文越长, Token 消耗越多, 成本越高.
坑 3

创建后没在右栏测试

很多新手填完 Prompt 就去发布了, 结果效果很差.
右栏调试 = 你的第一层质量保证. 每改一次 Prompt, 都要在右栏测几轮.
