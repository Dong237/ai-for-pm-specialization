(3)

Step 1-2: Bot + Persona

创建 Bot:

  • 名称: "Vitamin 身体翻译器"
  • 描述: "专注女性健康的 AI 助手, 帮你解读身体信号"
  • 模型: 选豆包 (中文好, 免费额度大)
  • 粘贴 U3 的完整 Persona Prompt 到左栏
  • 设置欢迎语 (U4 的示例)

注意: 人设 Prompt 是最影响体验的部分. 如果只做一件事, 做好这个.

Step 3: 知识库 ↓
(4)

Step 3: 挂载知识库

3 个知识库:

  1. 经期健康百科 -- 上传 5 状态的医学科普文档
  2. 营养饮食指南 -- 上传饮食建议、食物营养表
  3. 运动与睡眠 -- 上传运动建议、睡眠改善内容

没有现成资料? 从百度百科 / 丁香医生 / 知乎科普复制几篇文章也行. MVP 阶段, 内容够用就行.

Step 4: 工作流 ↓
(5)

Step 4-5: 工作流 + 插件

工作流: 用 U5 的 5 节点设计:

开始 -> 意图识别 (LLM) -> 知识库检索 -> 回复生成 (LLM) -> 结束

如果时间紧, 可以先不建工作流 -- 直接用 Bot + 知识库也能工作, 工作流是加分项.

插件: 至少安装天气查询 + 搜索, 让 Bot 能回答"今天适合运动吗"这类依赖实时数据的问题.

Step 6-7: 调试 + 发布 ↓
(6)

Step 6-7: 调试 + 发布

调试 15 个 Case:

  • 5 种身体状态各 2 个问题 = 10 个
  • 2 个边界 case (医疗诊断 / 无关话题)
  • 3 个插件相关 (天气 / 计算 / 搜索)

发布:

  1. 发布到 Web SDK
  2. 把链接保存好 -- 这是你面试时要展示的
  3. 发给 2-3 个朋友测试, 收集反馈
