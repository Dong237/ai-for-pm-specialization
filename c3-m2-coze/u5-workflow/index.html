工作流: 让 Bot 按步骤办事

Bot 有了灵魂和记忆, 但复杂任务需要"按步骤来". 今天学工作流编排.

Bot 能聊天, 也能查知识库了...

但如果要"先判断意图, 再决定查哪个库"呢?

-- 这就是工作流干的事.

10 步学完, 你会:
  • 理解工作流 = 有向无环图 (DAG)
  • 认识 6 种节点类型
  • 搭建 Vitamin 的 5 节点工作流
  • 知道什么时候该用工作流, 什么时候不需要
走起 →
(1)

一个 Bot 处理不了的场景

用户问: "我经期第二天, 今天能跑步吗?" -- 这个问题需要 3 步才能回答好:

  1. 识别意图: 这是"运动建议"类问题
  2. 检索知识: 在知识库中找"经期运动"相关内容
  3. 生成回复: 结合知识 + Persona, 用温柔语气回答

没有工作流, Bot 只会一股脑把问题扔给大模型. 有了工作流, 每一步都有明确的节点负责, 质量可控.

工作流的核心概念: DAG ↓
(2)

工作流 = 有向无环图 (DAG)

3 个关键词:

  • 有向 -- 数据从左到右流, 不能倒流
  • 无环 -- 不能出现 A->B->C->A 的死循环 (循环节点除外)
  • 图 -- 节点 + 连线组成的网络

每个节点的执行条件: 所有输入边都有结果, 且至少一个成功. 这保证了数据按顺序流动.

认识 6 种节点类型 ↓
(3)

6 种节点类型

🧠 大模型节点
📚 知识库节点
💻 代码节点
🔀 条件判断
🔄 循环节点
🔌 插件节点

PM 最常用的 3 个:

  • 大模型节点 做文本生成、分类、总结 -- 工作流的"大脑"
  • 知识库节点 检索私有数据 -- 工作流的"记忆"
  • 条件判断 if-else 分支 -- 工作流的"路由"

怎么在 Coze 上实际搭? ↓
(5)

在 Coze 上创建工作流

操作步骤:

  1. 在 Bot 编辑页中栏, 点"工作流" -> "新建工作流"
  2. 进入可视化画布 -- 默认有开始和结束两个节点
  3. 从左侧面板拖入节点到画布
  4. 用连线连接节点 (从一个节点的输出拖到另一个的输入)
  5. 点击每个节点, 配置参数 (比如 LLM 节点要写 Prompt)
  6. 点"试运行"测试

PM 注意: 工作流搭建的核心不是"怎么拖拽", 而是"怎么拆解任务". 先在纸上画流程图, 再去 Coze 上搭, 效率高 10 倍.

每个节点怎么配? ↓
(6)

意图识别节点怎么配

LLM 节点的配置:

你是一个意图分类器. 根据用户消息, 判断意图类别.

只输出以下类别之一:
- health_query: 健康咨询类问题
- exercise: 运动相关问题
- diet: 饮食营养问题
- emotion: 情绪心理问题
- chitchat: 闲聊

用户消息: {{input}}
意图:

{{input}} 是变量, 运行时会被替换成用户的实际消息.

关键: 意图识别用T=0 (Temperature 最低) -- 分类任务需要确定性, 不要创意.

条件分支怎么加? ↓
(7)

进阶: 加条件分支

Vitamin 的进阶版工作流可以加一个条件判断 -- 不同意图走不同路径:

加了条件判断后:

  • 如果意图 = health_query -> 查健康知识库 -> 生成回复
  • 如果意图 = exercise -> 查运动知识库 + 调天气插件 -> 生成回复
  • 如果意图 = chitchat -> 直接用 LLM 闲聊, 不查知识库

好处: 闲聊类问题不查知识库, 省 Token + 省时间. 这就是 M2.2 学的模型分级路由在工作流中的实现.

工作流该不该用? 决策标准 ↓
(8)

什么时候用工作流, 什么时候不用

✓ 该用工作流

  • 任务有多个步骤 (先分类再检索再生成)
  • 需要条件分支 (不同意图走不同路)
  • 需要调用外部 API
  • 需要精确控制每步的输入输出

✗ 不需要工作流

  • 简单的一问一答 (直接用 Bot + 知识库)
  • 纯闲聊 (Bot + Persona 够了)
  • 还在探索阶段 (先跑通再加工作流)

PM 原则: 先简单, 后复杂. Vitamin 第一版用 Bot + 知识库就够了. 等验证了核心逻辑, 再加工作流做精细化控制.

Vitamin 视角 ↓
(9)

Vitamin 工作流规划

V1 (MVP): 开始 -> 知识库检索 -> LLM 生成 -> 结束 (3 节点, 够用)

V2 (进阶): 开始 -> 意图识别 -> 条件分支 -> 知识库/插件 -> LLM 生成 -> 结束 (5-6 节点)

V3 (终版): 多 Agent 协作, 每个 Agent 有自己的工作流 (U8 搭建)

面试金句: "Vitamin Bot 的工作流分三期迭代: MVP 只做知识检索 + 生成, 验证核心价值; 进阶版加意图路由和条件分支, 降低无效检索; 终版用多 Agent 协作, 每个 Agent 负责一个领域."
