给 Bot 喂知识

Bot 有灵魂了, 但还没有"记忆". 今天上传文档, 让它变成领域专家.

Bot 配了人设, 但问专业问题还是瞎编...

怎么让它回答 Vitamin 相关的医学知识?

-- 答案: 喂知识库.

10 步学完, 你会:
  • 在 Coze 上创建知识库并上传文档
  • 理解切片策略的区别 (段落 / 固定 / 语义)
  • 选择合适的检索策略 (混合 / 语义 / 全文)
  • 把知识库挂载到 Bot 上
走起 →
(1)

问一个 Bot 答不了的问题

试试问你的 Hello World Bot: "Vitamin 的 5 种身体状态分别是什么?"

结果

Bot 会一本正经地编一个答案.

因为它的训练数据里没有 Vitamin 的信息 -- 它在"猜"而不是"知道".

解决方案: 把 Vitamin 的资料上传到知识库. Bot 回答时先检索知识库, 再基于检索到的内容生成回复. 这就是 RAG (Retrieval-Augmented Generation).

Coze 的知识库怎么创建? ↓
(2)

Coze 知识库 = 内置 RAG

在 Coze 中栏的"工具"区域, 点击"添加知识库"就能开始.

创建知识库 3 步:

  1. 新建知识库 -- 取名"Vitamin 健康百科"
  2. 上传文档 -- 拖入 PDF / Word / TXT 文件
  3. 等待处理 -- Coze 自动切片 + 向量化 (通常 1-2 分钟)

支持的数据源:

  • 本地文件: PDF / Word / TXT / Markdown
  • 在线网页: 输入 URL, 自动抓取内容
  • 表格: Excel / CSV (适合 FAQ 数据)

    文件上传后, 最关键的一步: 切片 ↓
    (4)

    切片策略: 文档怎么切成小段?

    一份 10 页的 PDF 不能直接塞给模型 (太长了). 知识库会把它切成几十个小段 (chunk), 检索时只取最相关的几段.

    三种切片策略, 点击切换看效果:

    段落切分
    按自然段落分割, 保持语义完整
    固定字数
    每 300 字切一段, 简单粗暴
    语义切分
    按意思变化切, 最智能但最慢

    Vitamin 建议: 用段落切分. 医学科普文章通常按主题分段, 段落切分能保持每个健康知识点的完整性.

    Coze 默认推荐 ~300 字一段, 对中文健康内容来说刚好.

    切完了, 怎么检索? ↓
    (5)

    检索策略: 3 种找答案的方式

    混合检索

    全文 + 语义结合, 准确率最高. 推荐

    语义检索

    理解意思而非匹配关键词. 适合跨语言

    全文检索

    关键词匹配. 适合专有名词精确查找

    怎么选?

    • Vitamin Bot 用混合检索 -- 用户可能用各种方式描述症状, 需要语义理解 + 关键词匹配双保险
    • 默认返回 Top 5 最相关段落, 可以调整 (1-20)
    • 设置匹配率阈值 -- 低于阈值的结果不返回, 避免不相关内容干扰
    知识库建好了, 怎么挂到 Bot 上? ↓
    (6)

    挂载: 一键让 Bot 拥有知识

    知识库和 Bot 是分开的. 你需要在 Bot 编辑页手动挂载.

    挂载步骤:

    1. 进入 Bot 编辑页
    2. 在中栏找到"知识库"区域
    3. 点"添加" -> 选择你创建的知识库
    4. 完成! 现在 Bot 回答问题时会自动检索知识库

    小细节:

    • 一个 Bot 可以挂载多个知识库 (比如"经期知识" + "营养知识")
    • 在 Prompt 中用 {知识库: xxx} 可以指定什么时候检索哪个知识库
    • 知识库更新后, Bot 自动使用最新内容 -- 不用重新发布
    挂载后, 回答质量有多大变化? ↓
    (7)

    Before vs After: 知识库的魔力

    Before

    没有知识库

    "经期可以运动, 但要注意适度. 建议咨询医生."

    问题: 太笼统, 没有具体建议. 用户得不到有用信息.
    After

    有知识库

    "经期前两天建议轻柔运动 (散步、拉伸), 第 3-5 天可以恢复中等强度运动. 避免剧烈运动和倒立类动作哦~ [来源: 女性经期健康指南]"

    变化: 具体、分阶段、有来源标注. 用户信任度直线上升.

    关键认知: 知识库不只让回答更准确, 还让回答可溯源. 用户能看到"这个建议来自哪份资料", 信任度翻倍. 这对健康类产品至关重要.

    知识库维护有什么注意事项? ↓
    (8)

    知识库维护: 3 个常见坑

    坑 1

    文档太长, 切片太大

    一个 50 页的 PDF 如果不切, 检索精度会很差. 建议按主题拆成多个小文档再上传.
    坑 2

    信息过时不更新

    健康资讯会更新. 定期检查知识库内容是否过时, 及时替换. Coze 支持增量更新.
    坑 3

    质量参差不齐

    知识库内容质量决定回答质量. "垃圾进 = 垃圾出". Vitamin 的医学内容要确保来源可靠.
    回到 Vitamin -- 知识库该怎么规划? ↓
    (9)

    Vitamin 知识库规划

    建议创建 3 个知识库:

    知识库 内容 切片策略
    经期健康百科 5 状态解读、经期护理、常见问题 段落切分
    营养饮食指南 食物营养表、饮食建议、禁忌 固定 300 字
    运动与睡眠 运动建议、睡眠改善、情绪管理 段落切分

    面试金句: "Vitamin Bot 的知识库按 3 个主题域组织, 每个域选择不同的切片策略. 经期知识用段落切分保持语义完整, 营养表格用固定字数适应结构化数据. 检索用混合策略, Top 5 召回, 带来源标注让用户可溯源."

    一句话

    知识库 = Bot 的记忆.
    上传 -> 切片 -> 向量化 -> 检索 -> 回答, 5 步全自动.

    内容质量决定回答质量. 垃圾进 = 垃圾出.
