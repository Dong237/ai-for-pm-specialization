(1)

一个让你目瞪口呆的事实

2024 年初, Coze (扣子) 正式上线. 到 2026 年初, 它已经服务了数百万开发者和数万家企业. 但最让人惊讶的是 --

事实

用 Coze 搭一个能聊天的 AI Bot,

全程不写一行代码.

它是字节跳动出品的零代码 Agent 开发平台 -- 你只需要配置, 不需要编程.

对 PM 来说, 这意味着: 你可以自己做一个可演示的 Demo, 不用等研发排期.

面试时, 打开手机给面试官看一个真能跑的 Bot -- 这比说 100 句"我了解 AI"都管用.

那 Coze 的"零代码"到底是怎么做到的? 它有哪几块积木? ↓
(2)

错觉: "Coze 就是个聊天界面吧"

✗

错觉 A: Coze = ChatGPT 套壳

只是换了个界面调大模型, 没啥新东西.

为啥不对: ChatGPT 是一个通用对话工具. Coze 是一个开发平台 -- 你可以在上面建工作流、挂知识库、接 API, 做出完整的垂直应用.
✗

错觉 B: Coze 只能做聊天 Bot

最多做个客服机器人, 干不了复杂的事.

为啥不对: Coze 2.0 (2026 年 1 月发布) 已经升级为 Vibe Coding 平台 -- 能做 Agent、工作流、网站、甚至移动 App. 聊天只是它最基础的形态.
真相

Coze 是一个4 模块积木系统:

Bot + 工作流 + 知识库 + 插件

这 4 块积木可以自由组合, 从简单聊天到复杂 Agent 都能搭.

先来认识第一块积木 -- Bot ↓
(3)

积木 1: Bot -- 你的 AI 助手

Bot 是 Coze 的核心产物 -- 你搭的每个东西, 最终都以 Bot 的形式交付给用户.

一句话: Bot = Persona (人设) + 技能 (工具) + 发布渠道.

  • Persona: 你在 M3.1 写好的 System Prompt, 直接粘贴进去
  • 技能: 知识库 / 插件 / 工作流, 挂上就能用
  • 发布: 一键发到飞书、微信、抖音、Web SDK 等渠道

两种模式:

  • 单 Agent 一个 Bot 干所有事 -- 适合简单场景
  • 多 Agent 多个专长 Bot 协作 -- 适合复杂流程 (如 Vitamin 的"问诊 + 建议 + 记录"三角)
Bot 是"壳", 那"脑子"从哪来? 接知识库 ↓
(4)

积木 2: 知识库 -- 喂给 Bot 私有数据

大模型有通用知识, 但不知道你公司的产品文档. 知识库就是解决这个问题的 -- 本质上就是一个内置的 RAG 模块.

能喂什么进去?

  • 本地文件: PDF / Word / TXT / Markdown
  • 在线网页: 直接输入 URL, 自动抓取
  • 表格数据: Excel / CSV

它在背后做了什么?

  1. 把你的文档切片 (chunk) -- 默认 ~300 字一段
  2. 把每段话向量化 (embedding) -- 变成数学空间里的点
  3. 用户提问时, 用语义检索找到最相关的 5 段
  4. 把这 5 段塞给大模型, 让它"带着资料回答"

这就是 M3.3 要深入学的 RAG 三步走: Embed -> Retrieve -> Augment.

有了知识, 还得会"做事" -- 第三块积木, 插件 ↓
(5)

积木 3: 插件 -- 让 Bot 能"做事"

纯聊天的 Bot 只能"说". 但 PM 需要的 Bot 还得能"做" -- 查天气、查数据库、调 API、生成图片...

插件 = 一组 API 工具的封装. 三种来源:

  • 官方商店 60+ 预制插件, 直接安装 (搜索 / 天气 / 图片生成 / 计算器...)
  • 自定义 把你自己的 API 封装成插件 (比如 Vitamin 后端接口)
  • JSON/YAML 导入 如果你有 OpenAPI 规范文件, 直接导入

Vitamin 会用到的插件场景:

  • 接天气 API -- 回答"今天适合户外运动吗"
  • 接日历 API -- 记录用户的经期日期
  • 接搜索插件 -- 补充最新的健康资讯
四块积木的最后一块 -- 工作流, 把所有东西串起来 ↓
(6)

积木 4: 工作流 -- 把步骤串成链

当任务不只一步 (先识别意图 -> 再查知识库 -> 再生成回复), 你需要工作流.

工作流 = 一个有向无环图 (DAG). 每个节点做一件事:

  • 大模型节点 调 LLM 做文本生成 / 分类
  • 知识库节点 从知识库检索相关内容
  • 代码节点 写一小段 Python / JS 做数据处理
  • 条件判断 if-else 分支逻辑
  • 循环节点 重复执行某段流程
  • 插件节点 调用外部 API

(8)

4 模块的组合关系

Bot 是"壳", 其他三个是"内核". 它们的关系用一句话说清:

Bot = Persona + 工作流 + 知识库 + 插件

四块积木自由组合, 复杂度由你决定.

从简单到复杂的 4 档:

  1. 纯聊天 Bot + Persona 就够了 (Hello World)
  2. 带知识 Bot + Persona + 知识库 (客服 Bot)
  3. 能做事 Bot + Persona + 知识库 + 插件 (助理 Bot)
  4. 复杂流程 Bot + Persona + 工作流 (含知识库 + 插件节点) = Vitamin Bot

关键认知: 工作流 包含 知识库和插件 -- 知识库检索和插件调用都是工作流里的一个节点. 所以最终形态是: Bot 调用工作流, 工作流里编排知识库 + 插件 + LLM 节点.
