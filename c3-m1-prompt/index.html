从 Anthropic 4 段式结构到 Vitamin 5 状态模板库 — 9 个单元, 让你像工程师一样写 Prompt, 像 PM 一样管 Prompt.

学习路线

U1 四段式 → U2 System vs User → U3 Few-shot → U4 CoT → U5 XML Tags → U6 Negative → U7 版本管理 → U8 调试 → U9 Vitamin 模板
1

Anthropic 4 段式结构

Role + Task + Context + Output Format — 每个 Prompt 都该有这四段. 像填表一样写 Prompt, 质量立刻提升一个档次.

→
2

System vs User Prompt

System 是不变的"人设", User 是每次的"问题". 哪些放哪里, 决定可缓存性和成本.

→
3

Few-shot 示例的边际效用

0 → 1 → 3 → 5 个 example, 效果曲线递减. 3 个通常是性价比最高点.

→
4

Chain-of-Thought (CoT)

让模型"先想后答". 简单加一句 "step by step" 准确率能提 10-20%.

→
5

XML Tags 结构化

Anthropic 标志技巧: 用 XML 标签包不同部分, 模型理解更准, 输出更可控.

→
6

Negative Instructions

告诉模型"别做什么". Vitamin 安全机制的第一道防线, 但也要知道它的局限.

→
7

Prompt 版本管理

把 Prompt 当代码管理: 版本号、变更日志、A/B 实验. 生产级 AI 产品的标配.

→
8

Prompt 调试方法

模型答错了怎么 debug: 先确认输入, 再加 CoT, 再加 Few-shot, 最后换模型.

→
9

Vitamin 5 状态 Prompt 模板

用前 8 个单元学到的所有技巧, 为 Vitamin 5 个身体状态各写一个生产级 Prompt 模板.

→

学完这 9 个单元, 你能做到:

用 4 段式写出结构化 Prompt · 区分 System 和 User Prompt · 用 Few-shot 提升准确率 ·
用 CoT 让 AI "想清楚再答" · 用 XML 标签组织复杂 Prompt ·
设计安全边界 · 像管代码一样管 Prompt · 系统化 Debug · 产出 Vitamin 5 状态模板库
