AI 答错了, 怎么 Debug?

不是改一个字碰运气. 有一套系统方法: Observe → Reproduce → Analyze → Fix.

用户说 AI 回答"不靠谱", 但我不知道从哪里改...

随便改一个字, 有时更好有时更差

—— 4 步 Debug 法, 系统化定位问题.

10 步学完, 你会:

走起 →
(1)

"打地鼠"式调试: 修一个 bug, 冒三个

最常见的 Prompt 调试方式: 看到一个错误输出 → 在 Prompt 里加一句修补 → 推上线 → 过两天另一个场景又出问题了.

这叫 Whack-a-Mole Debugging:

没有分析根因, 只是修补表面症状. 每次修补可能破坏其他正常的场景.

系统的方法是什么? ↓
(2)

4 步 Debug 法: ORAH

方法论

Observe → Reproduce → Analyze → Hypothesize

→ 观察 → 复现 → 分析 → 假设并验证. 不是乱改, 是有假设地改.

每一步具体怎么做? ↓
(3)

Step 1: Observe — 记录出错的完整上下文

要记录什么:
  1. 完整的 System Prompt (当时用的版本号)
  2. 完整的 User Prompt (用户实际输入)
  3. 模型参数 (Temperature, Top-P, model name)
  4. AI 的完整输出 (不只是"错了", 要看全文)
  5. 期望的正确输出是什么
PM 提示:

很多时候"AI 答错了"不是 AI 的问题 — 是输入有问题. 比如用户的信息没有正确传递给 Prompt.

Step 2: 复现 ↓
(4)

Step 2: Reproduce — 在 Playground 里复现

复现方法:
  1. 把记录的 System + User Prompt 复制到 Playground
  2. 设置完全一样的参数 (Temperature, model, etc.)
  3. 跑 3-5 次 — LLM 是非确定性的, 可能要跑几次才能复现
  4. 如果复现了 → 继续分析
  5. 如果复现不了 → 可能是偶发的随机性, 或者输入有差异
Step 3: 分析 ↓
(5)

Step 3: Analyze — 定位问题出在哪一层

3 层定位法:

输入层 用户的信息有没有正确传到 Prompt? 有没有被截断?

Prompt 层 指令是否清晰? 有没有歧义? 有没有遗漏规则?

模型层 同样的 Prompt 换一个模型还错吗? 如果模型 A 错、模型 B 对, 说明是模型能力问题.

快速诊断技巧:

让 AI "解释"它为什么那样回答: "请解释你在上一步输出中的推理过程." 注意不要问"你为什么错了" — AI 会直接道歉, 没有诊断价值.

Step 4: 修复 ↓
(6)

Step 4: 按优先级尝试修复

修复顺序 (从简单到复杂):
  1. 确认输入: 检查数据传递有没有 bug (最常见的原因!)
  2. 加 CoT: 让 AI "先想后答", 很多推理错误一步解决
  3. 加/改 Few-shot: 给一个类似的正确示例
  4. 改指令: 让 Task 或 Rules 更明确
  5. 换模型: 如果以上都不行, 可能是模型能力不足

每次只改一个变量. 同时改两个, 你不知道是哪个起了作用.

修复后必须做的事 ↓
(7)

回归测试: 修了这个, 别破坏那个

最怕的事:

为了修一个边界案例, 加了一条规则, 结果 80% 的正常场景受影响.

黄金测试集 (Golden Set):

准备 20-30 个测试用例, 覆盖常见场景和边界场景. 每次修改 Prompt 后, 全部跑一遍.

→ 不需要复杂工具, 一个 Excel 表格就够: 输入 | 期望输出 | 实际输出 | Pass/Fail.

动手试: 4 步 Debug 流程

点击每个步骤, 执行诊断动作, 看 Debug 结果:

Debug 场景: AI 把"经前期"错误分类为"经期"

观察: 记录完整上下文

用户输入: "最近头疼, 想吃甜的, 心情不好"
AI 输出: {"state":"经期","mood":"低落"}
期望输出: {"state":"经前期","mood":"焦躁"}
Prompt 版本: v1.2.0 · 模型: Claude Sonnet · Temperature: 0.3
已记录. 关键发现: Prompt 中没有针对"经前期 vs 经期"的区分示例.

复现: 在 Playground 中测试

复制 Prompt 到 Playground, 设置 Temperature=0.3, 跑 5 次...
结果: 5 次中有 3 次输出"经期", 2 次输出"经前期".
已复现. AI 在两个状态间犹豫, 说明它没有足够的信号来区分.

分析: 定位问题层

检查 3 层:
1. 输入层: 用户信息正确传递 ✓
2. Prompt 层: Few-shot 中没有"经前期"的示例 ← 问题在这里!
3. 模型层: 换 GPT-4o 测试, 同样出错 → 不是模型问题, 是 Prompt 问题.
根因: Few-shot 示例中缺少"经前期"的典型案例, AI 把 PMS 症状默认归类为"经期".

修复: 添加经前期示例

修复方案: 在 Few-shot 中加一个经前期示例:
输入: "头疼想吃甜的心情不好" → 经前期 · 焦躁 · 饮食建议
跑黄金测试集 (25 个用例): 24/25 通过 ✓
修复成功! 加了 1 个经前期示例 → 分类准确率从 60% 提升到 96%. 版本号更新为 v1.3.0.
