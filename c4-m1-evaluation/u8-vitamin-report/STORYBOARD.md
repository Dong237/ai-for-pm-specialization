# U8 Storyboard: Vitamin 评测报告 (CAPSTONE)

## START
Hook: 7 个单元学完, 你手里有了: 三层架构 + Golden Set + LLM Judge + Pairwise + P/R/F1 + A/B 测试 + Pipeline. 现在, 把它们组装成一份完整的评测报告 — 这份报告, 90% 的转型 PM 拿不出来.

## (1) 为什么要写评测报告
面试拿出来 = 核武器级加分. 证明你不只会 "做" AI 产品, 还会 "验证" AI 产品.

## (2) 报告的 7 个章节
评测框架 / Golden Set / Judge 结果 / A/B 测试 / 指标汇总 / Pipeline 配置 / 结论与建议.

## (3) 第一章: 评测框架
三层架构图 + Vitamin 每层的具体指标定义.

## (4) 第二章: Golden Set
50 条样本统计: 真实/边缘/攻击占比, 覆盖的场景分类.

## (5) 第三章: Judge 结果 + Pairwise
LLM Judge 三维评分平均 + Pairwise v1 vs v2 对比结果.

## (6) 第四章: P/R/F1 + A/B 测试
RAG 召回率/精确率 + Prompt A/B 测试 p 值和效应量.

## (7) 第五章: Pipeline + 结论
自动化配置 + 总结论: Vitamin 评测达标/未达标, 改进建议.

## (8) WIDGET: Report Assembler
7 个 Tab 对应 U1-U7, 每个预填 Vitamin 数据. 点 "Generate Report" 生成结构化报告预览.

## (9) 报告怎么用
简历里写 "搭建评测体系, 产出完整报告"; 面试时 3 分钟讲结果.

## (10) 规则 + 作业 + 资源
规则: "评测报告 = 你的 AI PM 能力证明. 做了就赢了 90% 的竞争者."
Bridge to M4.2: 评测证明了好用, 但 '好用' ≠ '安全'. 下一个模块: 安全机制.
