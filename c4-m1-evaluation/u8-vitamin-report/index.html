①

为什么要写评测报告

评测报告的三个价值:

1. 面试核武器 — 90% 的转型 PM 没有评测报告. 你有, 你就赢了. 它证明你不只会 "做" AI 产品, 还会 "验证" AI 产品.

2. 团队信任 — 工程师看到你能拿出数据, 而不是说 "用户反馈还行", 信任度翻倍.

3. 决策依据 — 上线 / 回滚 / 继续优化, 全靠这份报告的数据说话.

报告长什么样? ↓
②

报告的 7 个章节

1评测框架
三层架构图 + Vitamin 每层的具体指标定义 (来自 U1)
2Golden Set
50 条样本统计: 真实/边缘/攻击占比, 覆盖场景 (来自 U2)
3Judge 结果
LLM Judge 三维评分 + Pairwise v1 vs v2 对比 (来自 U3 + U4)
4指标汇总
P/R/F1 + A/B 测试 p 值和效应量 (来自 U5 + U6)
5Pipeline 配置
自动化流程 + 上线门槛 + 报警机制 (来自 U7)
6典型 Case
Top 3 好 Case + Top 3 坏 Case + 改进建议
7结论与建议
总判定 (达标/未达标) + 下一步优化方向
每章写什么? 逐章展开 ↓
③

第一章: 评测框架

内容:

🔸 三层架构图 (主观 / 客观 / 业务)

🔸 Vitamin 每层的具体指标及目标值

🔸 评测方法: LLM Judge + Pairwise + Pipeline

篇幅: 半页. 用图 + 表, 不要长段文字.

第二章 ↓
④

第二章: Golden Set

内容:

🔸 Golden Set 版本号: v1.2

🔸 总条数: 50 条

🔸 分类分布: 真实 35 (70%) / 边缘 10 (20%) / 攻击 5 (10%)

🔸 覆盖场景: 月经期 / 排卵期 / 备孕 / 营养 / 情绪

🔸 标注方式: 2 人标注 + LLM 辅助

篇幅: 半页. 一个饼图 + 一个表格.

第三章 ↓
⑤

第三章: Judge 结果 + Pairwise

内容:

🔸 Judge 配置: Claude Sonnet, G-Eval, 三维打分

🔸 三维均分: 准确性 4.1 / 安全性 4.6 / 友好度 4.3

🔸 Pairwise: v2 vs v1, 50 轮, v2 胜率 62%

🔸 偏见缓解: 双向评 + 每周人工抽检 10%

篇幅: 一页. 三个分数仪表盘 + Pairwise 胜率图.

第四章 ↓
⑥

第四章: 指标汇总 + A/B 测试

内容:

🔸 RAG 指标: Precision 72% / Recall 84% / F1 77.5%

🔸 A/B 测试: v2 胜率 62%, p = 0.03 (显著)

🔸 效应量: 均分差 +0.4 分 (实际显著)

🔸 分场景: 真实问题 v2 赢 68% / 边缘 v2 赢 50% / 攻击 v2 赢 80%

篇幅: 一页. 指标表 + p 值标注.

第五-七章 ↓
⑦

第五-七章: Pipeline + Case + 结论

第五章 · Pipeline 配置:

5 阶段流程图 + 触发方式 + 上线门槛 + 成本 (¥4.20/轮)

第六章 · 典型 Case:

Top 3 好 Case (高分示例) + Top 3 坏 Case (低分示例) + 每条的改进建议

第七章 · 结论与建议:

总判定: ✅ 达标, 建议上线 v2

下一步: 1) 边缘 Case 覆盖率需提升 2) Recall 从 84% 优化到 90% 3) 增加多轮对话测试

60s: "三层架构: 用 Claude 做 Judge 评质量, 监控延迟成本, 看接受率留存."

60s: "50 条 Golden Set, A/B 测试 Prompt v2 显著更好, p=0.03."
