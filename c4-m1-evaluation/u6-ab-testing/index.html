①

凭感觉改 Prompt 的代价

最常见的 Prompt 迭代: 改完试 3 条, "感觉不错, 上!"

问题: 3 条样本的结论, 统计学上毫无意义. 就像掷 3 次硬币都是正面, 你能说硬币是双面正面吗?

也许第 4 条就翻车了. 也许这 3 条恰好是 v2 擅长的类型.

小样本 + 选择性偏差 = 自欺欺人.

正确的做法 ↓
②

A/B 测试的核心逻辑

核心

控制组 (Prompt v1) vs 实验组 (Prompt v2)

同样的 Golden Set, 比结果.

唯一变量是 Prompt. 一切其他条件 (模型 / 温度 / 测试集) 保持不变.

为什么要 50 条? ↓
③

为什么 3 条不够

统计学 101:

当两个版本的差异很小 (比如 v2 只比 v1 好 10%), 你需要足够多的样本才能区分 "真的好" 和 "碰巧好".

经验法则:

🔸 差异很大 (v2 赢 90%) → 20 条就够

🔸 差异适中 (v2 赢 65%) → 50 条是实用下限

🔸 差异很小 (v2 赢 55%) → 需要 200+ 条

大多数 Prompt 改进在 "适中" 范围, 所以 50 条 Golden Set 是起步线.

什么是 p 值? ↓
④

p 值: 一句话解释

定义

p 值 = "如果 v1 和 v2 其实一样好,

观察到这么大差异的概率."

p 越小, 说明 "碰巧" 的可能性越低, 差异越可信.

判定标准:

🟢 p < 0.05 → 统计显著. "v2 大概率真的更好, 不是碰巧."

🟡 0.05 < p < 0.1 → 边界. "有点信号, 但不够强. 建议增加样本."

🔴 p > 0.1 → 不显著. "可能是碰巧的. 不能下结论."

实操流程 ↓
⑤

A/B 测试操作流程

5 步操作:

1. 准备 Golden Set — 50+ 条, 覆盖真实/边缘/攻击

2. 运行 v1 — Golden Set 全部跑一遍, 收集输出

3. 运行 v2 — 同样的 Golden Set 再跑一遍

4. LLM Judge 评分 — 两组输出分别评分, 或 Pairwise 比较

5. 计算 p 值 — 看差异是否显著, 做决策

关键: 步骤 2 和 3 要同时用同一个模型, 避免模型更新导致不公平比较.

常见陷阱 ↓
⑥

三个常见陷阱

陷阱 1: 偷看中间结果

跑了 20 条 v2 赢了, 就停下来宣布胜利. 这叫 "p-hacking" — 如果继续跑, v1 可能追上来.

陷阱 2: 只看平均分

v2 平均分 4.2 vs v1 的 4.0. 但 v2 的方差是 v1 的 3 倍 — 它不稳定, 有时 5 分有时 2 分.

陷阱 3: 忘了测边缘 Case

v2 在真实问题上赢了, 但在攻击样本上全崩. 如果 Golden Set 不含攻击, 你发现不了.

统计显著 ≠ 实际显著 ↓
⑦

统计显著 vs 实际显著

关键区分:

🔸 统计显著: p < 0.05, 差异不是碰巧的.

🔸 实际显著: 差异大到值得上线.

例子: p = 0.01 (统计显著), 但 v2 只高 0.1 分 (4.1 vs 4.0). 为了这 0.1 分的提升, 值得改吗?

如果改 Prompt 意味着增加 20% token 消耗 (成本上升), 0.1 分的提升可能不值.

决策公式: 统计显著 + 效应量足够大 + 没有副作用 (成本/延迟不变) = 上线.

Vitamin Prompt A/B 测试实操

Vitamin A/B 测试计划:

🔸 Golden Set: 50 条 (35 真实 + 10 边缘 + 5 攻击)

🔸 v1: "你是健康助手" (通用版)

🔸 v2: "你是 Vitamin 女性健康 AI 助手" (专业版)

🔸 评测: Claude Judge, 三维打分 + Pairwise

🔸 判定标准: p < 0.05 且 v2 三维均分 ≥ v1, 安全分不下降

面试金句: "每次改 Prompt 我都会跑 A/B 测试: 50 条 Golden Set, 用 LLM Judge 评分, 算 p 值. 只有统计显著且安全分不下降才上线. 这是 PM 用科学方法做产品决策."

规则 + 作业 ↓
⑩

一句话规则 + 今日作业

一句话:

改 Prompt 不 A/B, 等于蒙眼开车.

50 条 Golden Set + p < 0.05 + 无副作用 = 上线通行证.

1. 用 ChatGPT 写两版 Vitamin 健康 Prompt (通用版 vs 专业版), 各跑 5 条同样的问题, 记录你觉得哪个更好.

2. 如果 5 条里 v2 赢了 3 次, 你能说 v2 更好吗? 为什么? (用 p 值思维分析)

3. 为 Vitamin 写一份 A/B 测试计划 (100 字): Golden Set 来源 / 评测方式 / 判定标准 / 上线决策流程.
