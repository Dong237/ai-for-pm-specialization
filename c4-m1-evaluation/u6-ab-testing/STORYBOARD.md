# U6 Storyboard: A/B 测试 in Prompts

## START
Hook: 你改了 Prompt, Judge 打分高了 0.3 分. 这是真的提升, 还是随机波动? 要回答这个问题, 你需要 A/B 测试.

## (1) 凭感觉 vs 凭数据
改 Prompt 跑了 3 条都变好了, 就上线? 也许第 4 条会翻车. 小样本不可信.

## (2) A/B 测试的核心逻辑
控制组 (Prompt v1) vs 实验组 (Prompt v2), 同样测试集, 比结果.

## (3) 样本量: 为什么 3 条不够
统计学 101: 效应量小时需要更多样本. 50 条是实用下限.

## (4) p 值是什么
"如果 v1 v2 其实一样好, 观察到这么大差异的概率." p < 0.05 = 显著.

## (5) 实际操作流程
准备 Golden Set → 分别跑 v1 和 v2 → 收集分数 → 算 p 值 → 决策.

## (6) 常见陷阱
偷看中间结果提前终止, 只看平均分忽略方差, 忘了测边缘 Case.

## (7) 统计显著 vs 实际显著
p < 0.05 但分数只高 0.1 分? 统计显著但实际没意义. 要看效应量.

## (8) WIDGET: A/B Test Runner
两版 Prompt 展示. 点 "Run Test" 模拟 50 条评测. 柱状图显示胜率. p 值指示器. 判定: "v2 显著更好" 或 "需要更多数据".

## (9) Vitamin Prompt A/B 测试
Vitamin 场景: v1 "你是健康助手" vs v2 "你是女性健康AI助手, 专注月经/营养/运动". 跑 50 条比较.

## (10) 规则 + 作业 + 资源
规则: "改 Prompt 不 A/B, 等于蒙眼开车."
