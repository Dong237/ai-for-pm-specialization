①

手动评测的痛

手动流程 (每轮 2 小时):

1. 打开 Golden Set CSV → 复制一条 → 粘贴到 AI → 复制输出 → 粘贴回 CSV (× 50 条 = 1 小时)

2. 把 50 条输出发给 Judge → 整理分数 → 算 P/R/F1 → 对比上次 (30 分钟)

3. 写报告: 截图 + 数据 + 结论 (30 分钟)

一天改 3 版 Prompt = 6 小时纯体力活.

自动化 = 5 分钟搞定同样的事 ↓
②

Pipeline: 一条命令搞定

核心

自动化 Pipeline =

一条命令, 5 个阶段自动跑完

加载 → 推理 → 评分 → 计算 → 报告. 人只需要看报告.

5 个阶段分别是什么? ↓
③

Pipeline 五阶段总览

Stage 1: 加载 Golden Set — 从 CSV/JSON 读取 50 条用例

Stage 2: 运行推理 — 逐条调 API, 收集模型输出

Stage 3: LLM Judge 评分 — Claude 对每条输出三维打分

Stage 4: 计算指标 — 算 P/R/F1, 平均分, 与上次对比

Stage 5: 生成报告 — 输出 Markdown 报告, 含好/坏 Case 示例

类比: 像工厂流水线. 原材料 (Golden Set) 进去, 成品 (报告) 出来. 中间每一步都是自动的.

Stage 1-2 详解 ↓
④

Stage 1-2: 加载 + 推理

Stage 1 — 加载 Golden Set:

输入: golden_set.csv (input / expected / category)

输出: 内存中的测试用例列表

要点: 支持版本号, 自动检查格式

Stage 2 — 运行推理:

输入: 50 条用户问题 + 当前 Prompt

输出: 50 条模型回答 + token 消耗 + 延迟

要点: 并发控制 (避免 API 限流), 超时重试, 记录每条的成本

Stage 3-4 ↓
⑤

Stage 3-4: 评分 + 计算

Stage 3 — LLM Judge 评分:

输入: 50 条 (问题 + 模型回答 + 理想输出)

输出: 每条三维分数 (准确性 / 安全性 / 友好度) + 理由

要点: 用 G-Eval 方法, Judge 先列理由再打分

Stage 4 — 计算指标:

输入: 150 个分数 (50 条 × 3 维) + 上次报告

输出: 均分, P/R/F1, 与上次对比 (Δ), 异常检测

要点: 如果某个维度分数下降 > 10%, 自动标红预警

Stage 5: 报告 ↓
⑥

Stage 5: 报告生成

自动报告包含:

📊 总览: 三维均分 + P/R/F1 + 总成本

📈 趋势: 与上次 / 上上次对比的变化

✅ Top 3 好 Case: 模型回答最好的 3 条 (含问题和输出)

❌ Top 3 坏 Case: 模型回答最差的 3 条 (含问题和输出)

⚠️ 预警: 安全分 < 3 的条目列表

💰 成本: 本轮评测总花费

CI/CD 集成 ↓
⑦

CI/CD: 像测代码一样测 Prompt

工程师改代码会自动跑测试. PM 改 Prompt 也应该自动跑评测.

CI/CD 集成思路:

1. 把 Prompt 存在 Git 仓库, 每个版本一个文件

2. 每次 push 新 Prompt, 自动触发 Pipeline

3. Pipeline 跑完, 结果自动发到飞书 / 钉钉群

4. 如果安全分 < 3 或均分下降 > 10%, 自动阻止上线

PM 不需要自己写代码. 你需要做的是:

1. 维护 Golden Set (这是你的活)

2. 定义 Judge Prompt 和评分标准 (这也是你的活)

3. 设定 "上线门槛" (均分 ≥ 3.5, 安全 ≥ 4.0)

4. 让工程师搭 Pipeline (给他上面的规格)

Vitamin Pipeline 配置 ↓
⑨

Vitamin Pipeline 配置

Vitamin 评测 Pipeline 规格:

🔸 Golden Set: 50 条 CSV, 版本 v1.2

🔸 推理模型: 豆包 Lite (主力) + DeepSeek V4 (对比)

🔸 Judge: Claude Sonnet, G-Eval, 三维打分

🔸 指标: 三维均分 + P/R/F1 + 成本 + 延迟

🔸 上线门槛: 均分 ≥ 3.5, 安全 ≥ 4.0, P95 延迟 < 5s

🔸 触发方式: 每次改 Prompt 手动触发 / 每日定时跑

🔸 通知: 报告发到飞书群, 不达标 @PM

面试金句: "我给 Vitamin 搭了自动化评测 Pipeline: 每改一次 Prompt 自动跑 50 条 Golden Set, Claude Judge 评分, 5 分钟出报告. 安全分低于 4 自动阻止上线. 成本每轮不到 5 块钱."

规则 + 作业 ↓
⑩

一句话规则 + 今日作业

一句话:

Prompt 改一次, Pipeline 跑一次. 没有例外.

5 分钟 + ¥5 = 一份完整评测报告. 没理由手动.
