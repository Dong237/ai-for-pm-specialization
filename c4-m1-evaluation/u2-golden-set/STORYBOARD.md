# U2 Storyboard: Golden Set 构建

## START
Hook: 你写了一条 Prompt, 改了一版, 怎么知道哪版更好? 靠试 3 条? 3 条能代表什么? 你需要一个 "标准答案库" — Golden Set.

## (1) 没有 Golden Set 时的痛
改 Prompt 凭感觉, 改完不知道是变好还是变差. 经典翻车: 改好了 A 类问题, 改坏了 B 类.

## (2) Golden Set 是什么
50-100 条精选测试用例, 每条有 input + 理想 output + 分类标签. 它是评测的地基.

## (3) 三类样本: 真实 + 边缘 + 攻击
真实: 用户最常问的 (70%). 边缘: 模糊/罕见 (20%). 攻击: 恶意/越界 (10%).

## (4) 怎么收集真实样本
从用户日志抽样, 按频率排序, 覆盖核心场景.

## (5) 怎么设计边缘 Case
找 "一般不会想到但真会遇到" 的情况: 空输入, 超长输入, 中英混输, 敏感话题.

## (6) 怎么设计攻击样本
Prompt 注入, 越狱尝试, 要求违规内容. 测试安全边界.

## (7) 标注理想输出的原则
不追求唯一正确, 而是标注 "可接受范围". 多人标注取交集.

## (8) WIDGET: Golden Set Builder
分类面板 (真实/边缘/攻击), 拖拽健康问题样本到对应类别. 雷达图实时显示覆盖度分布. 目标: 平衡覆盖.

## (9) Vitamin Golden Set 实例
Vitamin 50 条 Golden Set 结构: 月经期饮食 / 排卵期运动 / 敏感话题 / 注入攻击 等.

## (10) 规则 + 作业 + 资源
规则: "Golden Set 的质量 = 评测体系的上限. 垃圾进, 垃圾出."
