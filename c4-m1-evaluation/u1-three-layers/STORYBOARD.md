# U1 Storyboard: 评测三层架构

## START
Hook: 你的 Vitamin Bot 上线了. 老板问: "效果好吗?" 你说 "用户反馈还行". 老板追问: "什么叫还行? 给我数据." 你愣住了.
Preview: 掌握三层评测架构 / 区分主观-客观-业务指标 / 知道每层该测什么 / 动手给 Vitamin 分配指标

## (1) 观察: "效果好" 是最危险的一句话
场景: PM 拿不出数据, 只靠 "感觉不错" 汇报 AI 效果. 三个翻车案例.

## (2) 错觉 vs 真相
错觉 A: 只要用户没投诉就是好. 错觉 B: 跑个 benchmark 就够了.
真相: AI 评测必须三层同时看 — 主观打分 + 客观指标 + 业务指标, 缺一不可.

## (3) 第一层: 主观评分 (人评 / LLM 评)
人类打分 1-5 分, 或用 LLM 当 Judge. 评的是 "输出质量好不好".

## (4) 第二层: 客观指标 (延迟 / 成本 / 格式)
P50/P95 延迟, 单次成本, 输出格式合规率. 评的是 "系统表现稳不稳".

## (5) 第三层: 业务指标 (留存 / 接受率 / 转化)
用户接受建议率, 7 日留存, 转人工率. 评的是 "产品目标达不达".

## (6) 三层为什么缺一不可
案例: 质量高但太慢 (缺第二层), 便宜快但用户不买账 (缺第三层), 留存好但内容有毒 (缺第一层).

## (7) 三层之间的关系
第一层是因, 第三层是果, 第二层是约束. 画出因果链.

## (8) WIDGET: 评测金字塔
交互金字塔: 三层可点击展开. 拖拽 Vitamin 指标 (用户接受建议率 / 平均延迟 / LLM 评分等) 到正确层.

## (9) Vitamin 案例
Vitamin 三层各放什么: 主观 (LLM Judge 评健康建议质量), 客观 (P95 延迟 < 5s, 单次 < ¥0.02), 业务 (建议接受率 > 40%).

## (10) 一句话规则 + 作业 + 资源
规则: "AI 效果好不好, 永远用三层数据回答, 别用嘴回答."
