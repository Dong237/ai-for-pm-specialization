Cost per Successful Task

每 token 便宜 19 倍的模型, 实际可能更贵 — 因为它失败了 3 倍.

老板说 "换个便宜模型省钱" ...

(1)

$/token 是错误的指标

你在选模型, 两个选择:

模型 A (便宜)

$0.005/1K tokens
但成功率只有 40%
平均需要 2.5 次才成功

模型 B (贵)

$0.03/1K tokens
成功率 85%
通常 1 次就成功

模型 A 每 token 便宜 6 倍. 但:

  • 模型 A: 平均 2.5 次调用 × $0.005 = $0.0125 / 成功任务
  • 模型 B: 平均 1.18 次调用 × $0.03 = $0.035 / 成功任务

模型 A 还是便宜! 但考虑到用户流失, 每次失败用户可能走了 — 便宜模型的隐性成本远不止 token 费.

CPST 定义 ↓
(2)

定义: Cost per Successful Task

公式

CPST =
总花费 (含失败) / 成功任务数

Total cost = 所有 API 调用的 token 费 + 重试费 + 工具调用费 + 基础设施费.

关键: 分母是成功完成的任务, 不是总调用次数. 失败的调用也在烧钱, 但不创造价值.

CPST 包含什么 ↓
(3)

CPST 的 "全部成本" 包含什么

CPST 的完整分子:

  1. Input tokens 费: prompt / context 的成本
  2. Output tokens 费: 生成回复的成本
  3. 重试成本: 失败后再调用一次 (或多次)
  4. 工具调用费: Agent 调用搜索、数据库等
  5. 缓存写入摊销: Prompt Caching 的一次性写入成本
  6. 人工审核成本: 需要人检查的场景 (按时薪摊)

只报 token 费的团队, 通常低估真实成本 40-70%.

为什么不能只看 token 价 ↓
(4)

案例: 换便宜模型后 CPST 反而涨了

一个真实的模式:

Before (用 GPT-4o):

  • Token 成本: $0.03/次
  • 成功率: 85% → 平均 1.18 次/成功任务
  • CPST = $0.03 × 1.18 = $0.035

After (换 GPT-4o-mini):

  • Token 成本: $0.003/次 (便宜 10 倍!)
  • 成功率: 55% → 平均 1.82 次/成功任务
  • Regen Rate 也上升: 额外 30% 调用
  • CPST = $0.003 × 1.82 × 1.3 = $0.007

token 价格降了 10 倍, CPST 只降了 5 倍. 而且用户体验变差了, 用户流失带来的隐性成本没算.

结论: 换模型时, 不比 $/token, 比 $/成功任务 + 用户体验.

降低 CPST 的正确方式 ↓
(5)

降低 CPST 的四个策略

  1. 提高成功率 (分母变大): 优化 prompt, 让第一次就成功. 这是杠杆最大的.
  2. 减少重试 (分子变小): 降低 Regen Rate, 减少无效调用.
  3. Model Routing: 简单任务用便宜模型, 复杂任务用贵模型. 不是一刀切.
  4. Prompt Caching: 相似问题复用缓存, 省掉重复的 context 成本.

策略 3 最实用: Vitamin 的 "今天天气如何" 用 mini 模型就行, "我这个症状严不严重" 必须用大模型.

CPST 和毛利率 ↓
(6)

CPST 直接影响毛利率

AI 产品的毛利率比传统 SaaS 低得多:

  • 传统 SaaS: 毛利率 70-85%. 边际成本接近零.
  • AI-Native 产品: 毛利率 50-65%. 每个用户都在烧 token.
  • 早期 AI 产品: 毛利率可能只有 25%. 为了增长在牺牲利润.

CPST 越高, 毛利率越低. CPST 翻倍 = 你赚的钱减半 (至少).

所以 CPST 不只是技术指标, 是直接跟财务报表挂钩的商业指标.

怎么跟老板汇报 ↓
(7)

怎么跟老板讲 CPST

老板问: "为什么不换个便宜模型?"

你的回答:

"我算过了. 便宜模型 token 价格低 10 倍, 但成功率从 85% 降到 55%, 重试率上升 30%. 综合算下来, 每个成功任务的成本只降了 5 倍, 而用户体验明显变差, 可能导致留存下降."

"我的建议是: 不一刀切换模型, 而是做 Model Routing — 简单任务用便宜模型, 复杂任务保留好模型. 这样 CPST 降 40%, 用户体验不变."

核心: 用 CPST 而不是 $/token 来做模型选型的 ROI 分析.
