Unit 3·
Acceptance Rate

用户拿到 AI 的输出, 直接用了没改 — 这就是一次 "Accept".

AI 给了建议, 用户是直接用了, 还是改了才用?

这个比例到底怎么算? 多少算好?

—— Acceptance Rate, 质量指标里的核心.

10 步学完, 你会:
  • 理解 Acceptance Rate 的定义和计算方式
  • 知道不同产品类型的健康基准线
  • 看出 Acceptance Rate 太高或太低的问题
  • 给 Vitamin 设定合理的 Acceptance Rate 目标
走起 →
(1)

一个真实场景

GitHub Copilot 2024 年的数据: AI 生成的代码建议, 开发者接受率不到 44%.

也就是说, 超过一半的 AI 建议被开发者丢掉了. 这正常吗?

正常. 44% 的接受率在 AI 代码生成领域已经是很好的数字了.

因为代码要求精确, 用户(开发者)标准高. 而像邮件摘要这种场景, 接受率可以到 60-70%.

→ Acceptance Rate 的 "好不好", 取决于你的产品类型和场景.

什么是 Acceptance Rate ↓
(2)

定义: 用户直接用了, 没改

公式

Acceptance Rate =
直接采纳次数 / 总输出次数

"直接采纳" = 用户看到 AI 输出后, 没有修改、没有重新生成、没有删除, 直接使用.

三种用户反应:

  • Accept (采纳): 直接用了 → 计入 Acceptance Rate 分子
  • Edit (编辑): 改了再用 → 不计入 (但也不算完全失败)
  • Reject (拒绝): 不用, 重新生成或手写 → 不计入
怎么判断高低? ↓
(3)

基准线: 不同场景差很多

Acceptance Rate 没有一个通用的 "好" 数字. 看场景:

产品场景
健康 AR
邮件草稿 / 文案
60-70%
客服回复建议
50-60%
代码补全
35-45%
健康建议 (Vitamin)
40-50%
医疗诊断辅助
20-30%

规律: 风险越高, 用户越谨慎, Acceptance Rate 越低. 这是正常的.

太高也有问题 ↓
(4)

反直觉: 太高反而危险

Acceptance Rate 90%+ 在健康产品里不是好事 — 说明用户不加审核就接受 AI 建议.

⚠️

AR 太低 (< 20%)

AI 输出太差, 用户每次都要改或扔掉.

→ 需要优化 prompt、改模型、或调低 Temperature.
⚠️

AR 太高 (> 80%)

在健康领域, 意味着用户盲目信任 AI.

→ 如果 AI 有幻觉, 用户也直接采纳了. 需要加审核提示.

Vitamin 的甜蜜区间: AR 40-55%. 意味着用户会看一眼、有选择地采纳. 不是盲目信任, 也不是完全不信.

怎么提高 AR ↓
(5)

提高 Acceptance Rate 的四个手段

AR 低了怎么办? 不是把按钮变大让人更容易点 "接受" — 是让 AI 输出真的更好.

  1. 优化 Prompt: 加更多上下文 (用户历史、偏好). AR 往往能提升 10-15%.
  2. 调低 Temperature: 减少创意, 增加确定性. 健康场景用 T=0.2-0.3.
  3. 加 Few-shot Examples: 在 prompt 里给几个 "好输出" 的例子.
  4. 用户个性化: 记住用户上次的偏好, 下次更贴近. AR 随使用次数上升.

核心思路: AR 是果, 输出质量是因. 不要在 AR 上做文章 (比如隐藏 "编辑" 按钮), 要在输出质量上做文章.

AR 和 Edit Distance ↓
(6)

AR 的好搭档: Edit Distance

Acceptance Rate 只告诉你 "用了没改", 但 "改了" 这件事也有深浅:

Edit Distance (编辑距离): 用户在 AI 输出上做了多大的修改.

  • 低 Edit Distance: 改了个标点 / 换了个词 → AI 基本到位
  • 高 Edit Distance: 重写了 80% → AI 基本没用

两个指标一起看:

  • 高 AR + 低 Edit Distance = 完美
  • 低 AR + 低 Edit Distance = AI 输出不错但用户习惯性修改 (可能是 UI 问题)
  • 低 AR + 高 Edit Distance = AI 输出不行 (需要优化模型)
怎么在技术上追踪 ↓
(7)

技术实现: 怎么追踪 Acceptance Rate

AR 不是用户自己报告的, 是你在产品里埋点追踪的:

埋点方案:

  1. AI 输出展示时, 记录 event: ai_output_shown
  2. 用户直接使用 → ai_output_accepted
  3. 用户编辑后使用 → ai_output_edited + 记录编辑比例
  4. 用户关闭/忽略 → ai_output_dismissed
  5. 用户点重新生成 → ai_output_regenerated

AR = accepted / (accepted + edited + dismissed + regenerated)

注意: "用户看了 30 秒然后什么都没做" 也要考虑. 设一个超时 (比如 2 分钟无操作 = dismissed).

一句话 + 作业 ↓
(10)

一句话规则 + 今晚作业

一句话:

Acceptance Rate = 用户信任的温度计.
太低 = AI 不行, 太高 = 用户盲信.
健康产品的甜蜜区间: 40-55%.
