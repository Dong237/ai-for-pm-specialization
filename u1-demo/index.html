(1)

先承认: LLM 是有点神奇的

在拆解它怎么工作之前, 先看三件事 —— 它真的能做到, 而且五年前没人觉得它能.

事 1

写一首七言绝句, 主题"深圳春天"

鹏城三月柳含烟,
深湾水暖鹭鸶眠.
莲花山上花如海,
一夜东风到大鹏.
押韵, 平仄基本对, 还提到了真实地名
事 2

这段 Python 报错了, 帮我看看

def total(items):
    return sum(item.price for item in items)
# AttributeError: 'dict' object 
# has no attribute 'price'
→ AI 答: "items 是 dict 列表, 用 item['price']" ✓ 答对了
事 3

把这段中文翻译成法语 (带语气)

原: 啊呀, 你又迟到了!

→ Ah là là, tu es 
    encore en retard !
不只是字面翻译 —— 抓住了"啊呀"的撒娇/责备语气
凭什么???
三个不同领域 (诗 / 代码 / 翻译), 一个 AI 都搞定了. 它内部到底在干嘛 ↓
(2)

两个常见错猜测 —— 都不对

✗

猜测 A: AI "懂了" 中文/Python/法语

像人一样, 它有"理解" 这件事在脑子里发生.

为啥不对: 它会一本正经地编不存在的论文; 算 17×23 错; 同一句话两次问答案不一样. 真"懂"的东西不会这样.
✗

猜测 B: AI 是个超级数据库

它存了全互联网的答案, 你问啥它查啥.

为啥不对: "深圳春天" 这首七绝不在任何数据库里 — 是它当场写的. 数据库不会"创作", 它只能"检索".
真相

LLM 在做一件事, 反复地做:

"猜下一个字最可能是啥."

就这么简单. 不"懂" 也不"查". 就是猜下一个字.

→ 七绝是猜了 28 个字. Python 修复是猜了一段代码. 翻译是猜了一句法语. 都是同一件事.

"猜下一个字" 怎么就能猜出诗? 下一步用一个具体例子拆开 ↓
(3)

把这事拆给你看 — 半句话, 你来猜

先别想 AI, 想你自己. 我把这句话写一半:

输入:
今天天气真___

你脑子里立刻冒出来一堆候选 —— 不止一个对的答案, 是一堆答案的可能性排序:

好 棒 不错 糟 闷 饿
"好" 大概是你最先想到的. "饿" 你也想得到, 但你知道它非常不可能.
这就是"概率分布": 一堆候选, 各有可能性高低.

现在把同一句话喂给 LLM:

↓ 送进 LLM (140GB 的模型) ↓
出来的不是"一个字", 而是一张概率表 —— 下一步给你看 ↓
(4)

AI 给的不是答案, 是概率表

候选下一个字
概率
好
60%
棒
20%
不
8%
糟
5%
闷
3%
饿
1%
… (剩下 5 万个候选都接近 0)

注意 3 件事 ——

  1. AI 给的是所有候选的概率, 不只是 top 1
  2. "好" 60% 不是因为 AI 觉得"好" 是对的, 是因为训练时见过太多 "天气真好" 的句子
  3. 这张表跟你脑子里的排序差不多. 这就是它"像人" 的根源
那 60% 这个数到底怎么算出来的? ↓
(5)

60% 怎么来的 — 训练 + 推理两阶段

阶段 1 · 训练 (一次性, 训练完冻住)
  1. 读完互联网 ~10 TB 文本 (维基百科 + 书 + 论坛 + 代码)
  2. 每读一句话, 遮住一个字, 让模型猜
  3. 猜错了, 调一下脑子里的"参数" (140 GB 的数字)
  4. 重复 ~一万亿次. 训练完, 这 140 GB 就是模型的"全部记忆"
→ 出来一个 140 GB 的文件 (模型权重)
↓ 训练完了之后 ↓
阶段 2 · 推理 (你每用一次, 跑一次)
  1. 你输入"今天天气真__"
  2. 模型查它的 140 GB 记忆, 给所有候选打分
  3. "好" 在训练时跟 "天气真" 共现 60% → 输出 60%
  4. "饿" 几乎没共现过 → 输出 1%
→ 输出: 概率表
PM 关键认知: 训练 = 一次性的天文成本 (GPT-4 据估计 $1 亿). 推理 = 每次调用的小钱 ($0.001 量级).
这就是为啥"模型 API 按次收费, 不卖整个模型".
一个字猜完了, 怎么生成一整段话? ↓
(6)

一个字接一个字 — Autoregressive 接龙

猜完一个字, 把它追加到原句尾, 再猜下一个. 循环.

今天天气真|
轮 0 / 11
这个循环有个学名: Autoregressive (自回归).

"七绝" 不是一次输出 28 个字 — 是猜了 28 次, 每次一个 token, 把上一次的输出当下一次的输入.

→ 这也解释了为什么 ChatGPT 是一个字一个字蹦出来的, 不是"想好了一次发". 因为它真的就是一个字一个字算的.

回到开头那三件事 — 现在能解释了 ↓
(7)

回过头看那三件事 — 全是同一件事

写七绝
→
见过几百万首唐诗 + 现代诗. 概率分布偏向"押韵 + 五/七字".
改 Python
→
见过几亿行代码 + Stack Overflow 报错-修复对. 概率分布偏向"修复模式".
中→法翻译
→
见过亿级中法对照. 概率分布偏向"语义对齐 + 语气保留".

一句话:

LLM 把"互联网上各种风格的写作模式"
压缩进了 140 GB.

你给它前半句, 它按记忆里的模式接后半句. 就这么个事儿.

那 "概率最高的字" 一定是它输出的字吗? 不一定 ↓
(8)

Temperature — 让 AI "严谨" 还是 "放飞"

概率表是固定的. 但从这张表里挑哪个字, 是可以调的. 这个调节旋钮叫 Temperature.

0 严谨 0.7 默认 1.5 狂野
输入: "今天天气真"
→ 好.
T=0.7 时, AI 大概率输出"好" (top), 偶尔"棒". 自然中带点变化.
Vitamin 选档建议:
  • T=0.2 给身体状态总结 / 健康建议 (要稳, 不能瞎编)
  • T=0.7 跟用户对话 / 情绪安抚 (默认值, 自然)
  • T=1.2 生成趣味文案 / 起名 (要点新意)
理解了 LLM 怎么干活, 现在把它对照 Vitamin ↓
(9)

回到 Vitamin — 哪些能交给 LLM, 哪些不能

一句话规则: "模式" 它擅长. "事实" 它不擅长.

✓ 能交给 LLM

  • 把用户的"我有点累" 改写成 3 种温柔提醒
  • 根据用户记的早餐, 生成今日饮食小结
  • 用户问"经期能跑步吗", 给出参考性回答
  • 把医学术语翻译成大白话
  • 识别用户输入的情绪, 分类到 5 状态
  • 给一段对话总结 3 个关键点

✗ 不能交给 LLM

  • 算用户 BMI (它会算错小数)
  • 判断"用户能不能服 X 药" (医疗诊断, 法律风险)
  • 查"今天深圳天气几度" (实时数据, 它训练时不知道)
  • 记住用户上周三说过啥 (除非你显式塞进 context)
  • 生成"绝对正确"的健康建议 (它会自信地编)
  • 处理用户身份证号 / 隐私数据 (合规风险)
面试金句, 抄走:

"LLM 擅长生成式任务 (改写 / 分类 / 总结), 不擅长精确计算 / 实时数据 / 长期记忆. 所以我设计 Vitamin 时, 把数学算 BMI 这种交给规则, 把'用户输入分类到5状态' 这种交给 LLM."

→ 这个判断力, 就是 AI PM 跟普通 PM 的差别.

作业 + 一句话总结 ↓
(10)

一句话规则 + 今晚作业

一句话:

LLM 是一个 140 GB 的"模式压缩器" ——
把互联网的写作模式压进权重, 你给前半句, 它按模式接后半句.

看 AI 产品时, 永远先问: 这是"模式" 任务还是"事实" 任务?

课后作业 (今晚 30 分钟):
  1. 打开 ChatGPT / 豆包, 让它帮你写一段"Vitamin 早餐建议". 看 5 次, 每次它说的有哪些"模式相同, 措辞不同" 的地方.
  2. 问它一个事实题: "深圳今天最高气温几度?" 看它怎么答 (大概率会编).
  3. 把第 9 步的"能 / 不能" 对照 Vitamin 现有功能, 圈出来 3 个你之前以为该用 AI 但其实该用规则的地方.
  4. 写 200 字: 用今天学的"猜下一个字" 概念解释给一个非技术朋友听.

→ 第 4 项是面试自我介绍的素材库. 写下来.

一句话记住 Unit 1:

LLM = 一个用 140 GB 装下"全互联网写作模式" 的猜字机.

它不懂. 它没在查. 它在按模式猜下一个字, 一遍遍.

下节课 (Unit 2): Token

今天我们一直说 "猜下一个字" — 这是个善意的小谎言.
AI 看的不是"字", 是"token". 同一句中文, 在 GPT 眼里是 3 块, 在 DeepSeek 眼里是 4 块.

这件事直接决定你 Vitamin 一个月烧多少钱.
